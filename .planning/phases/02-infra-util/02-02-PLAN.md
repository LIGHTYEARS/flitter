## URI 类

```typescript
class URI {
  readonly scheme: string;
  readonly authority: string;
  readonly path: string;
  readonly query: string;
  readonly fragment: string;

  constructor(scheme: string, authority: string, path: string, query: string, fragment: string);

  // Factory methods
  static parse(value: string): URI;
  static file(path: string): URI;
  static from(components: { scheme?: string; authority?: string; path?: string; query?: string; fragment?: string }): URI;

  // Accessors
  get fsPath(): string;  // Convert file:// URI to OS path

  // Transforms
  with(change: { scheme?: string; authority?: string; path?: string; query?: string; fragment?: string }): URI;

  // Serialization
  toString(): string;
  toJSON(): object;

  // Path utilities
  static joinPath(uri: URI, ...paths: string[]): URI;
  static dirname(uri: URI): URI;
  static basename(uri: URI): string;
  static extname(uri: URI): string;
}
```

## 内部解析规则

```typescript
// RFC 3986 解析正则
// /^(([^:/?#]+?):)?(\/\/([^/?#]*))?([^?#]*)(\?([^#]*))?(#(.*))?/
//   scheme = $2
//   authority = $4
//   path = $5
//   query = $7
//   fragment = $9

// Scheme 验证正则
// /^\w[\w\d+.-]*$/

// fsPath 转换规则:
// - Unix: authority + path (通常 authority 为空)
// - Windows: 如果 path 以 /X: 开头 (盘符)，去掉前导 / 并转换 / 为 \
// - UNC path: //authority/path

// toString 编码规则:
// - scheme + "://" + 编码 authority + 编码 path
// - 如有 query: + "?" + 编码 query
// - 如有 fragment: + "#" + 编码 fragment
// - 路径中的 / 和 : 不编码
```

创建 `packages/util/src/uri/uri.ts`，实现完整 URI 类:

**构造函数与属性:**
- `constructor(scheme, authority, path, query, fragment)` — 所有组件均为只读
- Scheme 验证: 必须匹配 `/^\w[\w\d+.-]*$/`，验证失败抛出错误

**工厂方法:**
- `static parse(value: string): URI` — 使用 RFC 3986 正则解析
  - 正则: `/^(([^:/?#]+?):)?(\/\/([^/?#]*))?([^?#]*)(\?([^#]*))?(#(.*))?/`
  - 提取 scheme, authority, path, query, fragment
- `static file(path: string): URI` — 文件路径转 file:// URI
  - Windows 路径 (含盘符 C:\): 转换 \ 为 /，添加前导 /
  - Unix 路径: 直接使用
  - scheme = "file", authority = ""
- `static from(components)` — 从组件对象创建，缺省组件默认空字符串

**访问器:**
- `get fsPath(): string` — file:// URI 转 OS 路径
  - 有 authority: `//authority/path` (UNC 路径)
  - Windows 盘符路径 (`/C:/...`): 去掉前导 /，转换 / 为 \
  - 其他: 直接返回 path

**变换:**
- `with(change)` — 返回新 URI，仅覆盖指定组件

**序列化:**
- `toString()` — 编码后拼接完整 URI 字符串
  - scheme + "://" + encode(authority) + encodePath(path)
  - 路径中 / 和 : 不编码，其他特殊字符百分号编码
- `toJSON()` — 返回 { scheme, authority, path, query, fragment, $mid: 1, fsPath?, external? }

**路径工具:**
- `static joinPath(uri, ...paths)` — 合并路径段，规范化 ../ 和 ./
- `static dirname(uri)` — 返回父目录 URI
- `static basename(uri)` — 返回最后一段路径名
- `static extname(uri)` — 返回扩展名 (含点号)

1. **URI.parse — 标准 URI**
   - `http://example.com/path?query=1#frag` — 完整 URI
   - `https://user:pass@host:8080/path` — authority 含用户信息和端口
   - `file:///home/user/file.txt` — Unix file URI
   - `file:///C:/Users/file.txt` — Windows file URI
   - `custom-scheme://host/path` — 自定义 scheme
   - 仅 path: `relative/path`

2. **URI.file — 路径转 URI**
   - Unix 绝对路径: `/home/user/file.txt`
   - Windows 路径: `C:\Users\file.txt`
   - 含空格路径: `/path/with spaces/file.txt`

3. **URI.from — 组件创建**
   - 完整组件
   - 部分组件 (缺省默认空字符串)

4. **fsPath — URI 转 OS 路径**
   - Unix file URI → Unix path
   - Windows file URI → Windows path (含反斜杠)
   - UNC path: `file://server/share` → `//server/share`

5. **with — 不可变变换**
   - 修改 scheme 保持其他不变
   - 修改 path 保持其他不变

6. **toString — 序列化**
   - round-trip: parse → toString → parse 产生相同 URI
   - 特殊字符正确编码
   - 路径中 / 不被编码

7. **joinPath — 路径合并**
   - 基本合并: `http://host/a` + `b/c` → `http://host/a/b/c`
   - 规范化 `..`: `http://host/a/b` + `../c` → `http://host/a/c`
   - 规范化 `.`: `http://host/a` + `./b` → `http://host/a/b`

8. **dirname — 父目录**
   - `file:///a/b/c.txt` → `file:///a/b`
   - `file:///a` → `file:///`

9. **basename — 文件名**
   - `file:///a/b/c.txt` → `c.txt`
   - `file:///a/b/` → 空字符串或 `b`

10. **extname — 扩展名**
    - `file:///a/b/c.txt` → `.txt`
    - `file:///a/b/c` → 空字符串
    - `file:///a/b/.hidden` → 空字符串或 `.hidden`

11. **边界情况**
    - 空字符串 parse
    - 仅 scheme: `http:`
    - 含特殊字符的 query 和 fragment
    - IPv6 authority: `http://[::1]:8080/path`
