an external-tool-first strategy (rg/fd) and a pure NodeJS fallback. The scanner must respect
ignore patterns (gitignore-style), always-include overrides, depth/file limits, symlink policy,
and abort signals. All results are returned as ScanEntry objects with file:// URIs.

<interfaces>
```typescript
interface ScanEntry {
  uri: string;           // file:// URI
  path: string;          // absolute path
  name: string;          // basename
  isDirectory: boolean;
  isAlwaysIncluded: boolean;
}

interface ScanOptions {
  maxFiles?: number;           // default 50000
  maxDepth?: number;           // max recursion depth
  followSymlinks?: boolean;    // default false
  ignorePatterns?: string[];   // gitignore-style globs
  alwaysIncludePaths?: string[];
  abortSignal?: AbortSignal;
}

interface ScanResult {
  entries: ScanEntry[];
  scannedFiles: number;
  scannedDirectories: number;
  truncated: boolean;     // hit maxFiles limit
}

// External scanner (rg/fd)
async function scanWithExternalTool(root: string, options?: ScanOptions): Promise<ScanResult>;

// NodeJS fallback scanner
async function scanWithNodeJS(root: string, options?: ScanOptions): Promise<ScanResult>;

// Main scanner (tries external first, falls back to NodeJS)
class FileScanner {
  constructor(roots: string[], options?: ScanOptions);
  async scan(): Promise<ScanResult>;
  async initialize(): Promise<void>;
}
```
</interfaces>

<tasks>
1. **Create `packages/util/src/scanner/file-scanner.ts`** — FileScanner with external + NodeJS fallback
   - Implement `ScanEntry`, `ScanOptions`, `ScanResult` interfaces
   - Implement `scanWithNodeJS(root, options)`:
     - Recursive `fs.readdir` with `withFileTypes: true`
     - Respect `maxFiles` (stop early, set `truncated: true`)
     - Respect `maxDepth` (track current depth, skip deeper)
     - Apply `ignorePatterns` using a gitignore-style glob matcher (e.g. picomatch/minimatch)
     - Force-include `alwaysIncludePaths` even if they match ignore patterns; mark `isAlwaysIncluded: true`
     - Handle `followSymlinks` via `fs.stat` vs `fs.lstat`
     - Catch per-entry EACCES/EPERM and skip gracefully
     - Check `abortSignal` between iterations
   - Implement `scanWithExternalTool(root, options)`:
     - Detect `rg` (ripgrep --files) or `fd` availability via `which`
     - Build CLI args respecting ignore patterns and maxDepth
     - Parse stdout lines into ScanEntry objects
     - Fall back to `scanWithNodeJS` on spawn failure
   - Implement `FileScanner` class:
     - `constructor(roots, options)` — store roots and merged options with defaults
     - `initialize()` — detect available external tool once
     - `scan()` — iterate roots, merge results, enforce global maxFiles cap
   - Build `ScanEntry.uri` as `file://` + absolute POSIX path
   - Export all public types and the `FileScanner` class

2. **Create `packages/util/src/scanner/file-scanner.test.ts`** — comprehensive tests
   - **NodeJS scanner tests:**
     - Scans flat directory, returns correct entries with uri/path/name/isDirectory
     - Respects `maxFiles` — truncates and sets `truncated: true`
     - Respects `maxDepth` — does not recurse beyond limit
     - Applies ignore patterns (e.g. `*.log`, `node_modules/`)
     - `alwaysIncludePaths` overrides ignore patterns
     - Handles empty directories (returns no file entries)
     - Handles permission-denied directories (skips, no throw)
     - Handles symlinks: skipped by default, followed when `followSymlinks: true`
     - Aborts mid-scan when `abortSignal` fires
   - **External scanner tests:**
     - Falls back to NodeJS when external tool not found
     - Parses external tool output into ScanEntry objects
   - **FileScanner class tests:**
     - Merges results from multiple roots
     - Global maxFiles cap across all roots
     - `initialize()` caches tool detection
   - **Edge cases:**
     - Root path does not exist — returns empty result, no throw
     - Very long file paths
     - Special characters in file names
     - URI encoding for paths with spaces

3. **Update barrel exports**
   - Add scanner exports to `packages/util/src/index.ts`
</tasks>
