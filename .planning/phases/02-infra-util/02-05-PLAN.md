<objective>
Implement the Keyring credential storage module with two backends: a file-based JSON store
(FileSecretStore) for universal compatibility and a native OS keychain store (NativeSecretStore)
for enhanced security. Both implement a common SecretStore interface with reactive change
notification via a Subject. The file-based store must handle concurrent access safely and
enforce restrictive file permissions.
</objective>

<interfaces>
```typescript
interface SecretStore {
  get(key: string, url: string): Promise<string | undefined>;
  set(key: string, value: string, url: string): Promise<void>;
  delete(key: string, url: string): Promise<void>;
  changes: Subject<void>;  // emits when secrets change
}

// File-based storage — stores as JSON { "<key>@<normalized-url>": "<value>" }
class FileSecretStore implements SecretStore {
  constructor(secretsFilePath: string);
  get(key: string, url: string): Promise<string | undefined>;
  set(key: string, value: string, url: string): Promise<void>;
  delete(key: string, url: string): Promise<void>;
  changes: Subject<void>;
}

// Native keyring storage — uses OS keychain
class NativeSecretStore implements SecretStore {
  static async create(): Promise<NativeSecretStore | null>;  // returns null if native not available
  get(key: string, url: string): Promise<string | undefined>;
  set(key: string, value: string, url: string): Promise<void>;
  delete(key: string, url: string): Promise<void>;
  changes: Subject<void>;
}

// Factory function
async function createSecretStore(options: {
  secretsDir: string;
  useNativeKeyring: boolean;
}): Promise<SecretStore>;
```
</interfaces>

<tasks>
1. **Create `packages/util/src/keyring/keyring.ts`** — FileSecretStore + NativeSecretStore + factory
   - Implement `SecretStore` interface
   - Implement URL normalization helper:
     - Strip trailing slashes
     - Lowercase scheme and host
     - Compose storage key as `<key>@<normalized-url>`
   - Implement `FileSecretStore`:
     - `constructor(secretsFilePath)` — store path, initialize `changes` Subject
     - Internal `_readStore()` — read + parse JSON, return empty object if file missing
     - Internal `_writeStore(data)` — atomic write (write to temp file, rename) with 0600 permissions
     - `get(key, url)` — read store, lookup by composed key
     - `set(key, value, url)` — read store, set key, atomic write, emit changes
     - `delete(key, url)` — read store, delete key, atomic write, emit changes
     - Use advisory file locking (e.g. `proper-lockfile` or simple .lock file) for concurrent access
   - Implement `NativeSecretStore`:
     - `static async create()` — attempt dynamic import of native keyring module; return null on failure
     - `get/set/delete` — delegate to native module with composed key as service name
     - Emit `changes` on set/delete
   - Implement `createSecretStore(options)`:
     - If `useNativeKeyring`, try `NativeSecretStore.create()`
     - If native unavailable or not requested, create `FileSecretStore`
     - Ensure secrets directory exists (mkdir -p)
   - Export all public types, classes, and factory function

2. **Create `packages/util/src/keyring/keyring.test.ts`** — tests (file-based focus)
   - **FileSecretStore tests:**
     - `set` then `get` returns the stored value (round-trip)
     - `get` on missing key returns `undefined`
     - `delete` removes the key; subsequent `get` returns `undefined`
     - JSON file format: keys are `<key>@<normalized-url>`
     - URL normalization: trailing slashes stripped, scheme+host lowercased
     - Multiple keys for different URLs stored independently
     - Multiple keys for the same URL stored independently
     - `changes` Subject emits on `set()`
     - `changes` Subject emits on `delete()`
     - File created with 0600 permissions (Unix only)
     - Atomic write: file is not corrupted on concurrent writes (simulate with Promise.all)
     - Handles missing secrets file gracefully (auto-creates)
     - Handles empty/corrupt JSON file (recovers or throws meaningful error)
   - **NativeSecretStore tests:**
     - `NativeSecretStore.create()` returns `null` when native module unavailable
   - **Factory tests:**
     - `createSecretStore({ useNativeKeyring: false })` returns FileSecretStore
     - `createSecretStore({ useNativeKeyring: true })` falls back to FileSecretStore when native unavailable
     - Factory creates secrets directory if it does not exist
   - **Edge cases:**
     - Empty string key or URL
     - Very long key values
     - Special characters in key and URL

3. **Update barrel exports**
   - Add keyring exports to `packages/util/src/index.ts`
</tasks>
