<objective>
Implement a fuzzy file search module that ranks file entries using a multi-tier scoring
strategy: exact match, prefix match, suffix match, substring match, and fuzzy (non-contiguous
character) match. The matcher supports smart case detection, open/dirty file boosting,
test file penalties, char bag pre-filtering for performance, and returns match positions
for UI highlighting. It operates on ScanEntry arrays produced by the FileScanner (02-04).
</objective>

```typescript
import { ScanEntry } from '../scanner/file-scanner';

interface FuzzyMatchConfig {
  maxResults?: number;        // default 100
  caseSensitive?: boolean;    // default false
  smartCase?: boolean;        // default true (auto-detect from query)
  openFiles?: Set<string>;    // boost score for open files (+500)
  dirtyFiles?: Set<string>;   // boost score for dirty files (+300)
}

interface FuzzyMatchResult {
  entry: ScanEntry;
  score: number;
  matchPositions: number[];   // character indices for highlighting
}

class FuzzyMatcher {
  constructor(query: string, config?: FuzzyMatchConfig);

  // Main match — tries semantic first, falls back to fuzzy
  match(entries: ScanEntry[]): FuzzyMatchResult[];

  // Semantic scoring (strict, higher quality)
  matchWithSemanticScoring(entries: ScanEntry[]): FuzzyMatchResult[];

  // Fuzzy scoring (loose, fallback)
  matchWithFuzzyScoring(entries: ScanEntry[]): FuzzyMatchResult[];
}

// Scoring tiers:
// 1. Exact match (full filename match)         — highest
// 2. Prefix match (query is prefix of filename)
// 3. Suffix match (query is suffix of filename)
// 4. Substring match (query found within path)
// 5. Fuzzy match (characters in order but non-contiguous) — lowest
//
// Bonuses: open files +500, dirty files +300
// Penalties: test/story files
// Tiebreaker: shorter path wins
```
</interfaces>

<tasks>
1. **Create `packages/util/src/search/fuzzy-search.ts`** — FuzzyMatcher with multi-tier scoring
   - Implement `FuzzyMatchConfig`, `FuzzyMatchResult` interfaces
   - Implement internal helpers:
     - `normalizeQuery(query, config)` — apply smart case logic (if query has uppercase, force case-sensitive)
     - `buildCharBag(str)` — create character frequency map for pre-filtering
     - `charBagContains(haystack, needle)` — check if haystack bag contains all needle chars
     - `computeMatchPositions(text, query)` — find character indices of each query char in text
     - `isTestOrStoryFile(name)` — detect `.test.`, `.spec.`, `.story.`, `.stories.` patterns
   - Implement scoring functions:
     - `scoreExactMatch(name, query)` — full basename equals query (case-aware)
     - `scorePrefixMatch(name, query)` — basename starts with query
     - `scoreSuffixMatch(name, query)` — basename (sans extension) ends with query
     - `scoreSubstringMatch(path, query)` — query found as contiguous substring in path
     - `scoreFuzzyMatch(path, query)` — chars appear in order, non-contiguous; gap penalties
   - Implement `FuzzyMatcher`:
     - `constructor(query, config)` — normalize query, build query char bag, merge config defaults
     - `match(entries)` — call `matchWithSemanticScoring`; if too few results, augment with `matchWithFuzzyScoring`
     - `matchWithSemanticScoring(entries)` — score only exact/prefix/suffix/substring tiers
     - `matchWithFuzzyScoring(entries)` — score using fuzzy tier for remaining unmatched entries
     - For each entry: char bag pre-filter -> score -> apply bonuses/penalties -> collect
     - Sort by score descending, tiebreak by path length ascending
     - Slice to `maxResults`
   - Apply bonuses: +500 if entry.path in openFiles, +300 if in dirtyFiles
   - Apply penalty: reduce score for test/story files
   - Handle empty query: return all entries sorted by bonus (open > dirty > alphabetical), no scoring
   - Export all public types and the `FuzzyMatcher` class
