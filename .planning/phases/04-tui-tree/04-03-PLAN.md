```typescript
class RenderBox extends RenderObject {
  /** 布局后的实际尺寸 */
  size: Size;
  /** 在父坐标系中的偏移 */
  offset: Position;
  /** 最近一次 layout() 传入的约束 */
  protected _constraints: BoxConstraints | undefined;

  get constraints(): BoxConstraints | undefined;
  get hasSize(): boolean;

  /**
   * 接受约束并执行布局
   * 1. 保存 constraints
   * 2. 调用 performLayout()
   * 3. _needsLayout = false
   */
  layout(constraints: BoxConstraints): void;

  /** 子类覆写: 计算 size 并布局 children */
  abstract performLayout(): void;

  /**
   * 点击测试: 判断 (x, y) 是否在此节点的矩形范围内
   * parentX/parentY 是父偏移
   */
  hitTest(x: number, y: number): boolean;

  /**
   * 绘制: 以 offset 偏移绘制自身和 children
   * 默认实现: children.forEach(c => c.paint(screen, offsetX + c.offset.x, offsetY + c.offset.y))
   */
  paint(screen: Screen, offsetX: number, offsetY: number): void;
}
```

创建 `packages/tui/src/tree/render-box.ts`:

1. 导出抽象类 `RenderBox extends RenderObject`:

   **字段:**
   - `_size: Size = { width: 0, height: 0 }`
   - `_offset: Position = { x: 0, y: 0 }`
   - `_constraints: BoxConstraints | undefined`

   **属性:**
   - `size` getter/setter — setter 断言值为有限数
   - `offset` getter/setter
   - `constraints` getter → `_constraints`
   - `hasSize` → `_size.width > 0 || _size.height > 0`

   **布局方法:**
   - `layout(constraints: BoxConstraints)`:
     1. `_constraints = constraints`
     2. 如果 `_needsLayout` 或约束变化，调用 `performLayout()`
     3. `_needsLayout = false`
     4. `markNeedsPaint()` (布局变化后需要重绘)

   **绘制方法:**
   - 覆写 `paint(screen, offsetX, offsetY)`:
     1. `_needsPaint = false`
     2. 调用 `performPaint(screen, offsetX, offsetY)` (可选覆写的钩子)
     3. 递归: children 中每个 RenderBox child → `child.paint(screen, offsetX + child.offset.x, offsetY + child.offset.y)`

   - `performPaint(screen, offsetX, offsetY)` — 空默认实现，子类覆写添加自定义绘制

   **命中测试:**
   - `hitTest(x, y)`:
     1. 计算本地坐标: `localX = x - offset.x`, `localY = y - offset.y`
     2. 返回 `localX >= 0 && localX < size.width && localY >= 0 && localY < size.height`

2. JSDoc 中文注释
