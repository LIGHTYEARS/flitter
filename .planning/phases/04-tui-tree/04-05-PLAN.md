```typescript
class Key {
  constructor(public readonly value: string | number);
  equals(other: Key): boolean;  // value 比较
}

class GlobalKey extends Key {
  _element: Element | undefined;
  _setElement(element: Element): void;
  _clearElement(): void;
  get currentElement(): Element | undefined;
  get currentState(): State | undefined;
}

abstract class Widget {
  readonly key: Key | undefined;

  constructor(options?: { key?: Key });

  canUpdate(other: Widget): boolean;
  abstract createElement(): Element;
}
```

创建 `packages/tui/src/tree/widget.ts`:

1. 导出 `Key` 类:
   - `readonly value: string | number`
   - `constructor(value)` — 保存 value
   - `equals(other: Key)` — `this.value === other.value`

2. 导出 `GlobalKey` 类 (extends Key):
   - `_element: Element | undefined`
   - `_setElement(element)` — 保存引用
   - `_clearElement()` — 清除引用
   - `get currentElement()` → `_element`
   - `get currentState()` — 如果 element 是 StatefulElement，返回其 state

3. 导出抽象类 `Widget`:
   - `readonly key: Key | undefined`
   - `constructor(options?: { key?: Key })` — 保存 key

   - `canUpdate(other: Widget): boolean`:
     1. 检查 `this.constructor === other.constructor` (同类型)
     2. 如果两者都有 key → `this.key!.equals(other.key!)`
     3. 如果两者都无 key → `true`
     4. 如果只有一个有 key → `false`

   - `abstract createElement(): Element`

4. 同时在此文件中重新导出 Element 中的 Widget/Key 接口，让 element.ts 可以使用实际的 Widget 类而非前向声明接口。

注意: element.ts 中的 Widget 接口会在后续计划中与此文件的 Widget 类统一（通过修改 element.ts 的 import）。
