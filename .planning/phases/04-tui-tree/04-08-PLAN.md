创建 `packages/tui/src/tree/frame-scheduler.ts`:

1. 导出 `FramePhase` 类型: `"build" | "layout" | "paint" | "render"`

2. 导出 `FrameCallbackEntry` 接口: `{ callback, phase, priority, name }`

3. 导出类 `FrameScheduler`:

   **常量:**
   - `MIN_FRAME_INTERVAL = 16` (ms, ~60fps)

   **字段:**
   - `_callbacks: Map<string, FrameCallbackEntry> = new Map()`
   - `_postFrameCallbacks: Array<{ callback, name }> = []`
   - `_frameScheduled: boolean = false`
   - `_frameInProgress: boolean = false`
   - `_pendingFrameTimer: ReturnType<typeof setTimeout> | null = null`
   - `_lastFrameTimestamp: number = 0`
   - `_useFramePacing: boolean = true` (测试环境下可禁用)

   **属性:**
   - `isFrameScheduled` → `_frameScheduled || _frameInProgress`

   **方法:**
   - `requestFrame()`:
     1. 如果 `_frameInProgress`, 标记 `_frameScheduled = true`, return (当前帧完成后会自动调度)
     2. 如果已有 `_pendingFrameTimer`, return
     3. 计算距上次帧的时间差
     4. 如果 `_useFramePacing` 且时间差 < MIN_FRAME_INTERVAL:
        - `setTimeout(runScheduledFrame, remaining)` 延迟执行
     5. 否则直接 `executeFrame()`

   - `addFrameCallback(id, callback, phase, priority=0, name='')`:
     `_callbacks.set(id, { callback, phase, priority, name })`

   - `removeFrameCallback(id)`: `_callbacks.delete(id)`

   - `addPostFrameCallback(callback, name='')`:
     1. 添加到 `_postFrameCallbacks`
     2. `requestFrame()` (确保有帧来执行)

   - `executeFrame()`:
     1. `_frameInProgress = true`
     2. `_frameScheduled = false`
     3. `_lastFrameTimestamp = Date.now()`
     4. 按顺序执行四个阶段: `executePhase("build")` → `"layout"` → `"paint"` → `"render"`
     5. 执行 `_postFrameCallbacks` (逐个执行后清空)
     6. `_frameInProgress = false`
     7. 如果 `_frameScheduled`, 重新 `requestFrame()` (有新帧请求)

   - `executePhase(phase)`:
     1. 收集该阶段的所有 callbacks
     2. 按 priority 升序排序
     3. 逐个执行

   - `disableFramePacing()` — 测试用，同步执行
   - `dispose()` — 清除 timer + callbacks

4. JSDoc 中文注释
