# Plan 07-01: Fix ACP Types, Event Names, and Usage Data Model

<objective>
Fix the 4 wrong sessionUpdate event names in `app-state.ts` and replace the internal `UsageInfo` type with the SDK's `UsageUpdate` structure `{size, used, cost?: {amount, currency}}`. Update all consumers (ConversationState, BottomGrid, session.ts).
</objective>

<must_haves>
- `agent_thought_chunk` events are correctly dispatched (not `thinking_chunk`)
- `usage_update` events are correctly dispatched (not `usage`)
- `current_mode_update` events are correctly dispatched (not `current_mode`)
- `session_info_update` events are correctly dispatched (not `session_info`)
- `UsageInfo` type replaced with `{size: number; used: number; cost?: {amount: number; currency: string} | null}`
- BottomGrid displays `used`/`size` context window info instead of `inputTokens`/`outputTokens`
- No-op stubs added for `user_message_chunk`, `available_commands_update`, `config_option_update`
</must_haves>

<task id="2">
<title>Fix 4 event names and usage_update handler in app-state.ts</title>
<read_first>
- packages/flitter-amp/src/state/app-state.ts
- packages/flitter-amp/node_modules/@agentclientprotocol/sdk/dist/schema/types.gen.d.ts (lines 2537-2565 for SessionUpdate union)
</read_first>
<action>
In `packages/flitter-amp/src/state/app-state.ts`, fix the switch statement in `onSessionUpdate()`:

1. Change `case 'thinking_chunk':` to `case 'agent_thought_chunk':`
2. Change `case 'usage':` to `case 'usage_update':` AND fix the handler body:

FROM:
```typescript
case 'usage': {
  const usage = update.usage as { input_tokens?: number; output_tokens?: number; cost?: number };
  this.conversation.setUsage({
    inputTokens: usage.input_tokens ?? 0,
    outputTokens: usage.output_tokens ?? 0,
    cost: usage.cost,
  });
  break;
}
```

TO:
```typescript
case 'usage_update': {
  this.conversation.setUsage({
    size: (update as any).size ?? 0,
    used: (update as any).used ?? 0,
    cost: (update as any).cost ?? null,
  });
  break;
}
```

3. Change `case 'current_mode':` to `case 'current_mode_update':`
4. Change `case 'session_info':` to `case 'session_info_update':`

5. Add no-op stubs before the `default` case:
```typescript
case 'user_message_chunk':
case 'available_commands_update':
case 'config_option_update':
  break;
```
</action>
<acceptance_criteria>
- `app-state.ts` contains `case 'agent_thought_chunk':` (not `thinking_chunk`)
- `app-state.ts` contains `case 'usage_update':` (not `usage`)
- `app-state.ts` contains `case 'current_mode_update':` (not `current_mode`)
- `app-state.ts` contains `case 'session_info_update':` (not `session_info`)
- `app-state.ts` usage handler reads `update.size`, `update.used`, `update.cost`
- `app-state.ts` contains `case 'user_message_chunk':` and `case 'available_commands_update':` and `case 'config_option_update':` as no-ops
</acceptance_criteria>
</task>

<task id="3">
<title>Update BottomGrid to display size/used instead of inputTokens/outputTokens</title>
<read_first>
- packages/flitter-amp/src/widgets/bottom-grid.ts
</read_first>
<action>
In `packages/flitter-amp/src/widgets/bottom-grid.ts`, update the `buildTopLeft` method (around line 133-141):

FROM:
```typescript
if (w.tokenUsage) {
  const inTok = this.formatTokenCount(w.tokenUsage.inputTokens);
  const outTok = this.formatTokenCount(w.tokenUsage.outputTokens);
  return new Text({
    text: new TextSpan({
      text: `${inTok} in / ${outTok} out`,
      style: new TextStyle({ foreground: mutedColor, dim: true }),
    }),
  });
}
```

TO:
```typescript
if (w.tokenUsage) {
  const used = this.formatTokenCount(w.tokenUsage.used);
  const size = this.formatTokenCount(w.tokenUsage.size);

    ? ` · ${w.tokenUsage.cost.currency} ${w.tokenUsage.cost.amount.toFixed(4)}`
    : '';
  return new Text({
    text: new TextSpan({
      text: `${used}/${size} tokens${costStr}`,
      style: new TextStyle({ foreground: mutedColor, dim: true }),
    }),
  });
}
```
</action>
<acceptance_criteria>
- `bottom-grid.ts` contains `w.tokenUsage.used` and `w.tokenUsage.size`
- `bottom-grid.ts` does NOT contain `w.tokenUsage.inputTokens` or `w.tokenUsage.outputTokens`
- `bottom-grid.ts` displays cost with `currency` and `amount.toFixed(4)` when available
</acceptance_criteria>
</task>
