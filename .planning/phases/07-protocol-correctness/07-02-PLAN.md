# Plan 07-02: Fix Connection, Terminal, and Error Handling

<objective>
Fix clientCapabilities declaration, agent crash detection, terminal output collection, waitForTerminalExit return shape, and error path unification. These are the remaining 5 PROTO requirements.
</objective>

<must_haves>
- `clientCapabilities` sent in initialize request with `{ fs: { readTextFile: true, writeTextFile: true }, terminal: true }`
- Agent process exit triggers error state in AppState
- Terminal output collected persistently from createTerminal (no listener leak in terminalOutput)
- `waitForTerminalExit` returns `{ exitCode?, signal? }` (flat, not `{exitStatus: {code}}`)
- `terminalOutput` returns `{ output, exitStatus?: { exitCode?, signal? }, truncated }` matching SDK
- `handleSubmit` error path calls unified `handleError()` on AppState
- `AppState.handleError()` calls finalizeAssistantMessage, finalizeThinking, sets isProcessing=false, setError, notifyListeners
- `agentInfo.name` passed to `setConnected()` instead of hardcoded `'ACP Agent'`
</must_haves>

<task id="4">

In `packages/flitter-amp/src/acp/client.ts`:

1. Change the `terminals` Map to store both the process and output buffer. Replace:
```typescript
private terminals: Map<string, ChildProcess> = new Map();
```
WITH:
```typescript
private terminals: Map<string, { proc: ChildProcess; output: string; outputByteLimit: number }> = new Map();
```

2. In `createTerminal`, start persistent output collection after spawning:
After `const proc = spawn(...)` and before the return, add:
```typescript
const outputByteLimit = params.outputByteLimit ?? Infinity;
const terminal = { proc, output: '', outputByteLimit };

proc.stdout?.on('data', (chunk: Buffer) => {
  if (terminal.output.length < terminal.outputByteLimit) {
    terminal.output += chunk.toString();
  }
});
proc.stderr?.on('data', (chunk: Buffer) => {
  if (terminal.output.length < terminal.outputByteLimit) {
    terminal.output += chunk.toString();
  }
});

this.terminals.set(terminalId, terminal);
```
Remove the old `this.terminals.set(terminalId, proc);` line.

3. Rewrite `terminalOutput` to read from the persistent buffer (matching SDK `TerminalOutputResponse` shape):
```typescript
async terminalOutput(params: { terminalId: string; sessionId: string }): Promise<{
  output: string;
  exitStatus?: { exitCode?: number | null; signal?: string | null } | null;
  truncated: boolean;
}> {
  const terminal = this.terminals.get(params.terminalId);
  if (!terminal) {
    return { output: '', exitStatus: { exitCode: -1 }, truncated: false };
  }

  const truncated = terminal.output.length >= terminal.outputByteLimit;
  const exitStatus = terminal.proc.exitCode !== null
    ? { exitCode: terminal.proc.exitCode, signal: terminal.proc.signalCode ?? null }
    : null;

  return { output: terminal.output, exitStatus, truncated };
}
```

4. Update all other terminal methods that access `this.terminals` to use `.proc`:
- `waitForTerminalExit`: `const terminal = this.terminals.get(...)`, use `terminal.proc`
- `killTerminal`: `const terminal = this.terminals.get(...)`, use `terminal.proc`
- `releaseTerminal`: `const terminal = this.terminals.get(...)`, use `terminal.proc`
- `cleanup`: iterate and use `entry.proc` instead of `proc`
</action>

async waitForTerminalExit(params: { terminalId: string; sessionId: string }): Promise<{
  exitStatus: { code: number };
}> {
  const proc = this.terminals.get(params.terminalId);
  if (!proc) {
    return { exitStatus: { code: -1 } };

    return { exitStatus: { code: proc.exitCode } };

    });
  });
}
