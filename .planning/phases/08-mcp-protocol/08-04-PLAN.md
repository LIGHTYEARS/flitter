## 实现细节

### SSE 传输协议流程

```
Client                              Server
  |                                    |
  |--- GET url (Accept: text/event-stream) --->|
  |                                    |
  |<-- SSE stream starts --------------|
  |<-- event: endpoint                 |
  |    data: /mcp/session/post         |  (POST endpoint URL)
  |                                    |
  |--- POST /mcp/session/post -------->|  (JSON-RPC request)
  |<-- 200 JSON response --------------|
  |                                    |
  |<-- SSE data: {...} ----------------|  (server-initiated messages)
  |                                    |
```

### 关键实现点

1. **SSE 连接**: 使用 fetch() + ReadableStream 实现 (非浏览器 EventSource API)
   - 复用 SSEEventParser 从 Plan 08-03
   - 监听 `event: endpoint` 获取 POST URL

2. **Endpoint 安全校验**: POST URL 必须与 SSE 连接 URL 同源

3. **消息发送**: POST JSON 到 discovered endpoint
   - 附带 Authorization header (如有)
   - 处理 401 重新认证

4. **连接管理**: AbortController 控制 SSE 流生命周期

### 测试策略

使用 `http.createServer` 模拟 SSE 服务器:
- 连接建立: 发送 endpoint 事件 → 验证 POST URL 捕获
- Origin 校验: 跨域 endpoint 拒绝
- 消息收发: SSE 接收 + POST 发送
- 401 处理: 认证回调触发
- 关闭: 连接清理验证

预期测试数: ~20
