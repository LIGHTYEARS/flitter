## 实现内容

### 1. `packages/data/src/thread/thread-persistence.ts`

ThreadPersistence class:
- `constructor(options: { baseDir: string; throttleMs?: number })` — 基础目录 + 节流间隔
- `save(thread: ThreadSnapshot): Promise<void>` — 原子写入 JSON 到 `{baseDir}/{id}.json`
- `load(id: string): Promise<ThreadSnapshot | null>` — 读取并 Zod 校验
- `delete(id: string): Promise<void>` — 删除线程文件
- `list(): Promise<string[]>` — 扫描目录获取所有线程 ID
- `loadAll(): Promise<ThreadSnapshot[]>` — 加载所有线程
- `startAutoSave(store: ThreadStore): Disposable` — 定时检查 dirty 线程并保存

原子写入实现:
```
1. write to {baseDir}/{id}.json.tmp
2. fsync the fd
3. rename {id}.json.tmp → {id}.json
```

### 2. `packages/data/src/thread/thread-persistence.test.ts`

测试用例 (~20 tests):
- **保存**: 写入 JSON 文件，内容正确
- **加载**: 读取并返回 ThreadSnapshot，Zod 校验
- **删除**: 文件被移除
- **列表**: 扫描目录返回线程 ID 列表
- **原子写入**: 写入过程中不存在部分写入的文件
- **不存在的线程**: load 返回 null
- **损坏文件**: load 返回 null + 日志警告
- **AutoSave**: dirty 线程在节流间隔后自动持久化
- **AutoSave dispose**: dispose 后不再触发保存
- **并发保存**: 多个 dirty 线程批量保存

## 验收标准

1. 线程 save/load/delete/list 全流程通过

4. Zod 校验: 加载时自动校验格式
