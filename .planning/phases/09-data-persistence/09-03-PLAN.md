## 实现内容

### 1. `packages/data/src/config/jsonc.ts`

JSONC stripper (去除 `//` 和 `/* */` 注释):
- `stripJsonComments(input: string): string` — 状态机实现，处理字符串字面量中的假注释
- 不引入外部 JSONC 解析库 (KD-27)

### 2. `packages/data/src/config/jsonc.test.ts`

测试用例 (~12 tests):
- **行注释**: `// comment` 正确去除
- **块注释**: `/* ... */` 正确去除
- **多行块注释**: 跨行 `/* ... */`
- **字符串内注释**: `"http://example.com"` 不被误删
- **嵌套引号**: `"value with \" and // inside"` 正确处理
- **尾注释**: `"key": "value" // trailing` 去除注释保留值
- **纯 JSON**: 不含注释的文件原样返回
- **空输入**: 空字符串返回空字符串
- **连续注释**: 多个注释连续出现
- **JSONC 到 JSON 完整往返**: 去除注释后 JSON.parse 成功

### 3. `packages/data/src/config/settings-storage.ts`

FileSettingsStorage class:
- `constructor(options: { globalPath: string; workspacePath?: string })`
- `resolveSettingsPath(dir: string): Promise<string>` — 先 `.json`, 再 `.jsonc` fallback (从 f_0 翻译)
- `read(scope: ConfigScope): Promise<Partial<Settings>>` — 读取 + stripJsonComments + JSON.parse + Zod 校验
- `write(scope: ConfigScope, settings: Partial<Settings>): Promise<void>` — 原子写入纯 JSON (不写注释)
- `get(key: string, scope?: ConfigScope): Promise<unknown>` — 读取单键
- `set(key: string, value: unknown, scope?: ConfigScope): Promise<void>` — 写入单键
- `delete(key: string, scope?: ConfigScope): Promise<void>` — 删除单键
- `keys(scope?: ConfigScope): Promise<string[]>` — 列出所有键
- `changes: Subject<string[]>` — 键变更事件

路径解析规则 (KD-26):
```
global:    ~/.config/flitter/settings.json
workspace: {workspaceRoot}/.flitter/settings.json (or .jsonc fallback)
```

scope 写入校验 (从 Kk):
- admin scope: 抛错 (不可从文件写)
- application scope 键: 只能写 global

### 4. `packages/data/src/config/config-service.ts`

ConfigService class (实现 `@flitter/schemas` ConfigService 接口):
- `constructor(options: { storage: FileSettingsStorage; secretStorage: SecretStorage; workspaceRoot: BehaviorSubject<string | null>; homeDir: string; userConfigDir: string })`
- 三级合并: `default → global → workspace`, 数组键 (MERGED_ARRAY_KEYS) 使用 concat + dedup
- `config: BehaviorSubject<Config>` — 合并后的当前配置
- `get(key, scope?)` / `updateSettings(scope, key, value)` / `appendSettings(scope, key, value)` / `prependSettings(scope, key, value)` / `deleteSettings(scope, key)`
- `getLatest(): Promise<Config>` — 异步获取最新配置
- `updateSecret(key, value, url?)` — 密钥写入
- `unsubscribe()` — 清理订阅

合并算法:
```
1. 读取 global settings
2. 读取 workspace settings (如有)
3. 浅合并: {...global, ...workspace}
4. 数组键特殊处理: concat + dedup
5. GLOBAL_ONLY_KEYS 只从 global 取
6. 注入 default url
```

### 5. `packages/data/src/config/config-service.test.ts`

测试用例 (~20 tests):
- **全局配置**: 读写 global settings.json
- **工作区配置**: 读写 workspace .flitter/settings.json
- **三级合并**: workspace 覆盖 global
- **数组合并**: MERGED_ARRAY_KEYS concat + dedup
- **GLOBAL_ONLY_KEYS**: mcpServers 只从 global 读取
- **scope 写入校验**: admin scope 抛错, application 键不能写 workspace
- **JSONC 读取**: .jsonc 文件带注释正确解析
- **JSONC fallback**: .json 不存在时尝试 .jsonc
- **原子写入**: 回写为纯 JSON
- **单键操作**: get/set/delete 单个键
- **append/prepend**: 数组追加/前插
- **不存在文件**: 首次读取返回空对象
- **Zod 校验**: 非法键值被过滤
