## 实现内容

### 1. `packages/data/src/config/config-watcher.ts`

ConfigFileWatcher class:
- `constructor(options: { paths: string[]; debounceMs?: number })` — 监听的路径列表 + 防抖间隔
- `start(): Disposable` — 启动 `fs.watch` / `fs.watchFile` 监听
- `changes: Subject<{ path: string; type: 'change' | 'rename' }>` — 变更事件流
- `dispose()` — 停止所有文件监听

防抖策略:
- 收到文件变更事件后，等待 debounceMs (默认 300ms) 再触发
- 窗口内多次变更只触发一次

### 2. `packages/data/src/config/config-service.ts` (扩展)

在 09-03 的 ConfigService 基础上添加:
- `watchAndReload(): Disposable` — 启动文件监听，变更时自动重新合并配置
- 内部订阅 `ConfigFileWatcher.changes`，变更时:
  1. 重新读取变更的 scope 文件
  2. 重新合并三级配置
  3. diff 检查: 只有实际变更才推送 BehaviorSubject.next()
- `workspaceRoot` 变更时:
  1. 停止旧 workspace 路径监听
  2. 发现新 workspace settings 路径
  3. 重新加载 + 合并

Reactive 数据流 (从 LX 翻译):
```
configSubject = combineLatest(settingsSubject, secretsSubject)
  .pipe(
    distinctUntilChanged(deepEqual),
    map(([settings, secrets]) => ({ settings: { ...settings }, secrets })),
    shareReplay(1)
  )
```

### 3. `packages/data/src/config/config-watcher.test.ts`

测试用例 (~12 tests):
- **文件变更**: 修改 settings.json 后触发 changes 事件
- **防抖**: 快速多次修改只触发一次
- **新文件**: 创建 settings.json 后检测到
- **删除文件**: 删除 settings.json 后配置恢复为空
- **workspaceRoot 切换**: 旧监听停止，新监听启动
- **config 自动更新**: 文件变更后 config BehaviorSubject 推送新值
- **diff 过滤**: 文件重写但内容不变时不推送
- **dispose**: dispose 后不再触发事件
- **多路径监听**: 同时监听 global + workspace
- **JSONC 热重载**: .jsonc 文件变更正确解析

## 验收标准

1. 修改 settings.json 后 ConfigService.config 自动更新
2. 防抖正确: 快速连续变更只触发一次重新加载
3. workspaceRoot 切换时重新监听
4. dispose 后清理所有监听器
5. diff 过滤: 内容不变不推送
