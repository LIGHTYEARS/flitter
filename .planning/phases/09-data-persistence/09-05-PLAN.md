## 实现内容

### 1. `packages/data/src/skill/skill-types.ts`

```typescript
/** SKILL.md frontmatter */
interface SkillFrontmatter {
  name: string;
  description: string;
  mcpServers?: Record<string, MCPServerSpec>;
  includeTools?: string[];
  // ...扩展字段 passthrough
  [key: string]: unknown;
}

/** 解析后的 Skill */
interface Skill {
  name: string;
  description: string;
  baseDir: string;           // skill 目录绝对路径
  frontmatter: SkillFrontmatter;
  body: string;              // frontmatter 以下的正文
  files?: SkillFile[];       // 目录下的文件列表 (可选)
}

interface SkillFile {
  path: string;              // 相对路径
  fullPath: string;          // 绝对路径
  size: number;
}

/** 扫描结果 */
interface SkillScanResult {
  skills: Skill[];
  errors: Array<{ path: string; error: string }>;
  warnings: string[];
}

/** 安装结果 */
interface SkillInstallResult {
  success: boolean;
  skillName: string;
  installedPath: string;
  error?: string;
}
```

### 2. `packages/data/src/skill/skill-parser.ts`

SKILL.md 解析器:
- `parseSkillFrontmatter(content: string): { frontmatter: SkillFrontmatter; body: string }` — YAML frontmatter 提取 + 校验 (从 SqR 翻译)
- `validateSkillName(name: string): void` — 校验规则: lowercase alphanumeric + hyphens, no trailing hyphen, max 64 chars (从 vqR 翻译)
- `loadSkill(dir: string): Skill` — 加载单个 skill 目录 (从 OqR 翻译)
- `scanSkillFiles(dir: string): SkillFile[]` — 递归扫描 skill 目录下的文件 (从 SaT 翻译, 跳过 symlinks + 隐藏 + 非法路径)

YAML 解析使用简化自实现 (不引入 js-yaml):
- 仅支持 `key: value` 简单键值对 + 简单数组
- 足够处理 SKILL.md frontmatter 的实际格式

### 3. `packages/data/src/skill/skill-service.ts`

SkillService class:
- `constructor(options: { configService: ConfigService; workspaceRoot: BehaviorSubject<string | null>; debounceMs?: number })`
- `skills: BehaviorSubject<Skill[]>` — 当前发现的技能列表
- `errors: BehaviorSubject<Array<{ path: string; error: string }>>` — 解析错误
- `mcpServersFromSkills: BehaviorSubject<Record<string, MCPServerSpec>>` — 从 skills 派生的 MCP server 配置
- `scan(): Promise<SkillScanResult>` — 手动触发扫描
- `install(source: string, options?: { name?: string; overwrite?: boolean }): Promise<SkillInstallResult[]>` — 安装 skill (本地路径 copy, 从 J5T 翻译)
- `remove(name: string): boolean` — 删除 skill (从 TzT 翻译)
- `list(): SkillFrontmatter[]` — 列出已安装 skills (从 bqR/yqR 翻译)
- `startWatching(): Disposable` — 启动文件监听 (debounced reload)
- `dispose()` — 清理

发现流程:
1. 获取 workspaceRoot
2. 构建搜索路径列表 (KD-30)
3. 对每个路径扫描子目录，查找 SKILL.md / skill.md
4. 解析每个 skill，合并结果
5. 检测 MCP server 声明，构建 mcpServersFromSkills

文件监听:
- 对每个发现路径启动 `fs.watch` (递归)
- 只关注 `SKILL.md` / `skill.md` 文件变更
- 变更触发 debounced rescan (默认 500ms)
- workspaceRoot 变更时重新设置监听

### 4. `packages/data/src/skill/skill-service.test.ts`

测试用例 (~22 tests):
- **Frontmatter 解析**: name + description 必填，缺失报错
- **Name 校验**: 合法名 ok, 大写/特殊字符/空名/超长 报错
- **加载单个 skill**: 正确读取 SKILL.md + 文件列表
- **三路径扫描**: project + global 路径都被扫描
- **优先级**: project skill 优先于 global 同名 skill
- **无 skill 目录**: 扫描空目录返回空数组
- **损坏 SKILL.md**: 解析错误记录到 errors
- **安装 skill**: 从本地路径复制到 project skills 目录
- **安装覆盖**: overwrite=true 覆盖已有 skill
- **安装重名**: overwrite=false 且已存在时返回错误
- **安装重命名**: name 选项指定安装名称
- **删除 skill**: 目录被移除
- **删除不存在**: 返回 false
- **列表**: 返回所有已安装 skill 的 frontmatter
- **MCP 派生**: skill frontmatter 中的 mcpServers 被提取
- **文件监听**: SKILL.md 变更触发 rescan
- **文件监听 debounce**: 快速多次变更只触发一次
- **workspaceRoot 变更**: 重新扫描新路径
- **dispose**: 停止监听
