## 实现内容

### 1. `packages/data/src/guidance/guidance-types.ts`

```typescript
/** Guidance 文件类型 */
type GuidanceType = "project" | "parent" | "user" | "mentioned";

/** Guidance 文件 frontmatter */
interface GuidanceFrontmatter {
  globs?: string[];            // 文件 glob 过滤
  [key: string]: unknown;      // 扩展字段
}

/** 加载后的 Guidance 文件 */
interface GuidanceFile {
  uri: string;                 // 文件绝对路径
  type: GuidanceType;
  content: string;             // frontmatter 以下的正文
  frontMatter: GuidanceFrontmatter | null;
  exclude: boolean;            // globs 不匹配时标记排除
  lineCount: number;
}

/** Guidance 加载选项 */
interface GuidanceLoadOptions {
  workspaceRoots: string[];    // workspace root 路径列表
  userConfigDir?: string;      // ~/.config/flitter
  readFiles?: string[];        // 当前打开的文件 (用于 glob 过滤)
  maxBytesPerFile?: number;    // 单文件预算 (default: 32768)
  signal?: AbortSignal;        // 取消信号
}
```

### 2. `packages/data/src/guidance/guidance-loader.ts`

GuidanceLoader class:
- `discover(options: GuidanceLoadOptions): Promise<GuidanceFile[]>` — 主入口

发现流程 (从 kkR 翻译):
1. 从每个 workspaceRoot 开始向上遍历到文件系统根
2. 在每个目录搜索 `AGENTS.md` 和 `CLAUDE.md` (优先级: AGENTS.md > CLAUDE.md)
3. 同一目录只取优先级最高的文件 (dedup by dir)
4. 分类:
   - workspace root 内: `project`
   - workspace root 以上: `parent`
   - userConfigDir 内: `user`

加载流程 (从 ukR 翻译):
1. 读取文件内容
2. 解析 YAML frontmatter (从 XDT)
3. 检查 globs 过滤 (从 lkR): frontmatter.globs vs readFiles, 不匹配则 exclude=true
4. 提取 `@` 引用 (从 AkR): 正则 `/@([a-zA-Z0-9._~/*?[\]{}\\,-]+)/g`
5. 递归加载 `@` 引用的文件 (type: "mentioned")
6. 排序: 先 project/parent/user, 再 mentioned (插在引用者之后)
7. 截断: 超出 maxBytesPerFile 的文件截断

辅助函数:
- `parseFrontmatter(content: string): { frontMatter: GuidanceFrontmatter | null; content: string }` — YAML frontmatter 解析 (从 XDT)
- `matchGlobs(frontmatter: GuidanceFrontmatter | null, readFiles: string[], fileUri: string): boolean` — glob 过滤 (从 lkR)
- `extractAtReferences(content: string): string[]` — `@pattern` 提取 (从 AkR, 过滤代码块)
- `isRootDirectory(dir: string): boolean` — 根目录检测 (从 Q9T)

### 3. `packages/data/src/guidance/guidance-loader.test.ts`

测试用例 (~20 tests):
- **基本发现**: workspace root 下的 AGENTS.md 被发现
- **向上遍历**: 父目录中的 AGENTS.md 被发现 (type: "parent")
- **优先级**: AGENTS.md 优先于同目录 CLAUDE.md
- **去重**: 同一目录只取一个文件
- **user 类型**: userConfigDir 下的文件标记为 "user"
- **frontmatter 解析**: 正确提取 globs
- **globs 过滤**: globs 不匹配当前文件时 exclude=true
- **globs 匹配**: globs 匹配当前文件时 exclude=false
- **无 globs**: frontmatter 无 globs 时不排除
- **@ 引用**: 正文中 `@path/to/file.md` 被提取
- **@ 引用递归**: 被引用文件中的 `@` 再次递归
- **@ 代码块排除**: 代码块内的 `@` 不作为引用
- **预算截断**: 超出 maxBytesPerFile 的内容被截断
- **不存在文件**: 文件不存在时静默跳过
- **空目录**: 无 AGENTS.md/CLAUDE.md 返回空数组
- **根目录停止**: 遍历到根目录停止
- **排序**: project > parent > user, mentioned 紧随引用者
- **AbortSignal**: 取消时中止扫描
- **无效 YAML**: frontmatter 解析失败时 content 为全文
