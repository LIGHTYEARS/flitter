## 实现内容

### 1. `packages/data/src/context/token-counter.ts`

近似 token 计数器:
- `countTokensApprox(text: string): number` — 混合语言近似算法:
  - ASCII/Latin: chars / 4
  - CJK (中日韩): chars / 2
  - 遍历字符分段统计
- `countMessageTokens(message: ThreadMessage): number` — 统计单条消息所有 content block 的 token
- `countThreadTokens(thread: ThreadSnapshot): number` — 统计整个线程的 token

设计决策 (KD-29):
- 不使用 tiktoken 等精确分词器 (避免 WASM 依赖)
- 近似算法足够用于压缩阈值判断 (误差 ±20% 可接受)

### 2. `packages/data/src/context/context-manager.ts`

ContextManager class:
- `constructor(options: { configService: ConfigService; compactFn: CompactFunction; modelContextWindow?: number })`
- `compactionState: BehaviorSubject<CompactionState>` — "idle" | "compacting"
- `checkAndCompact(thread: ThreadSnapshot): Promise<CompactionResult>` — 主入口:
  1. 计算当前 token 数
  2. 比较阈值: `tokens > modelContextWindow * (compactionThresholdPercent / 100)`
  3. 超过阈值时触发压缩
  4. 返回压缩后的线程快照

CompactFunction 类型:
```typescript
type CompactFunction = (messages: ThreadMessage[]) => Promise<string>;
// 由上层 (agent-core) 注入 LLM 调用
// 返回摘要文本
```

压缩流程:
1. `compactionState.next("compacting")`
2. 选择要压缩的消息范围:
   - 保留最后 N 条消息 (确保最近上下文完整)
   - 选择前面的消息进行摘要
3. 调用 `compactFn(messagesToSummarize)` 获取摘要
4. 构建摘要 content block: `{ type: "summary", summary: { type: "message", summary } }`
5. 替换被压缩的消息为摘要 content block
6. 修剪尾部不完整的 tool_use 序列 (从 _IR 翻译)
7. `compactionState.next("idle")`

修剪逻辑 (从 _IR):
- 如果最后一条 assistant 消息的 stop_reason 是 "tool_use"，但没有对应的 user tool_result → 删除该 assistant 消息
- 递归检查直到消息序列完整

### 3. `packages/data/src/context/context-manager.test.ts`

测试用例 (~18 tests):
- **Token 计数 ASCII**: 纯英文文本近似 chars/4
- **Token 计数 CJK**: 纯中文文本近似 chars/2
- **Token 计数混合**: 中英混合文本正确统计
- **Token 计数消息**: 多 content block 消息累加
- **Token 计数线程**: 多消息线程累加
- **阈值未达**: token 数低于阈值时不触发压缩
- **阈值触发**: token 数超过阈值时触发压缩
- **压缩状态流转**: idle → compacting → idle
- **摘要插入**: 压缩后消息列表包含 summary content block
- **消息保留**: 最近 N 条消息被保留不压缩
- **tool_use 修剪**: 尾部不完整 tool_use 被删除
- **空线程**: 空线程不触发压缩
- **compactFn 调用**: 正确传入待压缩消息
- **compactFn 失败**: LLM 调用失败时恢复 idle 状态，不丢失消息
- **configService 读取**: 从 ConfigService 获取 compactionThresholdPercent
- **自定义 modelContextWindow**: 使用传入的窗口大小
- **连续压缩**: 多次压缩不累加 summary
- **CompactionResult 返回值**: 包含压缩前后 token 数、摘要内容

## 验收标准

1. 近似 token 计数: ASCII ~chars/4, CJK ~chars/2, 误差在 ±30% 内
2. 阈值检测正确: 超过 compactionThresholdPercent 触发压缩
3. 摘要正确插入到线程消息中, 被压缩的消息被替换
4. 尾部不完整 tool_use 序列被修剪
5. compactionState 状态流转正确
6. compactFn 失败时优雅恢复, 不丢失消息
