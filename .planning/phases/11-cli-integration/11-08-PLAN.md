    **CR-01 fix: Replace exec() with execFile() to prevent command injection.**

    In `packages/cli/src/auth/oauth.ts`, find the `defaultOpenBrowser` function (around line 325-334). Replace the entire function body with:

    ```typescript
    async function defaultOpenBrowser(url: string): Promise<void> {
      const { execFile } = await import("node:child_process");
      const cmd =
        process.platform === "darwin"
          ? "open"
          : process.platform === "win32"
            ? "cmd"
            : "xdg-open";

      if (process.platform === "win32") {
        execFile(cmd, ["/c", "start", "", url]);
      } else {
        execFile(cmd, [url]);
      }
    }
    ```

    Key changes:
    - Import `execFile` instead of `exec`
    - Pass the URL as an argument in an array, NOT interpolated into a shell command string
    - On Windows, use `cmd /c start "" <url>` with arguments array
    - On macOS/Linux, use `open <url>` or `xdg-open <url>` with arguments array
    - `execFile` does NOT spawn a shell, so shell metacharacters in the URL (like `$(...)` or backticks) are NOT interpreted

    This eliminates the command injection vulnerability where an attacker-controlled URL could execute arbitrary shell commands.
  </action>
