# Plan 11-10: Gap Closure — Ghost Domain Removal + OAuth Rewire + Configurable Providers

## Problem

UAT found 3 issues (2 blockers, 1 major) plus a new user requirement for configurable LLM providers. All stem from mechanical `ampcode.com → flitter.dev` domain replacement during migration and a disconnected OAuth implementation.

### Gaps

| # | Severity | Gap | Root Cause |
|---|----------|-----|------------|
| 1 | Blocker | `api.flitter.dev` DNS fails — ghost domain | Mechanical domain replacement; no Flitter backend exists |
| 2 | Blocker | CLI OAuth disconnected from `@flitter/llm` providers | Phase 11-04 translated Amp's ampURL OAuth verbatim instead of wiring to existing `@flitter/llm/oauth` |
| 3 | Major | `update.flitter.dev` DNS fails + no try/catch on fetch | Same ghost domain issue; missing error handling |
| 4 | New | User needs configurable Anthropic-compatible endpoint (ARK/Volcengine) | Settings schema lacks `anthropic.baseURL`, AnthropicProvider ignores custom baseURL, SecretStorage is global not per-provider, Registry has no config injection, custom model names not in MODEL_REGISTRY |

## must_haves

### Truths

1. **No ghost domains**: No hardcoded reference to `api.flitter.dev` or `update.flitter.dev` exists anywhere in the codebase
2. **OAuth uses @flitter/llm providers**: `flitter login` presents provider selection and delegates to `@flitter/llm/oauth` registry providers
3. **Update check is resilient**: `checkForUpdate()` wraps `fetch()` in try/catch, returns `null` on any network error, update URL is configurable
4. **Custom Anthropic baseURL works**: User can set `anthropic.baseURL` in settings.json and `AnthropicProvider._createClient()` reads it
5. **Per-provider API keys**: SecretStorage scopes credentials by provider ID, not by ghost ampURL
6. **Custom models resolve**: Unknown model names (e.g., `ep-20260331120931-5lxqv`) resolve via settings `internal.model` override or prefix rules

### Artifacts

- `packages/schemas/src/config.ts` — New settings keys: `anthropic.baseURL`, `anthropic.apiKey`, `openai.apiKey`, `openai.baseURL`, `gemini.apiKey`, `update.url`
- `packages/llm/src/types.ts` — `MODEL_REGISTRY` gains `registerModel()` function for dynamic model registration
- `packages/llm/src/providers/anthropic/provider.ts` — `_createClient()` reads `anthropic.baseURL` from settings
- `packages/llm/src/providers/registry.ts` — `createProvider()` accepts optional config injection; `resolveProvider()` falls back to settings-based provider override
- `packages/cli/src/auth/oauth.ts` — **Deleted** (replaced by `@flitter/llm/oauth` providers)
- `packages/cli/src/auth/api-key.ts` — Per-provider key storage; relaxed validation for non-standard keys
- `packages/cli/src/commands/auth.ts` — Rewired: provider selection → `@flitter/llm/oauth` login → per-provider credential storage
- `packages/cli/src/update/checker.ts` — try/catch on fetch, configurable URL, no ghost domain default
- `packages/cli/src/main.ts` — Remove ampURL; register OAuth providers on startup; pass settings to container
- `packages/flitter/src/container.ts` — Remove `ampURL` from `ContainerOptions`

```typescript
  apiKey: string,
  settings: Record<string, unknown>,
  model: string,

  const baseURL = settings["anthropic.baseURL"] as string | undefined;

  return new Anthropic({
    ...(isOAuthToken ? { authToken: apiKey } : { apiKey }),
    ...(baseURL ? { baseURL } : {}),
    defaultHeaders,
  });
}
```

### Step 4: Provider Registry — Config Injection + Settings Override

**File**: `packages/llm/src/providers/registry.ts`

Add settings-based provider override to `resolveProvider()` and config injection to `createProvider()`:

```typescript
/**
 * 创建 Provider 实例 (单例缓存)
 *
 * @param name - Provider 名称
 * @param opts - 可选配置注入 (settings 用于读取 baseURL 等)
 */
export function createProvider(
  name: ProviderName,
  opts?: { settings?: Record<string, unknown> },
): LLMProvider {
  const cacheKey = opts?.settings ? `${name}:custom` : name;
  const cached = _cache.get(cacheKey);
  if (cached) return cached;

  let provider: LLMProvider;
  switch (name) {
    case "anthropic":
      provider = new AnthropicProvider();
      break;
    case "openai":
      provider = new OpenAIProvider();
      break;
    case "gemini":
      provider = new GeminiProvider();
      break;
    case "xai":
      provider = new OpenAICompatProvider({ name: "xai" });
      break;
    case "openai-compat":
      provider = new OpenAICompatProvider();
      break;
    default:
      // Dynamic provider — try OpenAICompatProvider with custom config
      provider = new OpenAICompatProvider({ name });
      break;
  }

  _cache.set(cacheKey, provider);
  return provider;
}
```

Update `resolveProvider()` to check settings for provider override:

```typescript
/**
 * 从模型名推断 Provider
 *
 * 优先级:
 * 1. "provider/model" 格式 → 提取 provider 前缀
 * 2. MODEL_REGISTRY 精确查找 → 返回 provider
 * 3. 前缀匹配 fallback
 * 4. 未知模型 → "anthropic" (默认, 因为 Anthropic SDK 兼容自定义 baseURL)
 */
export function resolveProvider(model: string): ProviderName {
  // 1. "provider/model" format
  const slashIdx = model.indexOf("/");
  if (slashIdx !== -1) {
    const prefix = model.slice(0, slashIdx);
    const providerName = PROVIDER_ALIASES[prefix];
    if (providerName) return providerName;
  }

  // 2. MODEL_REGISTRY lookup
  const info = MODEL_REGISTRY[model];
  if (info) return info.provider;

  // 3. Prefix fallback
  for (const rule of PREFIX_RULES) {
    if (rule.match(model)) return rule.provider;
  }

  // 4. Unknown model — default to anthropic (supports custom baseURL)
  return "anthropic";
}
```

Key change: `resolveProvider()` no longer throws for unknown models. It defaults to `"anthropic"` which supports `anthropic.baseURL` for custom endpoints.

### Step 5: CLI Auth — Rewire to @flitter/llm OAuth Providers

**File**: `packages/cli/src/commands/auth.ts`

Complete rewrite of `handleLogin` and `handleLogout`:

```typescript
/**
 * 认证命令处理器
 *
 * 处理 `flitter login` 和 `flitter logout` 命令。
 * login 支持多 Provider: API Key 直接输入 / OAuth 浏览器流程。
 * logout 清除指定 Provider 或所有凭据。
 *
 * 逆向参考: eF0() (login) / tF0() (logout) in cli-entrypoint.js:1032-1052
 */
import type { CliContext } from "../context";
import type { SecretStorage } from "@flitter/flitter";
import type { OAuthCredentials, OAuthProviderInterface } from "@flitter/llm";
import {
  getOAuthProviders,
  registerOAuthProvider,
  AnthropicOAuthProvider,
  OpenAICodexOAuthProvider,
  GitHubCopilotOAuthProvider,
} from "@flitter/llm";
import {
  getApiKeyFromEnv,
  validateApiKey,
  storeApiKey,
  promptApiKey,
  promptProviderSelection,
} from "../auth/api-key";

/** 确保 OAuth providers 已注册 (幂等) */
let providersRegistered = false;
function ensureOAuthProviders(): void {
  if (providersRegistered) return;
  registerOAuthProvider(new AnthropicOAuthProvider());
  registerOAuthProvider(new OpenAICodexOAuthProvider());
  registerOAuthProvider(new GitHubCopilotOAuthProvider());
  providersRegistered = true;
}

/**
 * 认证命令依赖
 */
export interface AuthCommandDeps {
  /** 秘密存储服务 */
  secrets: SecretStorage;
}

/**
 * 认证方式选项
 */
const AUTH_METHODS = [
  { id: "api-key", name: "API Key (直接输入)" },
  { id: "anthropic", name: "Anthropic (Claude Pro/Max OAuth)" },
  { id: "openai-codex", name: "OpenAI (ChatGPT Plus/Pro OAuth)" },
  { id: "github-copilot", name: "GitHub Copilot (Device Code)" },
] as const;

/**
 * 处理 login 命令
 *
 * 认证流程:
 * 1. 环境变量 FLITTER_API_KEY → 直接存储
 * 2. 交互式选择认证方式:
 *    a. API Key 直接输入 → 验证格式 → 存储
 *    b. OAuth Provider → 浏览器/设备码认证 → 存储 credentials
 *
 * @param deps - 认证所需的依赖服务
 * @param context - CLI 运行上下文
 */
export async function handleLogin(
  deps: AuthCommandDeps,
  context: CliContext,
): Promise<void> {
  const { secrets } = deps;
  ensureOAuthProviders();

  // 1. 检查环境变量
  const envKey = getApiKeyFromEnv();
  if (envKey) {
    if (validateApiKey(envKey)) {
      await storeApiKey(secrets, "default", envKey);
      process.stderr.write("Logged in using FLITTER_API_KEY environment variable.\n");
      return;
    }
    process.stderr.write("Warning: FLITTER_API_KEY has invalid format, ignoring.\n");
  }

  // 2. 非交互模式无法继续
  if (!context.isTTY) {
    process.stderr.write("No API key found. Set FLITTER_API_KEY or run in a TTY terminal.\n");
    process.exitCode = 1;
    return;
  }

  // 3. 交互式选择认证方式
  const method = await promptProviderSelection(AUTH_METHODS);
  if (!method) {
    process.stderr.write("Login cancelled.\n");
    return;
  }

  if (method === "api-key") {
    // API Key 直接输入
    const key = await promptApiKey();
    if (key) {
      if (validateApiKey(key)) {
        await storeApiKey(secrets, "default", key);
        process.stderr.write("API key saved successfully.\n");
        return;
      }
      // 放宽验证: 允许非标准前缀的 key (如 ARK 端点 key)
      await storeApiKey(secrets, "default", key);
      process.stderr.write("API key saved (non-standard format — ensure it works with your provider).\n");
      return;
    }
    process.stderr.write("No key entered.\n");
    return;
  }

  // 4. OAuth Provider 流程
  const provider = getOAuthProviders().find((p) => p.id === method);
  if (!provider) {
    process.stderr.write(`Unknown auth method: ${method}\n`);
    process.exitCode = 1;
    return;
  }

  process.stderr.write(`Starting ${provider.name} authentication...\n`);
  try {
    const credentials = await provider.login({
      onAuth: (info) => {
        // 打开浏览器
        defaultOpenBrowser(info.url);
        process.stderr.write(`Opening browser for authentication...\n`);
        if (info.instructions) {
          process.stderr.write(`${info.instructions}\n`);
        }
        process.stderr.write(`If browser doesn't open, visit: ${info.url}\n`);
      },
      onPrompt: async (prompt) => {
        const { createInterface } = await import("node:readline/promises");
        const rl = createInterface({ input: process.stdin, output: process.stderr });
        try {
          const answer = await rl.question(`${prompt.message} `);
          return answer.trim();
        } finally {
          rl.close();
        }
      },
      onProgress: (msg) => {
        process.stderr.write(`${msg}\n`);
      },
      onManualCodeInput: async () => {
        const { createInterface } = await import("node:readline/promises");
        const rl = createInterface({ input: process.stdin, output: process.stderr });
        try {
          const code = await rl.question("Enter authorization code: ");
          return code.trim();
        } finally {
          rl.close();
        }
      },
    });

    // 存储 OAuth credentials (per-provider)
    await secrets.set("oauthCredentials", JSON.stringify(credentials), provider.id);
    // 同时存储提取的 API key 用于 Provider 查找
    const apiKey = provider.getApiKey(credentials);
    await storeApiKey(secrets, provider.id, apiKey);
    process.stderr.write(`${provider.name} authentication successful.\n`);
  } catch (err: unknown) {
    const message = err instanceof Error ? err.message : String(err);
    process.stderr.write(`Authentication failed: ${message}\n`);
    process.exitCode = 1;
  }
}

/**
 * 处理 logout 命令
 *
 * 清除所有已存储的凭据 (API Key + OAuth tokens)。
 */
export async function handleLogout(
  deps: AuthCommandDeps,
  _context: CliContext,
): Promise<void> {
  const { secrets } = deps;

  // 删除默认 API key
  await secrets.delete("apiKey", "default");
  // 删除所有 OAuth provider credentials
  const providers = getOAuthProviders();
  for (const provider of providers) {
    await secrets.delete("oauthCredentials", provider.id);
    await secrets.delete("apiKey", provider.id);
  }
  process.stderr.write("Logged out. All credentials have been removed.\n");
}

/**
 * 默认浏览器打开函数 (使用 execFile 防止命令注入)
 */
async function defaultOpenBrowser(url: string): Promise<void> {
  const { execFile } = await import("node:child_process");
  const cmd =
    process.platform === "darwin"
      ? "open"
      : process.platform === "win32"
        ? "cmd"
        : "xdg-open";

  if (process.platform === "win32") {
    execFile(cmd, ["/c", "start", "", url]);
  } else {
    execFile(cmd, [url]);
  }
}
```

### Step 6: API Key Module — Per-Provider Storage + Provider Selection

**File**: `packages/cli/src/auth/api-key.ts`

Add `promptProviderSelection()` and update `storeApiKey`/`hasApiKey` to use provider-scoped keys:

```typescript
/**
 * API Key 认证模块
 *
 * 提供 API Key 验证、环境变量读取、交互式输入、存储与检查功能。
 * 支持标准前缀 ('sk-', 'flitter-') 和非标准 key (如 ARK 端点 key)。
 */
import type { SecretStorage } from "@flitter/flitter";
import { createInterface } from "node:readline/promises";

/** 标准 API Key 前缀列表 */
const API_KEY_PREFIXES = ["sk-", "flitter-"];

/**
 * 检查是否已有存储的 API Key
 *
 * @param secrets - 秘密存储接口
 * @param providerId - Provider 标识 (如 "default", "anthropic", "openai-codex")
 * @returns 是否已存储 API Key
 */
export async function hasApiKey(secrets: SecretStorage, providerId: string): Promise<boolean> {
  const key = await secrets.get("apiKey", providerId);
  return Boolean(key);
}

/**
 * 验证 API Key 格式
 *
 * 规则:
 * - 非空字符串 (去除首尾空白后)
 * - 以标准前缀开头 (sk-, flitter-) 视为有效
 * - 非标准前缀也允许 (用于自定义端点 key)
 *
 * @param key - 待验证的 API Key
 * @returns 是否为有效格式 (非空即有效)
 */
export function validateApiKey(key: string): boolean {
  if (!key || key.trim().length === 0) return false;
  return true;
}

/**
 * 检查 API Key 是否为标准格式 (sk- 或 flitter- 前缀)
 *
 * @param key - 待检查的 API Key
 * @returns 是否为标准格式
 */
export function isStandardApiKey(key: string): boolean {
  return API_KEY_PREFIXES.some((prefix) => key.startsWith(prefix));
}

/**
 * 从环境变量获取 API Key
 *
 * 读取 FLITTER_API_KEY 环境变量。优先级最高的认证方式。
 */
export function getApiKeyFromEnv(): string | undefined {
  return process.env.FLITTER_API_KEY;
}

/**
 * 交互式提示用户输入 API Key
 */
export async function promptApiKey(): Promise<string | undefined> {
  const rl = createInterface({ input: process.stdin, output: process.stderr });
  try {
    const key = await rl.question("Enter your API key: ");
    return key.trim() || undefined;
  } finally {
    rl.close();
  }
}

/**
 * 交互式选择认证方式
 *
 * @param methods - 可选的认证方式列表
 * @returns 选择的 method id, 空输入返回 undefined
 */
export async function promptProviderSelection(
  methods: ReadonlyArray<{ id: string; name: string }>,
): Promise<string | undefined> {
  process.stderr.write("\nSelect authentication method:\n");
  methods.forEach((m, i) => {
    process.stderr.write(`  ${i + 1}. ${m.name}\n`);
  });
  process.stderr.write("\n");

  const rl = createInterface({ input: process.stdin, output: process.stderr });
  try {
    const answer = await rl.question("Enter number (or press Enter to cancel): ");
    const idx = parseInt(answer.trim(), 10) - 1;
    if (idx >= 0 && idx < methods.length) {
      return methods[idx].id;
    }
    return undefined;
  } finally {
    rl.close();
  }
}

/**
 * 存储 API Key 到秘密存储
 *
 * 按 providerId 分存, 不同 provider 的 key 互不干扰。
 *
 * @param secrets - 秘密存储接口
 * @param providerId - Provider 标识 (如 "default", "anthropic")
 * @param key - 要存储的 API Key
 */
export async function storeApiKey(
  secrets: SecretStorage,
  providerId: string,
  key: string,
): Promise<void> {
  await secrets.set("apiKey", key, providerId);
}
```

```typescript
 *
 * 向版本检查端点发送 HTTP GET 请求,返回最新版本信息。
 * 如果当前版本已是最新、更新被禁用、或网络不可达,返回 null。
 *
 * @param currentVersion - 当前版本号
 * @param opts - 可选配置
 * @param opts.checkUrl - 自定义版本检查端点 URL (默认不检查)
 * @param opts.updateMode - 更新模式 ("auto" | "warn" | "disabled")
 * @returns 更新信息对象,或 null (无更新/已禁用/网络错误)
 */
export async function checkForUpdate(
  currentVersion: string,
  opts?: { checkUrl?: string; updateMode?: string },
): Promise<UpdateInfo | null> {
  if (opts?.updateMode === "disabled") return null;

  // No default URL — update check is opt-in via config or env var
  const url = opts?.checkUrl ?? process.env.FLITTER_UPDATE_URL;
  if (!url) return null;

  try {
    const resp = await fetch(url, {
      headers: { "X-Current-Version": currentVersion },
      signal: AbortSignal.timeout(10_000), // 10s timeout
    });
    if (!resp.ok) return null;

    const info = (await resp.json()) as UpdateInfo;
    if (compareVersions(currentVersion, info.version) >= 0) return null;

    return info;
  } catch {
    // Network errors (DNS failure, timeout, etc.) — silently return null
    return null;
  }
}
```

Key changes:
1. **No default ghost URL**: If no `checkUrl` or `FLITTER_UPDATE_URL` env var, return `null` immediately
2. **try/catch**: All network errors silently return `null`
3. **10s timeout**: `AbortSignal.timeout(10_000)` prevents hanging
4. **Env var fallback**: `FLITTER_UPDATE_URL` for configuration without settings file

## Risk Assessment

| Risk | Mitigation |
|------|------------|
| Removing `ampURL` breaks container code that references it | Grep all usages before removal; container.ts only uses it for logging |
| OAuth provider selection UX is basic (numbered list) | Acceptable for v1; can enhance with `inquirer` later |
| `validateApiKey` relaxation allows garbage keys | Non-standard keys get a warning message; user explicitly chose to enter them |
| `resolveProvider` defaulting unknown models to "anthropic" may misroute | Correct: Anthropic SDK supports `baseURL` override, so custom endpoints work; other providers don't have this flexibility |
| Deleting `oauth.ts` removes callback server code | Already exists in `@flitter/llm/oauth/callback-server.ts`; no functionality lost |
