# Phase 12: Native Bootstrap and Runtime Shell — Plan 01

**Wave:** 1 of 1
**Scope:** BOOT-01 through BOOT-04

## Tasks

### Task 1: Create `flitter-cli` package scaffold (BOOT-01)

Add a new workspace package under `packages/flitter-cli` with:
- `package.json`
- `tsconfig.json`
- `src/index.ts`

The package must expose a `flitter-cli` bin target and a `start` script.

### Task 2: Move bootstrap config to `flitter-cli` namespace (BOOT-02)

Add native config parsing for:
- `--cwd`
- `--editor`
- `--debug`
- `--help`

Read config from `~/.flitter-cli/config.json`, not `~/.flitter-amp/config.json`.

### Task 3: Add native logging bootstrap (BOOT-03)

Create a `flitter-cli` logger that:
- writes under `~/.flitter-cli/logs`
- uses a `flitter-cli-YYYY-MM-DD.log` naming convention
- initializes before TUI startup

### Task 4: Boot a minimal runtime shell with clean exit semantics (BOOT-04)

Start a minimal `flitter-core` TUI shell that proves:
- the new package can bootstrap independently
- terminal lifecycle is owned by `flitter-cli`
- `Ctrl+C` / `Esc` / `q` can stop the shell cleanly
