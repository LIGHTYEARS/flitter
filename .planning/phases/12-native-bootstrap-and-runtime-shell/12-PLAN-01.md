# Phase 12: Native Bootstrap and Runtime Shell — Plan 01

## Tasks

### Task 1: Create the `flitter-cli` workspace package (BOOT-01)

**Files:**
- `packages/flitter-cli/package.json`
- `packages/flitter-cli/tsconfig.json`

Create a first-class workspace package with its own bin target, start/test/typecheck scripts, and TypeScript project configuration consistent with the monorepo.

### Task 2: Establish native CLI config and namespace (BOOT-02)

**File:** `packages/flitter-cli/src/state/config.ts`

Implement a minimal native config parser for `flitter-cli` that:
- uses `~/.flitter-cli/config.json`
- resolves cwd/editor/log-level settings
- supports scaffold-safe CLI flags
- avoids any ACP/coco-specific boot assumptions

### Task 3: Establish native logging and log retention (BOOT-03)

**File:** `packages/flitter-cli/src/utils/logger.ts`

Implement file-based logging under `~/.flitter-cli/logs` with simple retention and stderr fallback so the package owns its own runtime-shell observability.

### Task 4: Boot a minimal native runtime shell (BOOT-04)

**Files:**
- `packages/flitter-cli/src/index.ts`
- `packages/flitter-cli/src/bootstrap-shell.ts`

Create a minimal but real TUI bootstrap shell using `flitter-core` that:
- starts from `flitter-cli`
- initializes config and logs before boot
- renders a visible runtime shell
- supports clean exit without ACP/coco involvement

### Task 5: Add scaffold smoke coverage

**File:** `packages/flitter-cli/src/__tests__/config.test.ts`

Add a focused scaffold test so the new package has at least baseline verification coverage from day one.
