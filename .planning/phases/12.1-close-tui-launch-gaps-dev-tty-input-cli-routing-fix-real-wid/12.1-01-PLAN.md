<objective>
Port amp's /dev/tty terminal input strategy and refactor TuiController to use it instead of process.stdin.

Purpose: The current TuiController reads from process.stdin which fails when stdin is piped or redirected. /dev/tty is the actual controlling terminal device and works regardless of stdin redirection. This is the root cause preventing TUI launch.

Output: TtyInputSource module with /dev/tty primary + process.stdin fallback, refactored TuiController, output target strategy with /dev/tty write fallback.
</objective>

Refactor TuiController to use TtyInputSource/TtyOutputTarget (per D-03, D-15):

1. **Add private fields:**
   - `private ttyInput: TtyInputSource | null = null;`
   - `private ttyOutput: TtyOutputTarget | null = null;`

2. **Refactor init():**
   - Replace `process.stdin.isTTY` + `process.stdin.setRawMode(true)` + `process.stdin.on("data", ...)` with:
     ```typescript
     this.ttyInput = createTtyInput();  // Returns /dev/tty or stdin fallback
     this.ttyOutput = createTtyOutput(); // Returns stdout/stderr/dev-tty
     this.ttyInput.on("data", (data: Buffer) => { this.parser?.feed(data); });
     ```
   - Remove the `process.stdin.unref()` non-TTY guard (no longer needed with /dev/tty)

3. **Refactor render():**
   - Replace `process.stdout.write(output)` with `this.ttyOutput?.stream.write(output)`

4. **Refactor enterAltScreen()/exitAltScreen():**
   - Replace `process.stdout.write(...)` with `this.ttyOutput?.stream.write(...)`

5. **Refactor enableMouse()/enableBracketedPaste():**
   - Replace `process.stdout.writable` + `process.stdout.write(...)` with `this.ttyOutput?.stream.write(...)`

6. **Refactor deinit():** Per D-03 signal handler cleanup chain from amp's XXT.deinit():
   - Build ANSI reset sequence (MOUSE_OFF + PASTE_OFF + SHOW_CURSOR + ALT_SCREEN_OFF)

| process signals | SIGINT/SIGTERM must trigger cleanup before exit |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-12.1-01 | D (Denial of Service) | tty-input.ts | mitigate | fd leak on crash: dispose() always calls stream.destroy() + register in signal handlers |
| T-12.1-02 | T (Tampering) | tty-input.ts | accept | /dev/tty is OS-controlled, no app-level tampering risk |
