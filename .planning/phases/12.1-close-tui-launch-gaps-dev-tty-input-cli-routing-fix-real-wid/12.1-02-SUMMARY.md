**RenderScrollable as RenderBox with unbounded-height child layout, scroll-offset paint clipping, and layout widget barrel re-exports from @flitter/tui**

## Accomplishments

- RenderScrollable extends RenderBox: performLayout() passes Infinity maxHeight to child, sizes self to viewport, updates maxScrollExtent on ScrollController
- paint() applies Math.floor(scrollController.offset) as Y offset to child, Screen bounds checking handles viewport clipping
- Scrollable rewritten from plain class to Widget + RenderObjectWidget using SingleChildRenderObjectElement pattern
- 12 layout/widget re-exports added to @flitter/tui index (Column, Row, Expanded, Flexible, SizedBox, Padding, Container, RichText, TextSpan, Stack, Positioned, EdgeInsets)
- 38 total tests passing (12 RenderScrollable + 26 Scrollable/ScrollKeyHandler)

## Decisions Made

- **RenderScrollable single-child via adoptChild:** Uses `this._children[0]` pattern consistent with RenderPadding, rather than a dedicated `_child` field
- **ScrollController already had listener API:** No modifications needed to ScrollController — addListener/removeListener was already implemented in Phase 6
- **Threat T-12.1-04 mitigation:** Child height capped at MAX_CHILD_HEIGHT=100000 with console.warn, preventing absurd scroll extents
- **Retained backward compat:** Static `Scrollable.computeMaxScrollExtent()` kept for any existing callers
