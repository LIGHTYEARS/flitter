# Phase 12.1 — UI Design Contract

> Visual and interaction contract for TUI launch gap closure. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** This is a terminal-based UI (TUI) project using a custom Flutter-style widget framework with ANSI character-cell rendering. All specifications use terminal rows/columns as the unit of measure, ANSI 16/256/RGB color codes, and text attributes (bold/dim/italic/underline) rather than CSS pixels, hex colors, or web fonts.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom TUI framework) |
| Preset | not applicable |
| Component library | @flitter/tui (Flutter-style three-tree engine: Widget/Element/RenderObject) |
| Icon library | none (Unicode box-drawing characters + emoji for role indicators) |
| Font | Terminal monospace (user's terminal emulator font) |

**Source:** Project uses a custom widget framework ported from amp-cli. No web UI frameworks (shadcn, tailwind, etc.) are involved. Layout uses Column/Row/Expanded/SizedBox/Padding widgets rendering to a character-cell Screen buffer.

## Spacing Scale

Declared values in terminal rows/columns (character cells, not pixels):

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 rows/cols | Flush-mounted elements (separator lines) |
| xs | 1 row or 1 col | Single-row separators, minimal gaps between inline elements |
| sm | 2 cols | Left/right padding inside InputField border, status bar internal padding |
| md | 1 row | Vertical gap between message blocks, gap above/below separator |
| lg | 2 rows | Vertical padding at top of ConversationView (before first message) |

Exceptions:
- InputField height is dynamic (minimum 1 row for single-line input, grows with content up to 5 rows maximum).
- Status bar is fixed at exactly 1 row.
- Separator line is fixed at exactly 1 row.
- ConversationView uses `Expanded` to fill all remaining vertical space.

**Source:** Terminal UIs operate in character cells. The 8-point pixel scale does not apply. The scale above derives from amp's `_70()` layout (CONTEXT.md D-06, RESEARCH.md Layer 3).

## Typography

Terminal text has no font-size control -- all text is the same monospace cell size. "Typography" in TUI means text style attributes:

| Role | Weight | Style | Decoration | Color Token | Usage |
|------|--------|-------|------------|-------------|-------|
| Body | normal | normal | none | `text` (#a9b1d6) | Message content, default text |
| Muted | normal | normal | none | `mutedText` (#565f89) | Timestamps, secondary info, placeholder text |
| Role indicator | bold | normal | none | Per-role (see Color section) | "You:", "Assistant:", "System:" prefixes |
| Heading | bold | normal | none | `text` (#a9b1d6) | Markdown H1-H3 in conversation messages |
| Code inline | normal | normal | none | `accent` (#bb9af7) on `surface` (#1a1b26) bg | Inline code in markdown |
| Code block | normal | normal | none | `text` on `surface` bg | Fenced code blocks with syntax highlighting |
| Error | bold | normal | none | `error` (#f7768e) | Error messages in conversation view |
| Status bar | normal | normal | none | `mutedText` (#565f89) | Model name, token count, thread info |
| Input text | normal | normal | none | `text` (#a9b1d6) | User input in InputField |
| Cursor | normal | normal | none | Inverse video (swap fg/bg) | Cursor position in InputField |

**Source:** Terminal monospace rendering. Font size/weight in the CSS sense is not applicable. Bold maps to ANSI SGR attribute 1. Colors from `defaultThemeData` in interactive.ts.

## Color

All colors use the existing `ThemeData` interface defined in `packages/tui/src/widgets/theme.ts` and `packages/cli/src/modes/interactive.ts`. The concrete values come from the `defaultThemeData` Tokyo Night-inspired dark theme.

### Surface Distribution (60/30/10)

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `background` | #16161e | Terminal background, ConversationView background, primary surface |
| Secondary (30%) | `surface` | #1a1b26 | Code block backgrounds, InputField background area, status bar background |
| Accent (10%) | `primary` | #7aa2f7 | See reserved-for list below |

### Accent Reserved For

The `primary` accent color (#7aa2f7) is reserved for exactly these elements:
1. InputField border when focused
2. User role indicator prefix ("You:")
3. Active/selected items in thread picker (if implemented)
4. Cursor blinking highlight (alternative to inverse video)

### Semantic Colors

| Token | Value | Reserved For |
|-------|-------|-------------|
| `error` | #f7768e | Error message text, error block background tint, retry prompt highlight |
| `success` | #9ece6a | Successful operation status, "complete" indicators |
| `warning` | #e0af68 | Token count warning (approaching limit), deprecation notices |
| `accent` | #bb9af7 | Inline code highlighting, assistant role indicator ("Assistant:") |
| `secondary` | #9ece6a | System role indicator ("System:"), markdown link text |
| `border` | #3b4261 | Separator lines, InputField unfocused border, code block borders |
| `mutedText` | #565f89 | Placeholder text, timestamps, status bar text, dim/secondary info |
| `text` | #a9b1d6 | Primary body text, message content, heading text |

### ANSI Rendering

Colors are rendered via the existing `Color` class (`packages/tui/src/screen/color.ts`) which supports:
- Named 16-color ANSI codes (fallback for basic terminals)
- 256-color indexed palette
- 24-bit RGB true color (primary mode, used by `defaultThemeData` hex values)

The `AppColorScheme` class provides an alternative 15-color semantic palette using `Color` objects. Both systems coexist: `ThemeData` (hex strings for higher-level widgets) and `AppColorScheme` (Color objects for lower-level rendering).

**Source:** `defaultThemeData` in interactive.ts (Tokyo Night palette), `AppColorScheme.default()` in color-scheme.ts, CONTEXT.md D-14.

## Layout Contract

### Screen Regions (Top to Bottom)

```
+─────────────────────────────────────────────+
|  ConversationView (Expanded -- fills rest)   |  <- Scrollable, markdown-rendered messages
|                                              |
|  [role indicator] [message content]          |
|  [role indicator] [message content]          |
|  ...                                         |
|                                              |
+──────────────── separator ──────────────────+  <- 1 row, border color, "─" repeated
|  StatusBar                                   |  <- 1 row: model + tokens + thread
+──────────────── separator ──────────────────+  <- 1 row, border color, "─" repeated
|  InputField                                  |  <- 1-5 rows, bordered, with cursor
+─────────────────────────────────────────────+
```

### Widget Tree (Target State After Phase 12.1)

```
AppWidget (StatefulWidget)
  └── ThemeController (InheritedWidget -- injects ThemeData)
      └── ConfigProvider (InheritedWidget -- injects ConfigService)
          └── ThreadStateWidget (StatefulWidget -- subscribes to ThreadStore + ThreadWorker.events$)
              └── Column (layout)
                  ├── Expanded
                  │   └── Scrollable (RenderScrollable -- new RenderBox subclass)
                  │       └── ConversationView (StatefulWidget -- renders message list)
                  │           └── Column (message list)
                  │               ├── MessageWidget (role indicator + markdown content)
                  │               ├── MessageWidget ...
                  │               └── StreamingIndicator (if inference running)
                  ├── SizedBox (height: 1) -- separator line ("─" * width, border color)
                  ├── StatusBar (StatelessWidget -- model + tokens + thread)
                  ├── SizedBox (height: 1) -- separator line ("─" * width, border color)
                  └── InputField (StatefulWidget -- FocusNode + TextEditingController)
```

### Separator Line Specification

- Character: U+2500 "BOX DRAWINGS LIGHT HORIZONTAL" (`─`)
- Repeated to fill full terminal width (from MediaQuery)
- Foreground color: `border` (#3b4261)
- Background color: `background` (#16161e)

### InputField Visual Specification

- Border: 1-character box using box-drawing characters, color = `primary` (#7aa2f7) when focused, `border` (#3b4261) when unfocused
- Interior padding: 1 column left, 1 column right
- Placeholder: "Type a message..." in `mutedText` (#565f89), shown when text is empty
- Cursor: inverse video at cursor position (swap foreground/background)
- Minimum height: 1 row (single-line mode)
- Maximum height: 5 rows (multi-line expansion)

### ConversationView Visual Specification

- Scrollable via ScrollController + RenderScrollable (new RenderBox subclass per D-09)
- Key bindings: j/k (line), Ctrl+u/d (half-page), g/G (top/bottom), PageUp/PageDown (existing ScrollKeyHandler)
- Messages rendered via existing `MarkdownParser` + `MarkdownRenderer` from Phase 6
- Each message block: role indicator line (bold, colored) + rendered content + 1 empty row separator
- Auto-scroll to bottom on new message (ScrollController.jumpTo(maxScrollExtent))

### Status Bar Visual Specification

- Single row, full terminal width
- Left-aligned: model name in `text` color
- Center gap: 2+ spaces
- Right-aligned: "{tokenCount} tokens" in `mutedText` color
- Background: `surface` (#1a1b26)

**Source:** CONTEXT.md D-05, D-06, D-07, D-08, D-09, D-14. RESEARCH.md Layer 3 (Widget Tree Composition), Layout Strategy.

## Interaction Contract

### Input Handling

| Input | Action | Source |
|-------|--------|--------|
| Printable character | Insert at cursor position via TextEditingController | Existing InputField |
| Enter (no Shift) | Submit message (if non-empty), clear input, trigger runInference() | D-14, existing InputField |
| Shift+Enter | Insert newline (multi-line input) | Existing InputField (to extend) |
| Backspace | Delete character before cursor | Existing InputField |
| Ctrl+C | Graceful exit: cleanup signal chain per D-03 | D-03 |
| Ctrl+D | Graceful exit (EOF) | Standard terminal convention |
| Up/Down arrow | Scroll ConversationView (when ConversationView has focus) | D-08 |
| j/k | Scroll ConversationView line by line | Existing ScrollKeyHandler |
| Ctrl+u/Ctrl+d | Half-page scroll | Existing ScrollKeyHandler |
| g/G | Jump to top/bottom | Existing ScrollKeyHandler |
| PageUp/PageDown | Page scroll | Existing ScrollKeyHandler |
| Mouse scroll | Scroll ConversationView (if mouse reporting enabled) | D-06 |
| Paste (bracketed) | Insert pasted text at cursor | Existing InputField |
| SIGWINCH (resize) | MediaQuery updates terminal dimensions, triggers relayout | Existing WidgetsBinding |

### State Transitions

```
[Launch] --> [Thread Init] --> [Idle (waiting for input)]
                                     |
                              [User types Enter]
                                     |
                              [Inference Running]
                                     |  (streaming deltas update ConversationView)
                              [Turn Complete] --> [Idle]
                                     |
                              [Inference Error] --> [Error Displayed] --> [Idle]
```

### Reactive Data Flow

```
ThreadStore.observeThread(id) --> BehaviorSubject<ThreadSnapshot>
  --> ThreadStateWidget.setState() --> rebuild child tree

ThreadWorker.events$ --> Subject<AgentEvent>
  --> ThreadStateWidget.setState() --> update inferenceState/error
  --> ConversationView rebuilds with new messages/streaming delta

ScrollController.offset$ --> RenderScrollable.markNeedsPaint()
  --> next frame paints with updated scroll offset
```

**Source:** CONTEXT.md D-10, D-11, D-13. RESEARCH.md Layer 4 (Reactive Data Flow).
