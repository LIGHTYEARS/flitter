## Gaps Summary

Two related gaps found from a single root cause: **ThreadStateWidget.build() does not forward `_inferenceState` and `_error` state to ConversationView**.

Both variables are correctly tracked via `threadWorker.events$.subscribe()` and updated via `setState()`, but the `new ConversationView({ messages: this._messages })` call at line 256-258 only passes `messages`, omitting the `inferenceState` and `error` optional props. This means:

1. **Streaming indicator**: When inference is running, ConversationView's "..." indicator will not show because `inferenceState` defaults to undefined.
2. **Error display**: When inference errors occur, ConversationView's red error block with retry prompt will not show because `error` defaults to null/undefined.

**Fix:** Add `inferenceState: this._inferenceState, error: this._error` to the ConversationView constructor call in ThreadStateWidget.build(). This is a 1-line change.

The rest of the phase goal is solidly achieved. The three-layer gap is closed: Layer 1 (/dev/tty input), Layer 2 (CLI routing), and Layer 3 (real widget tree) are all implemented with substantive code, proper test coverage, and correct wiring throughout.
