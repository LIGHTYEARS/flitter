    - Calling cancelStream() transitions streaming/processing -> cancelled, finalizes messages, notifies listeners
    - Calling handleError(error) transitions any active state -> error, preserves error info, notifies listeners
    - Calling reset() transitions any terminal state (complete/error/cancelled) -> idle
    - Invalid transitions (e.g., idle -> streaming) are no-ops that log a warning

Create `packages/flitter-cli/src/provider/provider.ts` defining the abstract backend provider contract per CONTEXT.md decisions:

```typescript
/** Abstract provider interface — backends implement this contract. */
export interface Provider {
  /** Send a prompt and receive a stream of events. Caller owns the AbortController. */
  sendPrompt(
    messages: Array<{ role: string; content: string }>,
    options: PromptOptions,
  ): AsyncIterable<StreamEvent>;

  /** Cancel the current in-flight request. */
  cancelRequest(): void;

  /** Provider display name for status UI. */
  readonly name: string;

  /** Currently configured model identifier. */
  readonly model: string;
}

export interface PromptOptions {
  model?: string;
  maxTokens?: number;
  systemPrompt?: string;
  abortSignal?: AbortSignal;
}
```

Create `packages/flitter-cli/src/provider/anthropic.ts` implementing the Anthropic provider:
- Constructor takes `{ apiKey: string; model?: string; baseUrl?: string }`
- Resolves apiKey from constructor arg, falling back to `process.env.ANTHROPIC_API_KEY`
- Throws descriptive error if no API key found
- `sendPrompt()` makes HTTP POST to `{baseUrl}/v1/messages` with streaming enabled
- Parses SSE stream, yielding `StreamEvent` variants: `text_delta` for content_block_delta with text type, `thinking_delta` for thinking content, `tool_call_start`/`tool_call_end` for tool_use blocks, `usage_update` from message_start/message_delta usage fields, `message_complete` from message_stop, `error` from error events
- `cancelRequest()` aborts the current AbortController
- Uses native `fetch()` with no external HTTP dependencies (Bun provides this)
- Handles HTTP error responses (400, 401, 429, 500+) by yielding appropriate `StreamEvent.error` with retryable flag set for 429/500+
- Logs request/response metadata via the logger

Add function-level JSDoc comments on all exported items.
