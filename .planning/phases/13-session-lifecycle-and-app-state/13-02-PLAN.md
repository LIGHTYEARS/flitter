<task type="auto" tdd="true">
  <name>Task 1: Implement PromptController — the prompt submission and cancellation orchestrator</name>
  <files>packages/flitter-cli/src/state/prompt-controller.ts, packages/flitter-cli/src/__tests__/prompt-controller.test.ts</files>
  <behavior>
    - submitPrompt(text) drives session through processing -> streaming -> complete as stream events arrive
    - submitPrompt(text) while already processing is a no-op (prevents double submission)
    - Each StreamEvent type is dispatched to the correct SessionState method (text_delta -> appendAssistantChunk, etc.)
    - cancel() aborts the in-flight request via Provider.cancelRequest() and transitions session to cancelled
    - cancel() when not processing is a no-op
    - Provider errors (HTTP failures, stream errors) are caught and routed to session.handleError()
    - After completion or error, session lifecycle reflects the terminal state
    - Elapsed timer ticks during processing/streaming and stops on completion/error/cancel
  </behavior>
  <action>
Create `packages/flitter-cli/src/state/prompt-controller.ts`:

Constructor takes `{ session: SessionState; provider: Provider }`.

Implement `submitPrompt(text: string): Promise<void>`:
1. Guard: if session.lifecycle is not 'idle' and not 'complete' and not 'cancelled' and not 'error', return (prevent double-submit). If lifecycle is a terminal state (complete/cancelled/error), call session.reset() first.
2. Call `session.startProcessing(text)` to add user message and transition to processing
3. Start an elapsed-time interval timer (updates every 100ms via session metadata)
4. Build messages array from session.items (convert ConversationItems to role/content pairs for the provider)
5. Call `provider.sendPrompt(messages, { abortSignal })` to get the AsyncIterable
6. On first text_delta or thinking_delta event, call `session.beginStreaming()`
7. Dispatch each StreamEvent to SessionState:
   - `text_delta` -> `session.appendAssistantChunk(event.text)`
   - `thinking_delta` -> `session.appendThinkingChunk(event.text)`
   - `tool_call_start` -> `session.addToolCall(event.toolCallId, event.title, event.kind, 'pending')`
   - `tool_call_end` -> `session.updateToolCall(event.toolCallId, event.status, undefined, event.rawOutput)`
   - `usage_update` -> `session.setUsage(event.usage)`
   - `message_complete` -> `session.completeStream(event.stopReason)`, stop timer
   - `error` -> `session.handleError(event.error)`, stop timer
8. Wrap the stream iteration in try/catch. On exception, create a SessionError and call `session.handleError()`, stop timer.

Implement `cancel(): void`:
1. Guard: if session.lifecycle is not 'processing' and not 'streaming', return
2. Call `provider.cancelRequest()` to abort HTTP
3. Call `session.cancelStream()`
4. Stop elapsed timer

Private: `_buildMessages(items)` converts conversation items into the `{role, content}[]` format the provider expects. Map user_message to role:'user', assistant_message to role:'assistant'. Skip thinking, tool_call, plan, system_message items for the basic provider message format.

Write tests using a mock Provider that yields controlled StreamEvent sequences. Verify the full lifecycle dispatch, cancellation, error handling, and double-submit prevention.
  </action>

  </verify>

  <name>Task 2: Implement AppState — top-level state composition for the TUI</name>
  <files>packages/flitter-cli/src/state/app-state.ts</files>
  <action>
Create `packages/flitter-cli/src/state/app-state.ts` — the top-level application state that composes SessionState with UI-facing properties. Port the proven pattern from flitter-amp's AppState but replace ACP ClientCallbacks with native session delegation.

The class must:

- Own a `readonly session: SessionState` instance
- Own a `readonly promptController: PromptController` (set after construction to break circular init)
- Expose its own observer pattern: `addListener(fn)`, `removeListener(fn)`, `notifyListeners()` — when SessionState notifies, AppState relays to its own listeners (double-dispatch: session changes flow to AppState listeners)
- In the constructor: create SessionState, subscribe to session changes, relay to AppState listeners
- Expose session-delegated computed properties for UI consumption:
  - `get isProcessing(): boolean` — delegates to session.lifecycle being 'processing' or 'streaming'
  - `get isStreaming(): boolean` — delegates to session.lifecycle === 'streaming'
  - `get isInterrupted(): boolean` — session.lastStopReason is not null and not 'end_turn' and session is idle/complete
  - `get lifecycle(): SessionLifecycle` — delegates to session.lifecycle
  - `get error(): SessionError | null` — delegates to session.error
  - `get items()` — delegates to session.items
  - `get usage()` — delegates to session.usage
  - `get metadata()` — delegates to session.metadata
- Expose UI-specific state (not session concerns):
  - `denseView: boolean` (default false) — compact view toggle
  - `currentMode: string | null` (default null) — agent mode
  - `selectedMessageIndex: number | null` (default null) — Tab navigation
- Expose action methods that delegate to PromptController:
  - `submitPrompt(text: string): Promise<void>` — delegates to promptController.submitPrompt()
  - `cancelPrompt(): void` — delegates to promptController.cancel()
- Port convenience methods from flitter-amp AppState:
  - `newThread()` — delegates to session.newThread(), clears UI state
  - `toggleDenseView()` — flips denseView, notifies
  - `clearError()` — delegates to session.reset() if in error state, notifies
  - `getLastAssistantMessage(): string | null` — scans items in reverse
  - `getUsageSummary(): string | null` — formats usage data
  - `selectPrevMessage()`, `selectNextMessage()`, `clearMessageSelection()` — Tab/Shift+Tab navigation matching flitter-amp

- Static factory: `static create(config: { cwd: string; provider: Provider }): AppState` — constructs AppState, creates PromptController, wires them together, initializes session metadata with sessionId (crypto.randomUUID()), cwd, model from provider.

Add function-level JSDoc comments on all public methods.
