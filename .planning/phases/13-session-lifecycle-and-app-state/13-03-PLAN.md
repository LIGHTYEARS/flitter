Reuse the MockProvider from the app-state tests (or extract to a shared `__tests__/helpers/mock-provider.ts` if cleaner).

**Test: Happy path — complete prompt lifecycle**
1. Create AppState with MockProvider configured to yield: [text_delta("Hello"), text_delta(" world"), message_complete("end_turn")]
2. Assert lifecycle is 'idle'
3. Call `appState.submitPrompt("Hi")`
4. After promise resolves, assert lifecycle is 'complete'
5. Assert items has 2 entries: user_message("Hi") and assistant_message("Hello world") with isStreaming=false
6. Assert metadata.turnCount === 1
7. Assert lastStopReason === 'end_turn'
8. Assert isInterrupted is false

**Test: Thinking + text stream**
1. Configure MockProvider: [thinking_delta("Let me think"), text_delta("Response"), message_complete("end_turn")]
2. Submit prompt
3. Assert items contain thinking item and assistant message

**Test: Tool call lifecycle**
1. Configure MockProvider: [tool_call_start({...}), tool_call_end({...}), text_delta("Done"), message_complete("end_turn")]
2. Submit prompt
3. Assert tool call item exists with correct status

**Test: Cancellation mid-stream**
1. Configure MockProvider to yield text_delta events slowly (use a delayed async generator)
2. Start submitPrompt (do not await)
3. After brief delay, call `appState.cancelPrompt()`
4. Assert lifecycle transitions to 'cancelled'
5. Assert MockProvider.cancelCalled is true
6. Assert any partial assistant message is finalized (isStreaming=false)

**Test: Provider error propagation**
1. Configure MockProvider to yield: [text_delta("partial"), error({ message: "rate limited", code: "429", retryable: true })]
2. Submit prompt
3. Assert lifecycle is 'error'
4. Assert error.message === "rate limited"
5. Assert error.retryable is true
6. Assert partial assistant text is finalized

**Test: Provider exception propagation**
1. Configure MockProvider sendPrompt to throw an Error
2. Submit prompt
3. Assert lifecycle is 'error'
4. Assert error.message contains the thrown error message

**Test: Double-submit prevention**
1. Configure MockProvider with a slow stream
2. Start submitPrompt (do not await)
3. Immediately call submitPrompt again
4. Assert only one user message is added (second submit was no-op)

**Test: Recovery from terminal states**
1. Complete a prompt (lifecycle -> complete)
2. Submit a new prompt
3. Assert lifecycle transitions through reset -> idle -> processing (session.reset() called automatically)
4. Assert turnCount increments to 2

**Test: Session metadata accuracy**
1. Submit and complete a prompt
2. Assert metadata.turnCount === 1
3. Submit and complete another prompt
4. Assert metadata.turnCount === 2
5. Assert metadata.sessionId is unchanged between turns

**Test: Listener notification ordering**
1. Set up a listener that records lifecycle transitions
2. Submit a prompt with MockProvider yielding [text_delta, message_complete]
3. Assert the listener captured transitions in order: processing, streaming, complete

**Test: newThread() resets conversation**
1. Complete a prompt, assert items.length > 0
2. Call newThread()
3. Assert items.length === 0
4. Assert lifecycle === 'idle'
5. Assert metadata.sessionId unchanged
