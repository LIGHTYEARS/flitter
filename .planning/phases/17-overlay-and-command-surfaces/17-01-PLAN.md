# Plan 17-01: Overlay Infrastructure and OverlayManager Integration

**Phase:** 17 — Overlay and Command Surfaces
**Requirements:** OVLY-01 (partial), OVLY-02 (partial), OVLY-03 (partial), OVLY-04 (partial)
**Depends on:** Phase 16 complete (AppShell, InputArea, shortcut stubs)
**Status:** Pending
**Estimated new tests:** 0 (infrastructure wiring; tests in Plan 17-05 or Phase 22)

## Goal

Port the OverlayManager, overlay IDs/priorities, shortcut registry, and fuzzy-match utilities from `flitter-amp` into `flitter-cli`. Wire the OverlayManager into AppShellState so that all subsequent overlay plans (17-02 through 17-05) can show/dismiss overlays through a single, priority-sorted stack. Update AppShell's Escape handler to dismiss the top overlay before falling through to exit.

## Context

- `flitter-amp/src/state/overlay-manager.ts` is a self-contained 259-line class with no ACP-specific dependencies. It manages a priority-sorted stack of `OverlayEntry` objects and builds a `Stack` widget tree layering them on top of base content.
- `flitter-amp/src/state/overlay-ids.ts` defines well-known overlay identifiers and numeric priorities in a single constant object. This prevents magic strings/numbers across overlay consumers.
- `flitter-amp/src/shortcuts/registry.ts` provides a `ShortcutRegistry` for centralized shortcut dispatch and introspection. The help overlay (Plan 17-05) reads from this registry as its source of truth.
- `flitter-amp/src/utils/fuzzy-match.ts` provides `fuzzyMatch()`, `fuzzyMatchDetailed()`, and `scoreCommand()` used by both the command palette (Plan 17-03) and file picker (Plan 17-04).
- `flitter-cli/src/widgets/app-shell.ts` currently has inline shortcut stubs and an Escape handler that exits the app. This plan modifies Escape to check for active overlays first.
- `flitter-cli/src/state/app-state.ts` does not yet hold an `OverlayManager` reference.

## Implementation Steps

### Step 1: Port `OverlayManager` to `flitter-cli`

Create `packages/flitter-cli/src/state/overlay-manager.ts`.

Copy the full `OverlayManager` class, `OverlayEntry` interface, and `OverlayPlacement` type from `flitter-amp/src/state/overlay-manager.ts`. Adapt the import of `StateListener` to use the local type from `flitter-cli/src/state/session.ts` (which already exports `StateListener` as `() => void`).

No behavioral changes. The manager stays a standalone observable with `show()`, `dismiss()`, `dismissTop()`, `dismissAll()`, `buildOverlays()`.

### Step 2: Port `overlay-ids.ts` to `flitter-cli`

Create `packages/flitter-cli/src/state/overlay-ids.ts`.

Copy `OVERLAY_IDS` and `OVERLAY_PRIORITIES` from `flitter-amp/src/state/overlay-ids.ts`. Keep the same priority ranges:
- 100+: system modals (permission dialog)
- 50-99: user-triggered modals (command palette, shortcut help)
- 25-49: contextual popups (file picker, autocomplete)
- 1-24: passive/non-modal (toasts)

Remove `THREAD_LIST` and `PROMPT_HISTORY` entries (Phase 21 scope). Remove `MYSTERY_MODAL` (not needed in flitter-cli).

### Step 3: Port fuzzy-match utilities to `flitter-cli`

Create `packages/flitter-cli/src/utils/fuzzy-match.ts`.

Copy `fuzzyMatch()`, `fuzzyMatchDetailed()`, and `scoreCommand()` from `flitter-amp/src/utils/fuzzy-match.ts`. No changes needed — the module has zero external dependencies.

### Step 4: Port `ShortcutRegistry` to `flitter-cli`

Create `packages/flitter-cli/src/shortcuts/registry.ts`.

Copy the `ShortcutRegistry` class, `ShortcutEntry` interface, and `ShortcutCategory` type from `flitter-amp/src/shortcuts/registry.ts`.

Adapt `ShortcutContext` to reference `flitter-cli` types:
- Replace `AppState` import with `flitter-cli/src/state/app-state`
- Replace `OverlayManager` import with `flitter-cli/src/state/overlay-manager`
- Remove `PromptHistory` reference (Phase 21 scope — leave as optional)
- Keep `ShortcutHooks` interface for widget-level actions

### Step 5: Create `flitter-cli` default shortcuts

Create `packages/flitter-cli/src/shortcuts/defaults.ts`.

Register the Phase 16 shortcut matrix using the registry pattern. Map each existing AppShellState handler to a registry entry:

| Shortcut | ID | Category | Action |
|----------|----|----------|--------|
| Escape | `dismiss-overlay` | general | `overlayManager.dismissTop()` or fall through |
| Ctrl+O | `open-command-palette` | general | Toggle command palette overlay |
| Ctrl+C | `cancel-operation` | general | Cancel prompt or exit |
| Ctrl+L | `clear-conversation` | general | `appState.newThread()` |
| Alt+T | `toggle-dense-view` | display | `appState.toggleDenseView()` |
| Ctrl+G | `open-editor` | navigation | Stub (INPT-06) |
| Ctrl+R | `history-search` | navigation | Stub (Phase 21) |
| ? | `toggle-shortcut-help` | general | Toggle shortcut help overlay (guard: not processing, no overlays) |

Create `packages/flitter-cli/src/shortcuts/index.ts` that re-exports the registry and defaults.

### Step 6: Wire `OverlayManager` into `AppState`

Modify `packages/flitter-cli/src/state/app-state.ts`:

- Add an `overlayManager: OverlayManager` property, constructed in the `AppState` constructor
- Expose it as a public readonly field
- In `AppState.create()`, the overlay manager is constructed alongside session state

This makes `OverlayManager` accessible to all code that has an `AppState` reference.

### Step 7: Wire `OverlayManager` and `ShortcutRegistry` into `AppShellState`

Modify `packages/flitter-cli/src/widgets/app-shell.ts`:

1. **Create `ShortcutRegistry` in `initState()`**: Construct a registry, call `registerDefaultShortcuts(registry)`.

2. **Replace inline key handlers with registry dispatch**: Change `_handleKey()` to call `registry.dispatch(event, context)` first. If the registry returns `'handled'`, return immediately. Otherwise fall through to any remaining non-registered handlers.

3. **Build `ShortcutContext`**: Construct the context object from `AppState`, `OverlayManager`, `ShortcutHooks`, and `setState`.

4. **Wrap content with `buildOverlays()`**: In `build()`, after constructing the layout column, wrap it with `appState.overlayManager.buildOverlays(layoutWidget)`. This inserts the overlay Stack layer at the root.

5. **Register `OverlayManager` listener**: In `initState()`, add a listener on `overlayManager` to trigger `setState()` when overlays change.

6. **Update Escape handling**: The Escape shortcut in the registry handles overlay dismissal. The remaining Escape-exit behavior is either a fallback in the registry action or stays inline for the no-overlay case.

### Step 8: Remove redundant inline handlers

After Step 7, the following inline handlers in `AppShellState` become dead code because they are now in the registry:

- `_handleCtrlO()` — replaced by `open-command-palette` registry entry
- `_handleCtrlL()` — replaced by `clear-conversation` registry entry
- `_handleAltT()` — replaced by `toggle-dense-view` registry entry
- `_handleCtrlR()` — replaced by `history-search` registry entry
- `_handleCtrlG()` — replaced by `open-editor` registry entry
- `_handleQuestionMark()` — replaced by `toggle-shortcut-help` registry entry
- `_handleEscape()` — replaced by `dismiss-overlay` registry entry (with exit fallback)

Remove these methods and the `switch` statements in `_handleKey()`. Keep `_handleCtrlC()` in the registry as well.

## Verification

- [ ] `OverlayManager` can show, dismiss, and dismissTop entries by id
- [ ] `buildOverlays(baseContent)` returns base content directly when no overlays active
- [ ] `buildOverlays(baseContent)` returns a Stack with modal masks and positioned overlay widgets when entries exist
- [ ] `OVERLAY_IDS` and `OVERLAY_PRIORITIES` constants are importable from `flitter-cli`
- [ ] `fuzzyMatch('cl', 'Clear conversation')` returns a positive score; `fuzzyMatch('zzzz', 'Clear')` returns null
- [ ] `ShortcutRegistry` dispatches Ctrl+L to the `clear-conversation` action
- [ ] `ShortcutRegistry` dispatches Escape to `dismiss-overlay`, which calls `dismissTop()` when overlays exist
- [ ] `AppState.overlayManager` is accessible and functional
- [ ] `AppShell.build()` wraps content in `overlayManager.buildOverlays()` so overlay entries render on top
- [ ] Pressing Escape with no overlays still exits the application (fallback behavior preserved)
- [ ] Pressing Escape with an active overlay dismisses it instead of exiting

## Decisions

- **OverlayManager lives on AppState, not AppShellState.** This allows non-widget code (e.g., permission request handlers in the provider layer) to show overlays without needing a widget reference.
- **ShortcutRegistry replaces the inline switch/case handlers.** This centralizes all shortcut definitions so the help overlay (Plan 17-05) can read the same source of truth.
- **Phase 21 shortcuts (Ctrl+R history search) remain as stubs** in the registry. They log a message and return 'handled'. The registry entry exists so the help overlay lists it.
- **No theme integration yet.** Colors in OverlayManager masks and overlay widgets use hardcoded defaults. Phase 20 adds the theme system.
