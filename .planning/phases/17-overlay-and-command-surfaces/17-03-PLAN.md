# Plan 17-03: Command Palette Overlay

**Phase:** 17 — Overlay and Command Surfaces
**Requirements:** OVLY-02, OVLY-05
**Depends on:** Plan 17-01 (OverlayManager, overlay IDs, ShortcutRegistry, fuzzy-match)
**Status:** Pending
**Estimated new tests:** 0 (widget construction verified manually; automated tests in Phase 22)

## Goal

Implement the command palette overlay for `flitter-cli`. The palette is invoked via Ctrl+O, renders a searchable list of actions with labels, descriptions, and shortcut hints, and dispatches the selected action. The command inventory and descriptions are derived from the `ShortcutRegistry` (Plan 17-01) so they stay in sync with bound shortcuts automatically.

## Context

- `flitter-amp/src/widgets/command-palette.ts` is a 228-line `StatefulWidget` with `TextField` fuzzy search and `SelectionList`. It uses `AmpThemeProvider` for colors (replaced with hardcoded defaults here).
- `flitter-amp` uses a static `COMMANDS` array as the default command list. In `flitter-cli`, we derive the command list from the `ShortcutRegistry` plus additional commands that are not keyboard-bound.
- `scoreCommand()` from `fuzzy-match.ts` (ported in Plan 17-01) provides combined label + description scoring.
- The command palette is shown at priority `OVERLAY_PRIORITIES.COMMAND_PALETTE` (50) as a non-modal overlay (no background dimming), matching Amp behavior.
- The Ctrl+O shortcut in `defaults.ts` (Plan 17-01) toggles the palette: opens it if closed, closes it if open.
- `SelectionList` from `flitter-core` provides arrow-key navigation, Enter selection, and Escape cancel.

## Implementation Steps

### Step 1: Define the command action registry

Create `packages/flitter-cli/src/commands/command-registry.ts`:

```typescript
/**
 * A single command available in the command palette.
 */
export interface CommandItem {
  /** Unique identifier (matches shortcut id when derived from registry). */
  readonly id: string;
  /** Display label in the palette. */
  readonly label: string;
  /** Detailed description shown below the label. */
  readonly description: string;
  /** Optional shortcut hint shown at the right edge (e.g., "Ctrl+L"). */
  readonly shortcutHint?: string;
  /** The action to execute. Receives an onDismiss callback for self-dismissal. */
  readonly execute: (onDismiss: () => void) => void;
}
```

Create a `buildCommandList()` function that takes a `ShortcutRegistry` and `AppState` and returns `CommandItem[]`:

1. **From ShortcutRegistry**: For each entry, create a `CommandItem` with `id`, `label` from description, `shortcutHint` from displayKey, and `execute` that calls the registry entry's action.
2. **Additional commands**: Add commands not bound to shortcuts:
   - "New thread" — `appState.newThread()`
   - "Copy last response" — clipboard copy of last assistant message
   - "View usage" — log/display usage summary
   - "Toggle tool calls" — `appState.toggleDenseView()`

The list is rebuilt each time the palette opens, so it always reflects current state.

### Step 2: Create `CommandPalette` widget

Create `packages/flitter-cli/src/widgets/command-palette.ts`:

Port the `flitter-amp/src/widgets/command-palette.ts` pattern with these changes:
- Accept `commands: SelectionItem[]` derived from `CommandItem[]` (mapped at show-time)
- Accept `onExecute: (commandId: string) => void` and `onDismiss: () => void`
- Replace `AmpThemeProvider` color lookups with hardcoded defaults (cyan for info, brightBlack for muted)

Widget structure:
```
FocusScope (autofocus, onKey: Escape -> onDismiss)
  Column (mainAxisAlignment: 'start', crossAxisAlignment: 'center')
    SizedBox (height: 2)  // top margin
    Container (bordered, padded, maxWidth: 60)
      Column (mainAxisSize: 'min', crossAxisAlignment: 'start')
        Text "Command Palette" (cyan, bold)
        SizedBox (height: 1)
        Container (bottom border underline)
          TextField (search input, autofocus, maxLines: 1)
        SizedBox (height: 1)
        Text "{shown}/{total} commands" (muted, dim)
        SelectionList (filtered items, onSelect, onCancel)
        OR Text "No matching commands" (muted, italic)
```

State management:
- `searchController: TextEditingController` — listens for text changes
- `filteredItems: SelectionItem[]` — recomputed on search change using `scoreCommand()`
- `currentQuery: string` — cached to avoid redundant refilter

### Step 3: Wire command execution dispatch

When the user selects a command from the palette:

1. `onExecute(commandId)` is called
2. The caller (AppShellState) looks up the `CommandItem` by id
3. Calls `commandItem.execute(onDismiss)`
4. The palette is dismissed (either by the execute handler or by the caller)

For shortcut-bound commands, execution re-uses the shortcut's action. For non-shortcut commands, dedicated handlers run directly.

### Step 4: Wire Ctrl+O shortcut to show/toggle palette

In `packages/flitter-cli/src/shortcuts/defaults.ts` (created in Plan 17-01), the `open-command-palette` entry uses `ctx.hooks.showCommandPalette()`.

In `packages/flitter-cli/src/widgets/app-shell.ts`, implement the `showCommandPalette` hook:

```typescript
showCommandPalette(): void {
  const overlayManager = this.widget.appState.overlayManager;
  if (overlayManager.has(OVERLAY_IDS.COMMAND_PALETTE)) {
    overlayManager.dismiss(OVERLAY_IDS.COMMAND_PALETTE);
    return;
  }
  const commands = buildCommandList(this.shortcutRegistry, this.widget.appState);
  const items: SelectionItem[] = commands.map(cmd => ({
    label: cmd.label,
    value: cmd.id,
    description: cmd.description,
  }));

  overlayManager.show({
    id: OVERLAY_IDS.COMMAND_PALETTE,
    priority: OVERLAY_PRIORITIES.COMMAND_PALETTE,
    modal: false,
    placement: { type: 'fullscreen' },
    builder: (onDismiss) => new CommandPalette({
      commands: items,
      onExecute: (cmdId) => {
        onDismiss();
        const cmd = commands.find(c => c.id === cmdId);
        if (cmd) cmd.execute(onDismiss);
      },
      onDismiss,
    }),
  });
}
```

### Step 5: Ensure Escape dismissal

The `CommandPalette` widget already handles Escape internally via its `FocusScope.onKey` handler, which calls `onDismiss()`. The OverlayManager's `dismiss-overlay` shortcut (Plan 17-01) also handles Escape at the shell level. Both paths converge on `overlayManager.dismiss(OVERLAY_IDS.COMMAND_PALETTE)`.

The `FocusScope` inside `CommandPalette` catches Escape first (because it has focus), calls `onDismiss`, which calls `overlayManager.dismiss()`. If for any reason the palette's FocusScope does not catch it, the shell-level `dismiss-overlay` shortcut catches it via `dismissTop()`.

## Decisions

- **Command inventory is derived from ShortcutRegistry.** This ensures the command palette always reflects the actual bound shortcuts. Additional commands (new thread, copy response, view usage) are appended as non-shortcut items.
- **Non-modal overlay.** The command palette does not dim the background, matching Amp behavior. The user can still see the chat behind it.
- **No theme integration yet.** Colors use hardcoded defaults (cyan info, brightBlack muted). Phase 20 adds theme support.
- **Fuzzy scoring via `scoreCommand()`.** Combined label + description scoring with description at half weight, matching the `flitter-amp` pattern.
