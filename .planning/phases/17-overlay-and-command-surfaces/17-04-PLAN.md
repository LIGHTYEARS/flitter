# Plan 17-04: File Picker and Autocomplete Overlays

**Phase:** 17 — Overlay and Command Surfaces
**Requirements:** OVLY-03
**Depends on:** Plan 17-01 (OverlayManager, overlay IDs, fuzzy-match)
**Status:** Pending
**Estimated new tests:** 0 (widget construction verified manually; automated tests in Phase 22)

## Goal

Implement the file picker overlay and wire the `@` trigger in InputArea's `Autocomplete` widget so that typing `@` in the input area opens a fuzzy-searchable file list. The selected file path is injected back into the input text. The file picker also has a standalone overlay mode triggered from the command palette or programmatically.

## Context

- `flitter-amp/src/widgets/file-picker.ts` is a 214-line `StatefulWidget` with `TextField` fuzzy search and `SelectionList`, anchored near the input area (bottom-aligned).
- `flitter-core/src/widgets/autocomplete.ts` already supports trigger-based completion. InputArea currently wraps TextField in `Autocomplete` with empty triggers (Phase 16, Step 9). This plan populates the `@` trigger.
- The Autocomplete widget handles the inline popup rendering via a Column that places the options list above the child. The file picker overlay (via OverlayManager) is a separate, richer surface used for the standalone invocation path.
- Two invocation paths:
  1. **Inline autocomplete** (`@` trigger): Uses the `Autocomplete` widget's built-in popup. Lightweight, appears inline above the input.
  2. **Standalone file picker** (from command palette or programmatic): Uses `OverlayManager` with anchored placement. Richer UI with search field and bordered container.
- File listing requires reading the working directory. A `listProjectFiles()` utility provides the file list, using `fd` or `readdir` with reasonable depth limits.

## Implementation Steps

### Step 1: Create file listing utility

Create `packages/flitter-cli/src/utils/file-list.ts`:

```typescript
/**
 * List project files from the working directory for autocomplete and picker.
 *
 * Uses a shallow traversal with reasonable depth limit (default: 4 levels).
 * Excludes common noise directories (node_modules, .git, dist, build).
 * Returns relative paths sorted alphabetically.
 */
export async function listProjectFiles(
  cwd: string,
  options?: { maxDepth?: number; maxFiles?: number },
): Promise<string[]>;
```

Implementation approach:
- Use `Bun.spawn(['fd', '--type', 'f', '--max-depth', depth, '.'], { cwd })` when `fd` is available (respects `.gitignore` by default).
- Fallback to `readdir` recursive traversal with manual exclusion patterns if `fd` is not found.
- Cap results to `maxFiles` (default: 500) to prevent huge directories from blocking the UI.
- Cache the result on AppState or a dedicated file cache with a short TTL (e.g., 5 seconds) to avoid re-scanning on every keystroke.

### Step 3: Wire `@` trigger in InputArea's Autocomplete

Modify `packages/flitter-cli/src/widgets/input-area.ts`:

Change the Autocomplete `triggers` from `[]` to include the `@` file trigger:

```typescript
const fileAutocompleteTrigger: AutocompleteTrigger = {
  triggerCharacter: '@',
  optionsBuilder: (query: string) => {
    // Return cached file list filtered by query
    // Files are loaded asynchronously and cached
    return this._getFileOptions(query);
  },
};

const autocompleteWrapped = new Autocomplete({
  child: textField,
  controller: this.controller,
  triggers: [fileAutocompleteTrigger],
  onSelected: (trigger, option) => {
    // The Autocomplete widget handles text replacement automatically
    // via its _selectOption method
  },
  maxOptionsVisible: 10,
});
```

The `_getFileOptions` method on `InputAreaState`:
- Reads from a cached file list (see Step 5)
- Returns `AutocompleteOption[]` with label = basename, value = relative path
- Filtering is handled by Autocomplete's built-in fuzzy match, but the options builder can also pre-filter for better performance

### Step 5: Implement file list caching

Add file list caching to `InputAreaState` to avoid re-scanning the filesystem on every keystroke:

```typescript
private _cachedFiles: string[] = [];
private _cacheTimestamp = 0;
private _cacheLoading = false;
private static CACHE_TTL_MS = 5000;

/**
 * Get file options for autocomplete, using a cached file list.
 * Refreshes the cache if it's older than CACHE_TTL_MS.
 */
private _getFileOptions(query: string): AutocompleteOption[] {
  const now = Date.now();
  if (now - this._cacheTimestamp > InputAreaState.CACHE_TTL_MS && !this._cacheLoading) {
    this._refreshFileCache();
  }
  // Return currently cached files (may be stale for one cycle)
  return this._cachedFiles.map(f => ({
    label: basename(f),
    value: f,
    description: f,
  }));
}
```

### Step 6: Wire standalone file picker via OverlayManager

In `packages/flitter-cli/src/widgets/app-shell.ts`, add a `showFilePicker` hook available from the command palette:

```typescript
showFilePicker(): void {
  const overlayManager = this.widget.appState.overlayManager;
  const cwd = this.widget.appState.metadata.cwd;

  listProjectFiles(cwd).then(files => {
    overlayManager.show({
      id: OVERLAY_IDS.FILE_PICKER,
      priority: OVERLAY_PRIORITIES.FILE_PICKER,
      modal: false,
      placement: { type: 'anchored', left: 2, bottom: 3 },
      builder: (onDismiss) => new FilePicker({
        files,
        onSelect: (filePath: string) => {
          onDismiss();
          // Inject @filePath into the input controller
          this.textController.insert('@' + filePath + ' ');
        },
        onDismiss,
      }),
    });
  });
}
```

The `placement: { type: 'anchored', left: 2, bottom: 3 }` positions the file picker just above the input area.

## Decisions

- **Two invocation paths, shared file list.** The inline `@` autocomplete and the standalone file picker both use `listProjectFiles()` but through different UI surfaces. The inline path uses Autocomplete's built-in popup. The standalone path uses OverlayManager with a richer search UI.
- **`fd` preferred, `readdir` fallback.** `fd` respects `.gitignore` and is fast. If `fd` is not available, a manual `readdir` traversal with exclusion patterns provides adequate functionality.
- **File list caching on InputAreaState.** The cache lives on the widget state, not on AppState, because file listing is a UI-specific concern. The 5-second TTL balances freshness with performance.
- **No theme integration yet.** Colors (green border, brightBlack muted) are hardcoded. Phase 20 adds theme support.
- **Anchored placement for standalone picker.** `bottom: 3` accounts for the input area height (3 rows at minimum). The picker appears just above the input, matching Amp's visual placement.
