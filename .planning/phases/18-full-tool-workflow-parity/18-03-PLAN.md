# Plan 18-03: Specialized Tool Renderers — Bash, Read, Grep, Edit, Create, WebSearch, Todo

**Phase:** 18 — Full Tool Workflow Parity
**Requirements:** TOOL-03, TOOL-05, TOOL-06 (partial)
**Depends on:** Plan 18-02 (ToolHeader, GenericToolCard, ToolCallWidget dispatch in ChatView)
**Status:** Pending
**Estimated new tests:** 0 (widget rendering; visual verification and Phase 22 tests)

## Goal

Create the seven specialized tool renderers that cover the most common tool types in the Amp workflow: BashTool, ReadTool, GrepTool, EditFileTool, CreateFileTool, WebSearchTool, and TodoListTool. Wire each into the ToolCallWidget dispatch switch. After this plan, all high-frequency tool types render with tool-specific formatting (command display, file paths, search patterns, diff stats, todo checklists) instead of the generic fallback.

## Context

- `flitter-amp/src/widgets/tool-call/` contains complete reference implementations for all seven renderers. Each follows the same pattern: ToolHeader with tool-specific details, optional expanded body, and tool-specific output extraction.
- Plan 18-01 ported all shared utilities: `raw-input.ts`, `tool-output-utils.ts`, `truncation-limits.ts`, `resolve-tool-name.ts`, `base-tool-props.ts`.
- Plan 18-02 created ToolHeader, GenericToolCard, and the ToolCallWidget dispatch shell. The dispatch currently routes everything to GenericToolCard.
- `flitter-cli` does not have Markdown, DiffView, or theme provider yet. Text output is rendered as plain `Text` widgets. Diff content is rendered as pre-formatted text with +/- line coloring. Phase 19 adds proper Markdown and DiffView.
- All seven renderers are StatelessWidget (no timers or animation). Only HandoffTool (Plan 18-04) requires StatefulWidget.

## Implementation Steps

### Step 1: Create BashTool

Create `packages/flitter-cli/src/widgets/tool-call/bash-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/bash-tool.ts`:
- **Header:** ToolHeader with name=resolveToolDisplayName, details=`['$ {command}']` (command extracted via `pickString(input, ['command', 'Command', 'cmd', 'shell_command', 'script', 'args'])`).
- **Body (expanded):** Streaming output (tail + block cursor during isStreaming, head + truncation after complete), exit code line (green for 0, red for non-zero).
- **Adaptations:** Replace AmpThemeProvider color lookups with direct `Color.*` constants. Import `extractShellOutput`, `extractRawNumber` from `./tool-output-utils`. Import `pickString` from `../../utils/raw-input`. Import `truncateInline`, `truncateText`, `HEADER_TRUNCATION_LIMIT`, `OUTPUT_TRUNCATION_LIMIT` from `./truncation-limits`.

### Step 2: Create ReadTool

Create `packages/flitter-cli/src/widgets/tool-call/read-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/read-tool.ts`:
- **Header:** ToolHeader with details=[shortenPath(filePath), optional line range `L{offset}-{offset+limit}`]. File path extracted via `pickString(input, ['file_path', 'path', 'filename', 'file'])`.
- **Body (expanded):** File content as dim pre-formatted text, truncated to `OUTPUT_TRUNCATION_LIMIT`.
- **Adaptations:** Replace AmpThemeProvider with direct colors. Import `pickString`, `pickNumber` from `../../utils/raw-input`.

### Step 3: Create GrepTool

Create `packages/flitter-cli/src/widgets/tool-call/grep-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/grep-tool.ts`:
- **Header:** ToolHeader with details=[`/{pattern}/`, shortenPath(path), optional `({matchCount} matches)`]. Pattern extracted via `pickString(input, ['pattern', 'query', 'glob', 'search_pattern', 'regex', 'search'])`.
- **Body (expanded):** Search results as dim pre-formatted text, truncated to `OUTPUT_TRUNCATION_LIMIT`.
- **Adaptations:** Replace AmpThemeProvider with direct colors. Import `extractRawNumber` from `./tool-output-utils` for match count.

### Step 4: Create EditFileTool

Create `packages/flitter-cli/src/widgets/tool-call/edit-file-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/edit-file-tool.ts`:
- **Header:** ToolHeader with details=[shortenPath(filePath)]. File path extracted via `pickString(input, ['file_path', 'path'])`.
- **Body (expanded):**
  1. Diff stats line: `+N` (green) / `-M` (red) lines count.
  2. Diff content as pre-formatted text. Lines starting with `+` (not `+++`) colored green; lines starting with `-` (not `---`) colored red; context lines dim.
  3. Fallback: if no diff found, show extractOutputText as dim text.
- **Diff extraction:** Use `extractDiff()` from tool-output-utils. If null, synthesize from `old_str`/`new_str` in rawInput using the same `--- a/+++ b/@@ @@` format as flitter-amp.
- **Adaptations:** Replace `DiffView` widget with inline diff coloring using TextSpan children (one span per line with color based on +/- prefix). This is a temporary approach until Phase 19 provides proper DiffView. Replace AmpThemeProvider with direct colors.

### Step 5: Create CreateFileTool

Create `packages/flitter-cli/src/widgets/tool-call/create-file-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/create-file-tool.ts`:
- **Header:** ToolHeader with details=[shortenPath(filePath)]. File path extracted via `pickString(input, ['file_path', 'path', 'filename', 'file', 'destination'])`.
- **Body (expanded):**
  1. Content preview (dim text from `pickString(input, ['content'])`, truncated to PREVIEW_TRUNCATION_LIMIT).
  2. Result text (green for success, red for failure).
- **Adaptations:** Replace AmpThemeProvider with direct colors.

### Step 6: Create WebSearchTool

Create `packages/flitter-cli/src/widgets/tool-call/web-search-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/web-search-tool.ts`:
- **Header:** ToolHeader with details=[query]. Query extracted via `pickString(input, ['query', 'url', 'search_query', 'q', 'search'])`.
- **Body (expanded):**
  1. Extracted URLs rendered as cyan-colored lines with `'\u2192'` arrow prefix.
  2. If no URLs found, fallback to extractOutputText as dim text.
- **URL extraction:** Use `extractRawArray(result, ['results', 'links'])`, filter with `isRecord`, extract `pickString(r, ['url', 'link', 'href'])`, limited to `MAX_DISPLAY_ITEMS`.
- **Adaptations:** Replace AmpThemeProvider with direct colors. Replace `icon('arrow.right')` with `'\u2192'` constant from tool-icons.ts.

### Step 7: Create TodoListTool

Create `packages/flitter-cli/src/widgets/tool-call/todo-list-tool.ts`.

Port from `flitter-amp/src/widgets/tool-call/todo-list-tool.ts`:
- **Header:** ToolHeader with details=[extractTitleDetail(toolCall)].
- **Body (expanded):** List of todo entries, each rendered as:
  `  {statusIcon} {content}` — with status-specific icon and color.
  - pending: empty circle, dim
  - in_progress: half circle, yellow
  - completed: checkmark, green (content dim/strikethrough)
  - cancelled: cross, dim
- **Entry extraction:** From rawInput.todos via `asArray(input['todos'], isTodoEntry)`, then from result.rawOutput.todos/items as fallback.
- **Adaptations:** Replace AmpThemeProvider with direct colors. Replace `icon('todo.status.*')` with `todoStatusIcon()` from tool-icons.ts.

### Step 8: Wire all renderers into ToolCallWidget dispatch

Modify `packages/flitter-cli/src/widgets/tool-call/tool-call-widget.ts`.

Replace the GenericToolCard-only dispatch with the full switch:

```typescript
switch (name) {
  case 'Read':
    return new ReadTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  case 'edit_file':
  case 'apply_patch':
  case 'undo_edit':
    return new EditFileTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  case 'create_file':
    return new CreateFileTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  case 'Bash':
  case 'shell_command':
  case 'REPL':
    return new BashTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  case 'Grep':
  case 'glob':
  case 'Glob':
  case 'Search':
  case 'LS':
    return new GrepTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  case 'WebSearch':
  case 'read_web_page':
    return new WebSearchTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  case 'todo_list':
  case 'todo_write':
  case 'todo_read':
    return new TodoListTool({ toolCall, isExpanded: expanded, onToggle: toggle });
  // Plan 18-04 adds: HandoffTool, TaskTool
  default:
    return new GenericToolCard({ toolCall, isExpanded: expanded, onToggle: toggle });
}
```

Also keep the `sa__`/`tb__` prefix check routing to GenericToolCard for now (Plan 18-04 changes this to TaskTool).

## Decisions

- **Inline diff coloring instead of DiffView widget.** EditFileTool renders diff as a Column of TextSpan lines with +/- colors. Phase 19 introduces proper DiffView. The EditFileTool will be updated at that point to delegate to DiffView. This is documented as a known upgrade path.
- **All seven renderers are StatelessWidget.** No timers, no animation, no internal mutable state. Render output is a pure function of toolCall props. Only HandoffTool (Plan 18-04) needs StatefulWidget for its blink animation.
- **No special handling for apply_patch or undo_edit.** They route to EditFileTool since their output structure (diff content) is the same. If future providers emit different shapes, the generic fallback already catches them.
- **URL extraction uses the same key candidates as flitter-amp.** This covers Anthropic's web_search, Coco's WebFetch, and common MCP shapes.
