# Plan 19-01: Markdown Renderer and Syntax Highlighting Integration

**Phase:** 19 — Markdown, Diff, Thinking, and Plan Rendering
**Requirements:** REND-01, REND-02
**Wave:** 1 (no dependencies on other plans in this phase)

## Goal

Replace the plain-text assistant message rendering in ChatView with flitter-core's `Markdown` widget, and wire syntax highlighting for code blocks. This covers headings, paragraphs, inline formatting (bold, italic, code spans, links), code fences, tables, blockquotes, horizontal rules, and mixed content blocks at Amp parity.

## Current State

1. **ChatView.buildAssistantTurnWidget** (chat-view.ts:276-289) renders `turn.message.text` as a raw `Text` widget with a block cursor (`\u2588`) suffix during streaming. No markdown parsing.
2. **GenericToolCard** (generic-tool-card.ts:140-174) renders diff and output sections as plain `Text`. Comments explicitly note "Phase 19 will swap Text -> DiffView / Markdown".
3. **BashTool** (bash-tool.ts:87-99) renders shell output as plain dim `Text`.
4. **flitter-core `Markdown`** widget (markdown.ts) already supports: headings (h1-h4), paragraphs, bullet lists, numbered lists, blockquotes, horizontal rules, code blocks with optional syntax highlighting, tables (GFM), and inline formatting (bold, italic, code spans, links with OSC 8).
5. **flitter-core `syntaxHighlight`** (syntax-highlight.ts) supports 25+ languages with `AppTheme`-driven token coloring.
6. **flitter-amp `StreamingCursor`** (streaming-cursor.ts) already wraps `Markdown` for streaming content with a blinking cursor — this is the reference implementation.

## Approach

Port the `StreamingCursor` pattern from flitter-amp into flitter-cli as a new `StreamingCursor` widget. Then replace the plain text rendering in `buildAssistantTurnWidget` with `StreamingCursor`, and replace plain text output in `GenericToolCard` with `Markdown`.

### Why StreamingCursor as a separate widget

The cursor blink timer is stateful (requires `StatefulWidget` + `setInterval`). Keeping this in a dedicated widget keeps ChatView stateless w.r.t. cursor animation, exactly matching the flitter-amp architecture.

## Changes

### File 1: `packages/flitter-cli/src/widgets/streaming-cursor.ts` (NEW)

**Action:** Create new file.

Port from `flitter-amp/src/widgets/streaming-cursor.ts` with these adaptations:
- Import paths use relative `../../../../flitter-core/...` instead of package specifiers.
- Replace `AmpThemeProvider.maybeOf(context)` color lookups with direct `Color.*` constants (consistent with flitter-cli convention; Phase 20 adds theme).
- Retain all timer lifecycle logic: `initState()` -> conditionally start blink timer; `didUpdateWidget()` -> start/stop on streaming transitions; `dispose()` -> stop timer.
- Retain the `Markdown` delegation for both streaming and non-streaming paths.
- Retain the blinking cursor character (`\u258C`) and blink interval (530ms).
- Retain the empty-streaming standalone cursor rendering.
- Export `CURSOR_BLINK_INTERVAL_MS` constant.
- Expose `isCursorVisible` and `isBlinking` getters for testability.

Key build logic (from reference):
```typescript
build(context: BuildContext): Widget {
  const { text, isStreaming } = this.widget;
  if (!isStreaming) {
    if (text.length > 0) return new Markdown({ markdown: text });
    return new Text({ /* '...' muted */ });
  }
  const cursorSuffix = this.cursorVisible ? ` ${CURSOR_CHAR}` : '';
  if (text.length > 0) return new Markdown({ markdown: text + cursorSuffix });
  return new Text({ /* standalone blinking cursor */ });
}
```
