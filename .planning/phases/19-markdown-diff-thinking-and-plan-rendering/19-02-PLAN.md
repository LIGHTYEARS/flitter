# Plan 19-02: DiffView Integration for Edit-File and Generic Tool Surfaces

**Phase:** 19 — Markdown, Diff, Thinking, and Plan Rendering
**Requirements:** REND-03
**Wave:** 1 (independent of other plans; can execute in parallel with 19-01)

## Goal

Replace the inline diff coloring in `EditFileTool` and the plain-text diff rendering in `GenericToolCard` with flitter-core's `DiffView` widget. This provides unified diff rendering with line numbers, hunk headers, word-level diff highlighting, and proper color semantics at Amp parity.

## Current State

1. **EditFileTool** (edit-file-tool.ts:109-119) renders diffs using a hand-rolled `buildDiffSpans()` method that colors `+` lines green and `-` lines red via TextSpan children. No line numbers, no hunk headers, no word-level diff.
2. **GenericToolCard** (generic-tool-card.ts:140-155) renders extracted diff text as plain dim Text — no coloring at all.
3. **flitter-core `DiffView`** (diff-view.ts) already provides: unified diff parsing, line-by-line coloring (green additions, red deletions, cyan hunk headers, dim context), word-level diff highlighting, optional line numbers, context filtering, and `Theme`-driven color resolution (`diffAdded`, `diffRemoved`, `info`, `text`).
4. **flitter-amp `DiffCard`** (diff-card.ts) wraps `DiffView` in a bordered Container with a file path header and `Theme` color overrides — this is the reference.

## Approach

Create a `DiffCard` widget in flitter-cli that wraps flitter-core's `DiffView` with a file path header and a bordered container. Then replace the inline diff rendering in `EditFileTool` and `GenericToolCard` with `DiffCard`.

The `DiffCard` follows the flitter-amp reference but uses direct `Color.*` constants instead of `AmpThemeProvider` (consistent with flitter-cli Phase 20 deferral).

## Changes

### File 1: `packages/flitter-cli/src/widgets/diff-card.ts` (NEW)

**Action:** Create new file.

Port from `flitter-amp/src/widgets/diff-card.ts` with these adaptations:
- Import paths use relative `../../../../flitter-core/...`.
- Replace `AmpThemeProvider.maybeOf(context)` with direct `Color.*` constants:
  - `borderColor` -> `Color.brightBlack`
  - `mutedForeground` for file path -> `Color.brightBlack` with `bold: true`
  - `diffAdded` -> `Color.green`
  - `diffRemoved` -> `Color.red`
- Import `DiffView` from `../../../../flitter-core/src/widgets/diff-view`.
- Import `Theme` from `../../../../flitter-core/src/widgets/theme` to inject diff colors.
- Layout: `Padding(left:4, right:2, bottom:1)` -> `Container(border: Border.all(rounded))` -> `Column(filePath header, DiffView)`.
- Constructor takes `{ filePath: string; diff: string }`.

Structure:
```typescript
export class DiffCard extends StatelessWidget {
  readonly filePath: string;
  readonly diff: string;

  build(context: BuildContext): Widget {
    const borderSide = new BorderSide({ color: Color.brightBlack, width: 1, style: 'rounded' });
    const diffContent = new Column({
      mainAxisSize: 'min',
      crossAxisAlignment: 'stretch',
      children: [
        new Text({ text: new TextSpan({ text: ` ${this.filePath}`, style: { foreground: Color.brightBlack, bold: true } }) }),
        new DiffView({ diff: this.diff, showLineNumbers: true }),
      ],
    });
    const themeData = { ...Theme.defaultTheme(), diffAdded: Color.green, diffRemoved: Color.red };
    return new Padding({
      padding: EdgeInsets.only({ left: 4, right: 2, bottom: 1 }),
      child: new Container({
        decoration: new BoxDecoration({ border: Border.all(borderSide) }),
        padding: EdgeInsets.symmetric({ horizontal: 1 }),
        child: new Theme({ data: themeData, child: diffContent }),
      }),
    });
  }
}
```
