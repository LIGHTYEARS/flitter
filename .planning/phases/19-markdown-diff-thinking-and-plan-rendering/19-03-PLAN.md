# Plan 19-03: ThinkingBlock Widget

**Phase:** 19 — Markdown, Diff, Thinking, and Plan Rendering
**Requirements:** REND-04 (thinking block portion)
**Wave:** 2 (depends on 19-01 for Markdown import; can execute after 19-01)

## Goal

Replace the dim placeholder `[thinking...]` / `[thinking]` text in ChatView with a full `ThinkingBlock` widget that matches Amp's thinking display: collapsible container with status icon, animated BrailleSpinner during streaming, Markdown rendering of thinking content, truncation for very long thinking text, and proper interrupted/completed/streaming states.

## Current State

1. **ChatView thinking rendering** (chat-view.ts:260-273) renders each `ThinkingItem` as a plain dim Text: `[thinking...]` when streaming, `[thinking]` when done. No collapse, no content display, no animation.
2. **ThinkingItem type** (types.ts:60-68) has: `text`, `timestamp`, `isStreaming`, `collapsed` fields. The `collapsed` field exists but is not wired to any visual behavior.
3. **flitter-amp `ThinkingBlock`** (thinking-block.ts) is the reference implementation: StatefulWidget with BrailleSpinner timer, status icon from icon-registry, Markdown content rendering with dim+italic style overrides, disclosure chevron, and streaming/done/interrupted states.
4. **flitter-core `BrailleSpinner`** (braille-spinner.ts) is available and already used by `ToolHeader` in flitter-cli.
5. **flitter-core `Markdown`** supports `styleOverrides` for customizing paragraph and code-block styles.

## Approach

Create a `ThinkingBlock` widget in flitter-cli that ports the flitter-amp reference. Replace `AmpThemeProvider` color lookups with direct `Color.*` constants. Replace `icon()` calls with inline Unicode characters (same pattern used by `tool-icons.ts`). Wire it into ChatView replacing the placeholder.

## Changes

### File 1: `packages/flitter-cli/src/widgets/thinking-block.ts` (NEW)

**Action:** Create new file.

Port from `flitter-amp/src/widgets/thinking-block.ts` with these adaptations:

- Import paths use relative `../../../../flitter-core/...`.
- Replace `AmpThemeProvider.maybeOf(context)` with direct `Color.*` constants:
  - streaming accent -> `Color.magenta`
  - done success -> `Color.green`
  - interrupted warning -> `Color.yellow`
  - muted foreground -> `Color.brightBlack`
  - content foreground -> `Color.defaultColor`
- Replace `icon('tool.status.done')` with `'\u2713'` (checkmark, matching tool-icons.ts).
- Replace `icon('disclosure.collapsed')` with `'\u25B6'` (right triangle).
- Replace `icon('disclosure.expanded')` with `'\u25BC'` (down triangle).
- Import `BrailleSpinner` from `../../../../flitter-core/src/utilities/braille-spinner`.
- Import `Markdown` from `../../../../flitter-core/src/widgets/markdown`.
- Import `ThinkingItem` type from `../state/types`.

**Widget structure** (StatefulWidget):

Constructor: `{ item: ThinkingItem }`

State lifecycle:
- `initState()`: start BrailleSpinner timer (200ms interval) if `item.isStreaming`.
- `didUpdateWidget()`: start/stop spinner on streaming transitions.
- `dispose()`: stop spinner timer.

Build logic:
```
Header line: [statusGlyph] [Thinking] [suffix?] [chevron?]
  - Streaming: BrailleSpinner glyph + "Thinking" in magenta dim
  - Done (has text): checkmark + "Thinking" in green dim + disclosure chevron
  - Done (no text): "Thinking" in yellow dim + "(interrupted)" suffix

Content (when not collapsed and text.length > 0):
  - Truncate text to 10000 characters with "..." suffix
  - Render via Markdown with styleOverrides:
    paragraph: { dim: true, italic: true, foreground: Color.defaultColor }
    code-block: { dim: true }
  - Padded left: 2, right: 2
```

### UAT Criteria

1. **REND-04 — Streaming thinking**: During an assistant response that includes thinking, the thinking block shows an animated BrailleSpinner glyph and "Thinking" label in magenta/accent color. The spinner visibly cycles through braille characters.
2. **REND-04 — Completed thinking**: After thinking completes and the assistant has text, the thinking block shows a green checkmark and "Thinking" label in green dim, with a disclosure chevron.
3. **REND-04 — Thinking content**: When the thinking block is expanded (not collapsed), the thinking text renders as Markdown with dim italic paragraph styling. Code blocks within thinking content are also dim.
4. **REND-04 — Truncation**: Very long thinking text (>10000 chars) is truncated with a "..." suffix to prevent layout explosion.
5. **REND-04 — Interrupted thinking**: If thinking is interrupted (done but empty text), the block shows a yellow "Thinking (interrupted)" label with no disclosure chevron.
6. **REND-04 — Collapse state**: The `collapsed` field on ThinkingItem controls whether content is visible. When collapsed, only the header line renders.
