# Plan 19-04: PlanView Widget and ChatView Integration Completion

**Phase:** 19 — Markdown, Diff, Thinking, and Plan Rendering
**Requirements:** REND-04 (plan block portion)
**Wave:** 1 (independent; can execute in parallel with 19-01 and 19-02)

## Goal

Replace the dim `[plan]` placeholder text in ChatView with a full `PlanView` widget that matches Amp's plan rendering: a "Plan" title header, checklist-style entries with status icons (completed/in_progress/pending), priority tags ([H]/[M]/[L] with color coding), and proper padding.

## Current State

1. **ChatView plan rendering** (chat-view.ts:332-344) renders each `PlanItem` as a plain dim Text: `[plan]`. No entries, no status icons, no priority tags.
2. **PlanItem type** (types.ts:71-81) has: `entries: Array<PlanEntry>` where each `PlanEntry` has `content`, `priority` ('high'|'medium'|'low'), and `status` ('pending'|'in_progress'|'completed').
3. **flitter-amp `PlanView`** (plan-view.ts) is the reference: StatelessWidget with "Plan" title, per-entry status icon + priority tag + content rendering.
4. **tool-icons.ts** already has `todoStatusIcon()` mapping pending/in_progress/completed/cancelled to Unicode characters. The plan status icons use a similar mapping.

## Approach

Create a `PlanView` widget in flitter-cli that ports the flitter-amp reference. Replace `AmpThemeProvider` color lookups with direct `Color.*` constants. Replace `icon('plan.status.*')` calls with inline Unicode characters (consistent with `tool-icons.ts`). Wire it into ChatView replacing the placeholder.

## Changes

### File 1: `packages/flitter-cli/src/widgets/plan-view.ts` (NEW)

**Action:** Create new file.

Port from `flitter-amp/src/widgets/plan-view.ts` with these adaptations:

- Import paths use relative `../../../../flitter-core/...`.
- Import `PlanEntry` type from `../state/types`.
- Replace `AmpThemeProvider.maybeOf(context)` with direct `Color.*` constants:
  - info (Plan title) -> `Color.cyan`
  - success (completed) -> `Color.green`
  - warning (in_progress) -> `Color.yellow`
  - muted (pending) -> `Color.brightBlack`
  - destructive (high priority tag) -> `Color.red`
  - warning (medium priority tag) -> `Color.yellow`
  - muted (low priority tag) -> `Color.brightBlack`
- Replace `icon('plan.status.completed')` with `'\u2713'` (checkmark).
- Replace `icon('plan.status.in_progress')` with `'\u25D4'` (half circle).
- Replace `icon('plan.status.pending')` with `'\u25CB'` (empty circle).

Constructor: `{ entries: PlanEntry[] }`

**Widget layout** (StatelessWidget):
```
Padding (vertical: 1)
  Column
    Text "  Plan" (cyan, bold)
    For each entry:
      Text
        "    " + statusIcon + " " (colored by status)
        + priorityTag + " " (colored by priority, bold)
        + content (colored by status)
```

Priority tag mapping (static method):
- `'high'` -> `[H]` in red
- `'medium'` -> `[M]` in yellow
- `'low'` -> `[L]` in brightBlack
