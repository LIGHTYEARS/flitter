# Plan 20-02: StatusBar Widget

**Phase:** 20 — Status Surfaces, Themes, and Motion
**Requirements:** STAT-03
**Wave:** 1 (no dependencies on other plans in this phase)

## Goal

Add a bottom status bar to flitter-cli that displays contextual state messages on the left (streaming hint, shortcut hint, copy confirmation) and cwd + git branch on the right. The bar occupies exactly 1 row at the bottom of the layout Column, below InputArea. This replaces the `// StatusBar placeholder — Phase 20` comment in `app-shell.ts:458`.

## Current State

1. **AppShell layout** (app-shell.ts:446-460) has a Column with `Expanded(content)`, `InputArea`, and a placeholder comment `// StatusBar placeholder — Phase 20`.
2. **flitter-amp has two reference implementations:**
   - `StatusBar` (status-bar.ts, deprecated) — simple `StatelessWidget` with left/right regions, WaveSpinner during streaming.
   - `BottomGrid` (bottom-grid.ts) — complex `StatefulWidget` with 4-corner overlay system, token usage display, and integrated InputArea. Its `buildBottomLeft()` and `buildBottomRight()` methods contain the status bar logic we need.
3. **AppState** exposes: `isProcessing`, `isStreaming`, `isInterrupted`, `metadata.cwd`, `metadata.gitBranch`.
4. **flitter-core `WaveSpinner`** (wave-spinner.ts) is available — cycles through `[' ', '∼', '≈', '≋', '≈', '∼']` at 200ms.
5. **No `StatusBar` widget exists in flitter-cli** — only the placeholder comment.

## Approach

Create a `StatusBar` StatelessWidget ported from flitter-amp's `BottomGrid.buildBottomLeft()` + `buildBottomRight()` methods (the bottom row only, not the full 4-corner system). The status bar is a single `Row` with an `Expanded` left region and a right cwd/branch `Text`. Wire it into AppShell by replacing the placeholder comment.

### Why StatelessWidget, not StatefulWidget

The status bar itself has no internal state — all values come from props. Animation (WaveSpinner during streaming) is handled by WaveSpinner's own StatefulWidget lifecycle. Unlike BottomGrid which owns a TextEditingController listener, our StatusBar is purely prop-driven.

Port the bottom-row logic from flitter-amp's `BottomGrid.buildBottomLeft()` (lines 378-464) and `buildBottomRight()` (lines 466-493):

```typescript
import { StatelessWidget, Widget, type BuildContext } from '../../../flitter-core/src/framework/widget';
import { Row } from '../../../flitter-core/src/widgets/flex';
import { Expanded } from '../../../flitter-core/src/widgets/flexible';
import { Text } from '../../../flitter-core/src/widgets/text';
import { TextStyle } from '../../../flitter-core/src/core/text-style';
import { TextSpan } from '../../../flitter-core/src/core/text-span';
import { Color } from '../../../flitter-core/src/core/color';
import { Padding } from '../../../flitter-core/src/widgets/padding';
import { EdgeInsets } from '../../../flitter-core/src/layout/edge-insets';
import { SizedBox } from '../../../flitter-core/src/widgets/sized-box';

interface StatusBarProps {
  cwd: string;
  gitBranch: string | null;
  isProcessing: boolean;
  isStreaming: boolean;
  isInterrupted: boolean;
  statusMessage?: string | null;
  copyHighlight?: boolean;
}
```

**StatusBar class structure:**

- Constructor stores all props as readonly fields.
- `build(context)`:
  - Reads colors from hardcoded defaults for now (Phase 20-03 wires theme).
  - `mutedColor = Color.brightBlack`, `keybindColor = Color.blue`.
  - Builds left widget using `buildLeft()`.
  - Builds right widget using `buildRight()`.
  - Returns a `Padding(horizontal: 1)` > `Row([Expanded(left), right])` — height is intrinsically 1 line.

- `buildLeft()` — contextual status (priority order):
  1. `copyHighlight` → `"Copied!"` (green, bold)
  2. `isInterrupted` → `"⚠ Response interrupted"` (yellow)
  3. `isProcessing` → `"Esc"` (blue) + `" to cancel"` (dim)
  4. Idle → `"?"` (blue) + `" for shortcuts"` (dim)
  5. If `statusMessage` is set, it overrides the default processing/idle text.

- `buildRight()` — cwd + branch:
  - Shorten path (`$HOME` → `~`).
  - If `gitBranch`, append ` (branchName)`.
  - Wrap in `─...─` dim text, matching BottomGrid pattern.
  - Return `SizedBox.shrink()` if no cwd.

- `shortenPath(fullPath)` — helper replacing `$HOME` with `~`.
