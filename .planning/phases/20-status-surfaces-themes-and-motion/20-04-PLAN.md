# Plan 20-04: Status Header + Token Usage + Motion

**Phase:** 20 — Status Surfaces, Themes, and Motion
**Requirements:** STAT-05, STAT-06
**Wave:** 2 (depends on 20-01 for CliThemeProvider, 20-02 for StatusBar layout placement)

## Goal

Add a HeaderBar widget above the InputArea that displays streaming status information: WaveSpinner animation + token usage percentage + cost + elapsed time during streaming, and a warning message when interrupted. Port the `WaveSpinner` integration pattern and `formatUsageDisplay`/`formatTokenCount`/`formatElapsed`/`thresholdColor` logic from flitter-amp's BottomGrid `buildTopLeft()` method. Ensure `UsageInfo` in the type system supports the fields needed for display.

## Current State

1. **AppShell layout** (app-shell.ts:446-460, after Plan 20-02) has: `Column([Expanded(content), InputArea, StatusBar])`. There is no header bar above InputArea.
2. **flitter-amp BottomGrid** (bottom-grid.ts:271-376) implements `buildTopLeft()` which shows:
   - During streaming with token usage: `WaveSpinner + " " + "XX% of YYk · $0.0012 · 3s"` with threshold coloring on the percentage.
   - During streaming without usage: `WaveSpinner + " " + "Streaming..."` (dim).
   - When interrupted: `"⚠ Response interrupted"` (yellow).
   - When idle: `SizedBox(height: 1)` (empty space).
3. **`formatTokenCount(count)`** — auto-scales to k/M: `"1.2k"`, `"3.5M"`, `"42"`.
4. **`formatElapsed(ms)`** — compact duration: `"12s"`, `"1m 23s"`, `"1h 2m"`.
5. **`thresholdColor(percent)`** — `<50%` → blue (dim), `50-80%` → yellow, `>80%` → red.
6. **flitter-core `WaveSpinner`** — cycles through `[' ', '∼', '≈', '≋', '≈', '∼']` at 200ms, available at `flitter-core/src/widgets/wave-spinner`.
7. **`UsageInfo`** (types.ts:84-88) currently has `{ size: number; used: number; cost?: { amount: number; currency: string } | null }`. The `inputTokens`, `outputTokens`, `costUsd`, `elapsedMs` fields are **not** on `UsageInfo` — they are separate props on flitter-amp's BottomGrid, fed from `AppState.inputTokens` etc.
8. **`SessionMetadata`** (types.ts:133-142) has `tokenUsage: { input: number; output: number }` but no `costUsd` or `elapsedMs`.
9. **`AppState`** delegates to `SessionState` for `usage: UsageInfo | null`.

## Approach

1. Create a `HeaderBar` StatelessWidget that encapsulates the streaming status display logic (ported from BottomGrid's `buildTopLeft` + `formatUsageDisplay`).
2. Place HeaderBar between `Expanded(content)` and `InputArea` in AppShell's layout Column.
3. Keep `HeaderBar` as a StatelessWidget — animation is handled by `WaveSpinner`'s own StatefulWidget. All data comes from props.
4. Add helper functions `formatTokenCount()`, `formatElapsed()`, and `thresholdColor()` as module-level utilities in the header-bar module.
5. Extend `AppState` to expose the token/cost/elapsed data that `HeaderBar` needs. The data sources already exist — `SessionState` tracks `usage` (which has `size`, `used`, `cost`); `SessionMetadata` has `tokenUsage.input`/`.output`. We just need to expose them through AppState getters.

### Why not WaveSpinner in StatusBar

The StatusBar (Plan 20-02) shows contextual text hints (Esc to cancel, ? for shortcuts). The WaveSpinner + token usage display is semantically different — it's streaming progress information. Keeping them as separate widgets matches flitter-amp's architecture where `buildTopLeft()` (header) and `buildBottomLeft()` (status hints) are distinct visual regions.

**Interface:**

```typescript
  isProcessing: boolean;
  isInterrupted: boolean;

```

**HeaderBar class structure:**

- Constructor stores all props as readonly fields.
- `build(context)`:
  - Reads theme: `const theme = CliThemeProvider.maybeOf(context);`
  - `mutedColor = theme?.base.mutedForeground ?? Color.brightBlack`
  - If `isInterrupted`: return `Text("⚠ Response interrupted")` in warning color.
  - If `isProcessing`:
    - If `tokenUsage` exists: return `Row([WaveSpinner, " ", formatUsageDisplay()])`.
    - Else: return `Row([WaveSpinner, " ", "Streaming..."])` (dim).
  - Else: return `SizedBox({ height: 1 })` (1-row empty space preserving layout).
  - Wrap in `Padding(horizontal: 1)`.

- `formatUsageDisplay(mutedColor)` — private method returning a `Widget`:
  - Compute `percent = size > 0 ? Math.round((used / size) * 100) : 0`.
  - Format: `"XX%"` (threshold colored) + `" of YYk"` (dim) + `" · $0.0012"` (dim, if cost > 0) + `" · 3s"` (dim, if elapsedMs > 0).

**Module-level exported helpers:**

- `formatTokenCount(count: number): string` — `>= 1M` → `"X.XM"`, `>= 1k` → `"X.Xk"`, else `"N"`.
- `formatElapsed(ms: number): string` — `< 60s` → `"Xs"`, `< 60m` → `"Xm Ys"`, else `"Xh Ym"`.
- `thresholdColor(percent: number): Color` — `> 80` → red, `>= 50` → yellow, else blue.
