# Plan 29-01: Handoff State Machine

## Objective

Implement the complete handoff mode lifecycle in flitter-cli, matching AMP's GhR.handoffState exactly: enter/exit/submit/abort state machine, countdown timer with auto-submit, and cross-thread handoff tracking with sourceThreadID/targetThreadID.

## Tasks

### Task 1: Add HandoffState type and HandoffMode to types.ts

**read_first:**
- `packages/flitter-cli/src/state/types.ts` (current types)
- `tmux-capture/amp-source/24_main_app_state.js` (handoffState shape)

**action:**
Add to `types.ts`:
1. `HandoffState` interface with 6 fields matching AMP exactly:
   - `isInHandoffMode: boolean`
   - `isGeneratingHandoff: boolean`
   - `isConfirmingAbortHandoff: boolean`
   - `pendingHandoffPrompt: string | null`
   - `spinner: string | null` (braille character, not a spinner object)
   - `countdownSeconds: number | null`
2. `DEFAULT_HANDOFF_STATE` constant with all-false/null defaults
3. `HandoffRequest` interface for cross-thread handoff:
   - `sourceThreadID: string`
   - `targetThreadID: string`
   - `goal: string`
   - `agentMode: string | null`

**acceptance_criteria:**
- `HandoffState` interface exported
- `DEFAULT_HANDOFF_STATE` constant exported
- `HandoffRequest` interface exported
- TypeScript compiles cleanly

### Task 2: Add handoff state machine methods to AppState

**read_first:**
- `packages/flitter-cli/src/state/app-state.ts` (current AppState)
- `tmux-capture/amp-source/30_main_tui_state.js` (handoff UI state methods)
- `tmux-capture/amp-source/23_ui_features.js` (enterHandoffMode, isProcessing)

**action:**
Add to `AppState`:
1. `handoffState: HandoffState` field initialized from `DEFAULT_HANDOFF_STATE`
2. `_countdownTimer: ReturnType<typeof setInterval> | null` private field
3. `enterHandoffMode()` — sets `isInHandoffMode = true`, notifies
4. `exitHandoffMode()` — resets all handoffState to defaults, clears countdown timer, notifies
5. `submitHandoff(goal: string)` — sets `isGeneratingHandoff = true`, `pendingHandoffPrompt = goal`, clears countdown, delegates to `threadPool.createHandoff()`, on completion calls `exitHandoffMode()` + switches thread
6. `abortHandoffConfirmation()` — two-stage: first call sets `isConfirmingAbortHandoff = true`, second call exits handoff mode
7. `startCountdown(seconds: number)` — starts countdown timer, auto-submits at 0
8. `cancelCountdown()` — clears countdown timer and countdownSeconds
9. Update `isProcessing` getter to include handoff check: `this.handoffState.isGeneratingHandoff`

**acceptance_criteria:**
- All 6 methods functional with correct state transitions
- `isProcessing` includes handoff generating check
- Countdown timer properly cleans up on exit/abort/submit
- Thread switch calls `exitHandoffMode()`
- TypeScript compiles cleanly

### Task 3: Add createHandoff to ThreadPool with cross-thread tracking

**read_first:**
- `packages/flitter-cli/src/state/thread-pool.ts` (current ThreadPool)
- `tmux-capture/amp-source/20_thread_queue_handoff.js` (handoff method)
- `tmux-capture/amp-source/28_thread_pool_class.js` (createHandoff on pool)

**action:**
Add to `ThreadPool`:
1. `pendingHandoff: HandoffRequest | null` field — tracks in-flight handoff
2. `createHandoff(goal: string, options?: { agentMode?: string | null })` method:
   - Captures `sourceThreadID` from `activeThreadContextID`
   - Creates new thread via `createThread()`
   - Sets `pendingHandoff` with source/target/goal
   - Returns the new `ThreadHandle`
3. `completedHandoffs: HandoffRequest[]` array — log of completed handoffs
4. `completeHandoff()` — moves `pendingHandoff` to `completedHandoffs`, clears `pendingHandoff`
5. `getHandoffSourceThreadID()` — returns sourceThreadID if active thread was created by handoff

**acceptance_criteria:**
- `createHandoff()` creates new thread and tracks source/target
- `pendingHandoff` cleared on completion
- `getHandoffSourceThreadID()` returns correct value for handoff-created threads
- Handoff state cleaned up in `dispose()`
- TypeScript compiles cleanly
