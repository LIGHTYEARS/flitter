### Task 2: Create SkillService for loading, caching, querying

**read_first:**
- `packages/flitter-cli/src/state/skill-types.ts` (Task 1 output)
- `tmux-capture/amp-source/03_skills_modal_state_f9T.js` (skills split: builtin:// vs file://)
- `tmux-capture/amp-source/26_toast_skills.js` (addPendingSkill/removePendingSkill on thread worker)

**action:**
Create `packages/flitter-cli/src/state/skill-service.ts` with:

1. `SkillService` class — centralized skill management:
   - `private _skills: SkillDefinition[]` — cached skill list
   - `private _errors: SkillError[]` — cached load errors
   - `private _warnings: SkillWarning[]` — cached load warnings
   - `private _loaded: boolean` — whether initial load has completed
   - `private _listeners: Set<() => void>` — change listeners

2. Public getters:
   - `get skills(): SkillDefinition[]`
   - `get errors(): SkillError[]`
   - `get warnings(): SkillWarning[]`
   - `get isLoaded(): boolean`
   - `get skillCount(): number`
   - `get warningCount(): number`

3. Public methods:
   - `setSkills(skills: SkillDefinition[], errors: SkillError[], warnings: SkillWarning[]): void` — bulk update from external source (tool registry, config), notifies listeners
   - `getSkillByName(name: string): SkillDefinition | undefined` — lookup by name
   - `addListener(fn: () => void): void`
   - `removeListener(fn: () => void): void`

4. Grouping helper (matching AMP's Cq0 function):
   - `groupSkillsByPath(skills: SkillDefinition[], cwd: string): SkillGroup[]`
   - Groups non-builtin skills by their baseDir path prefix
   - Local skills: baseDir contains `.agents/skills/` relative to cwd -> label "Local", pathHint ".agents/skills/"
   - Global skills: baseDir contains `~/.agents/skills/` -> label "Global", pathHint "~/.agents/skills/"
   - Other paths: use relative path as label

5. Path helper (matching AMP's A_R function):
   - `relativizePath(absolutePath: string, cwd: string): string` — make path relative to cwd, using ~ for home dir

### Task 3: Build SkillsModal StatefulWidget with list view

**read_first:**
- `tmux-capture/amp-source/03_skills_modal_m9T.js` (widget constructor props)
- `tmux-capture/amp-source/03_skills_modal_state_f9T.js` (full build method, layout logic)
- `packages/flitter-cli/src/widgets/thread-list.ts` (existing modal widget pattern)
- `packages/flitter-cli/src/widgets/command-palette.ts` (existing overlay widget pattern)

**action:**
Create `packages/flitter-cli/src/widgets/skills-modal.ts` with:

1. `SkillsModal` StatefulWidget (matching AMP m9T):
   - Props matching AMP exactly:
     - `skills: SkillDefinition[]`
     - `errors: SkillError[]`
     - `warnings: SkillWarning[]`
     - `onDismiss: () => void`
     - `onAddSkill?: () => void`
     - `onDocs?: () => void`
     - `onInsertPrompt?: (prompt: string) => void`
     - `onInvokeSkill?: (name: string) => void`
     - `cwd: string`

2. `SkillsModalState` (matching AMP f9T) — list-only view in this plan:
   - `listScrollController = new ScrollController()` with followMode = false
   - `detailScrollController = new ScrollController()` with followMode = false (prepared for 30-02)
   - `selectedSkill: SkillDefinition | null = null` (selection state, 30-02 renders detail)
   - `_viewportHeight = 20` (default viewport for scroll info)

3. `build()` method — LIST VIEW ONLY (no detail panel yet):

   a. **Width computation** matching AMP f9T exactly:
      - `available = screenWidth - 4`
      - `maxHeight = screenHeight - 4`
      - When no selection: `totalWidth = max(60, min(90, available))`
      - When selected: `totalWidth = max(100, min(150, available))`, `listWidth = floor(totalWidth * 2/5)` — but detail widget deferred to 30-02

   b. **Title bar** matching AMP:
      - Row: [SizedBox(w:2), Expanded("Skills (N)" primary+bold), "(o)wner's manual" button, "  ", "(a)dd" button, SizedBox(w:2)]
      - "(o)" and "(a)" use keybind color, rest uses secondary color
      - Buttons wrapped in MouseRegion with onClick if callbacks present

   c. **Skill grouping** matching AMP:
      - Split skills into `builtin` (baseDir starts with "builtin://") and `nonBuiltin`
      - Group nonBuiltin via `groupSkillsByPath()`
      - Render nonBuiltin groups first, then builtin as "Built-in"

   d. **Group rendering** (the L function in AMP):
      - Group header: Padding(h:2) -> Text("Label pathHint", secondary+bold, dim for pathHint)
      - Each skill: Padding(h:6) -> Row [Expanded(flex:1, skill.name), SizedBox(w:2), Expanded(flex:2, description)]
      - Skill name: clickable if onInvokeSkill, highlighted if selected (command+bold vs command)
      - Description: clickable -> selectSkill(), foreground color, maxLines:1, overflow:ellipsis
      - SizedBox(h:1) after each group

   e. **Empty state** matching AMP:
      - When no skills and no errors, OR only builtin skills with no errors:
        - "Skills give the agent specialized knowledge..." text (dim)
        - "or define MCP servers to load on demand." text (dim)
        - SizedBox(h:1)
        - "Create your own:" header (secondary+bold)
        - Two example prompts (quoted, clickable if onInsertPrompt)

   f. **Footer**: Center -> Text "Escape to close" (keybind color for "Escape", dim for rest)

   g. **Scroll + scrollbar** for list:
      - Row [Expanded(SingleChildScrollView(listScrollController, listColumn)), Scrollbar(listScrollController)]

   h. **Outer wrapper** matching AMP:
      - FocusScope(autofocus, onKey: handleKeyEvent)
        - Center
          - ConstrainedBox(minWidth/maxWidth: totalWidth, maxHeight)
            - BoxDecoration(background, Border.all(primary, 1, rounded))
              - Padding(all:1)
                - Row [Expanded(Column [titleBar, SizedBox(h:1), listArea, SizedBox(h:1), footer])]

4. `handleKeyEvent` — basic Escape handling only in this plan:
   - Escape: if selectedSkill -> clear selection; else -> onDismiss()
   - Other keys: return 'ignored' (i/a/o deferred to 30-02)
