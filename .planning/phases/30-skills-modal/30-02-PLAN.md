### Task 2: Implement full keyboard navigation matching AMP

**read_first:**
- `tmux-capture/amp-source/03_skills_modal_state_f9T.js` (handleKeyEvent switch block)
- `packages/flitter-cli/src/widgets/skills-modal.ts` (current handleKeyEvent from 30-01)

**action:**
Replace the basic Escape handler in `SkillsModalState.handleKeyEvent` with the full AMP keyboard mapping:

```
switch (event.key) {
  case 'Escape':
    if (this.selectedSkill) {
      // Back to list — clear selection
      this.setState(() => { this.selectedSkill = null; });
      return 'handled';
    }
    // Close modal
    return this.widget.onDismiss(), 'handled';
  
  case 'i':
    // Invoke selected skill
    if (this.selectedSkill && this.widget.onInvokeSkill) {
      this.widget.onInvokeSkill(this.selectedSkill.name);
      this.widget.onDismiss();
      return 'handled';
    }
    return 'ignored';
  
  case 'a':
    // Add skill (only when no skill selected — list view)
    if (!this.selectedSkill && this.widget.onAddSkill) {
      this.widget.onAddSkill();
      return 'handled';
    }
    return 'ignored';
  
  case 'o':
    // Owner's manual (only when no skill selected — list view)
    if (!this.selectedSkill && this.widget.onDocs) {
      this.widget.onDocs();
      return 'handled';
    }
    return 'ignored';
  
  default:
    return 'ignored';
}
```

Key behavioral rules from AMP:
- `i` only works when a skill IS selected (detail view)
- `a` and `o` only work when NO skill is selected (list view)
- Escape with selection goes back to list; Escape without selection closes modal
- `onInvokeSkill` dismisses the modal after invocation
