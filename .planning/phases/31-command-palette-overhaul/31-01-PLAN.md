# Plan 31-01: Redesign Command Palette Layout

**Wave**: 1
**Requirements**: CPAL-01, CPAL-02, CPAL-03

## Goal

Redesign the CommandPalette widget to match AMP's observed layout: centered box border, ">" search prefix, category+label dual-column format with right-aligned shortcut hints. Remove count indicator.

## Tasks

### Task 1: Add `category` field to CommandItem

**File**: `packages/flitter-cli/src/commands/command-registry.ts`

- Add `readonly category: string` to CommandItem interface
- Add category to every existing command push (use existing categories or infer)
  - shortcut-derived commands: `'general'`
  - thread-*: `'thread'`
  - toggle-tool-calls: `'mode'`
  - insert-file-mention: `'prompt'`

### Task 2: Rewrite CommandPalette build() for AMP layout

**File**: `packages/flitter-cli/src/widgets/command-palette.ts`

Changes:
1. **Title color**: Change from `Color.cyan` to `Color.yellow` (ANSI color 3) — AMP uses bold yellow
2. **Search prefix**: Add ">" text before the TextField input
3. **Remove count indicator**: Delete the `{N}/{M} commands` text entirely
4. **Remove placeholder**: Set TextField placeholder to empty string or remove
5. **maxWidth**: Change from 60 to 80 to match AMP box width
6. **Command list**: Replace SelectionList with custom rendering that shows:
   - Right-aligned dim category text (padded to max category width)
   - Two spaces separator
   - Bold label text
   - Right-aligned shortcut hint (bold cyan "Ctrl" + bold key)
7. **Selected item styling**: Yellow background (ANSI 3) + black foreground (ANSI 0) — matching AMP

### Task 3: Update app-shell CommandItem→SelectionItem mapping

**File**: `packages/flitter-cli/src/widgets/app-shell.ts`

- Pass `category` through to the CommandPalette widget (update the mapping in `_showCommandPalette`)
- The CommandPalette now needs access to category and shortcutHint per item, not just label/value/description

### Task 4: Extend CommandPaletteProps and internal item type

**File**: `packages/flitter-cli/src/widgets/command-palette.ts`

- Define `CommandPaletteItem` type with: `id`, `category`, `label`, `shortcutHint?`
- Replace `SelectionItem[]` with `CommandPaletteItem[]` in props
- Handle keyboard navigation (ArrowUp/Down, Enter, Escape) directly instead of delegating to SelectionList

## Verification

- `npx tsc --noEmit` passes
- Visual match: box border, ">" prefix, category+label columns, right-aligned shortcuts, yellow title, no count
