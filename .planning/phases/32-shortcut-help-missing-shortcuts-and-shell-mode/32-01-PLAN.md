# Plan 32-01: Rebuild Shortcut Help as Dual-Column Widget Embedded in InputArea

**Wave:** 1 (no dependencies)
**Requirements:** SHELP-01, SHELP-02, SHELP-03

## Objective

Replace the current ShortcutHelpOverlay (standalone modal overlay) with an AMP-matching shortcut help widget that renders **inside InputArea** as a `topWidget`. Uses dual-column layout (6 rows x 2 shortcuts = 12 total) with exact AMP `C_R` data.

## Tasks

### Task 1: Create `shortcut-help-data.ts` â€” static shortcut data matching AMP C_R

Create `packages/flitter-cli/src/widgets/shortcut-help-data.ts`:

```typescript
/** AMP C_R equivalent â€” 6 dual-column rows of shortcut pairs. */
export interface ShortcutHelpPair {
  left: { keys: string; description: string };
  right: { keys: string; description: string };
}

/** Column alignment width â€” sum of widest left entry keys + space + description. */
export const SHORTCUT_LEFT_COL_WIDTH = 27;

  { left: { keys: 'Shift+Enter', description: 'newline' }, right: { keys: 'Ctrl+S', description: 'switch modes' } },

```

### Task 2: Create `ShortcutHelpInline` widget â€” AMP v9T equivalent

Create `packages/flitter-cli/src/widgets/shortcut-help-inline.ts`:

- StatelessWidget accepting `submitOnEnter: boolean`
- Reads theme via `CliThemeProvider.of(context)` for `app.keybind` and `colorScheme.foreground`
- Detects tmux via `process.env.TMUX` presence
- Maps `SHORTCUT_HELP_DATA` to Text rows with dual-column alignment
- When `submitOnEnter` is false: replaces row 3 left with `"Ctrl/Cmd+Enter" "submit"`
- When tmux detected: replaces "Shift+Enter" with "Alt+Enter", appends tmux hint row
- Ends with a Divider using `colorScheme.border`
- Returns `Column(crossAxisAlignment: 'start', mainAxisSize: 'min', children: [...rows, divider])`

### Task 3: Add `topWidget` prop to InputArea

In `packages/flitter-cli/src/widgets/input-area.ts`:
- Add `topWidget?: Widget` to `InputAreaProps` and `InputArea` class
- In `InputAreaState.build()`, insert `topWidget` above `autocompleteWrapped` inside the Container child:
  - If topWidget is present: wrap in `Column([topWidget, autocompleteWrapped])`
  - If absent: use `autocompleteWrapped` directly

### Task 4: Wire shortcut help toggle in AppShell

In `packages/flitter-cli/src/widgets/app-shell.ts`:
- Add `isShowingShortcutsHelp: boolean` state field
- Change `_showShortcutHelp()` to toggle `isShowingShortcutsHelp` + `setState()` instead of OverlayManager
- In `build()`, pass `topWidget: this.isShowingShortcutsHelp ? new ShortcutHelpInline({ submitOnEnter: true }) : undefined` to InputArea
- The `?` shortcut's enabled guard also checks `!isShowingShortcutsHelp` OR always allows toggle
- Remove OverlayManager-based SHORTCUT_HELP overlay usage from `_showShortcutHelp()`
- Update Escape handler: if `isShowingShortcutsHelp`, dismiss it first

### Task 5: Mark ShortcutHelpOverlay as Deprecated

In `packages/flitter-cli/src/widgets/shortcut-help-overlay.ts`:
- Add `@deprecated` JSDoc to the class â€” no longer used, replaced by ShortcutHelpInline
