# Plan 32-02: Register Missing Shortcuts

**Wave:** 1 (no dependencies)
**Requirements:** KEYS-01, KEYS-02, KEYS-03, KEYS-04, KEYS-05

## Objective

Register the 5 missing keyboard shortcuts that AMP has but flitter-cli lacks. These are stub registrations â€” they register the binding, show in help, and log when triggered. Full functionality for some (paste images, edit message, thread picker) depends on future phase infrastructure.

## Tasks

### Task 1: Add KEYS-01 â€” Ctrl+V paste images

In `packages/flitter-cli/src/shortcuts/defaults.ts`:
```typescript
registry.register({
  id: 'paste-image',
  binding: { key: 'v', ctrl: true },
  displayKey: 'Ctrl+V',
  description: 'Paste images from clipboard',
  category: 'input',
  action: (ctx): KeyEventResult => {
    ctx.hooks.pasteImage?.();
    return 'handled';
  },
});
```

### Task 2: Add KEYS-02 â€” Shift+Enter / Alt+Enter newline

In `packages/flitter-cli/src/shortcuts/defaults.ts`:
```typescript
registry.register({
  id: 'insert-newline',
  binding: isTmux() ? { key: 'Enter', alt: true } : { key: 'Enter', shift: true },
  displayKey: isTmux() ? 'Alt+Enter' : 'Shift+Enter',
  description: 'Insert newline',
  category: 'input',
  action: (ctx): KeyEventResult => {
    ctx.hooks.insertNewline?.();
    return 'handled';
  },
});
```

Add `isTmux()` utility: `() => Boolean(process.env.TMUX)`

Add `insertNewline` to `ShortcutHooks` interface.

### Task 3: Add KEYS-03 â€” Tab/Shift+Tab navigate messages

In `packages/flitter-cli/src/shortcuts/defaults.ts`:
```typescript
registry.register({
  id: 'navigate-messages-forward',
  binding: { key: 'Tab' },
  displayKey: 'Tab',
  description: 'Navigate to next message',
  category: 'navigation',
  enabled: (ctx) => !ctx.appState.isProcessing,
  action: (ctx): KeyEventResult => {
    ctx.hooks.navigateMessages?.('forward');
    return 'handled';
  },
});

registry.register({
  id: 'navigate-messages-backward',
  binding: { key: 'Tab', shift: true },
  displayKey: 'Shift+Tab',
  description: 'Navigate to previous message',
  category: 'navigation',
  enabled: (ctx) => !ctx.appState.isProcessing,
  action: (ctx): KeyEventResult => {
    ctx.hooks.navigateMessages?.('backward');
    return 'handled';
  },
});
```

Add `navigateMessages` to `ShortcutHooks` interface.

### Task 4: Add KEYS-04 â€” Up arrow edits previous message

In `packages/flitter-cli/src/shortcuts/defaults.ts`:
```typescript
registry.register({
  id: 'edit-previous-message',
  binding: { key: 'ArrowUp' },
  displayKey: 'Up',
  description: 'Edit previous user message',
  category: 'input',
  enabled: (ctx) => !ctx.appState.isProcessing && ctx.hooks.canEditPreviousMessage?.() === true,
  action: (ctx): KeyEventResult => {
    ctx.hooks.editPreviousMessage?.();
    return 'handled';
  },
});
```

Add `editPreviousMessage` and `canEditPreviousMessage` to `ShortcutHooks` interface.

The @@ trigger is an InputArea text-level behavior, not a global shortcut. It's already wired via `onSpecialCommandTrigger` prop. Verify the existing wiring in InputArea and add a stub hook in AppShell if not connected.
