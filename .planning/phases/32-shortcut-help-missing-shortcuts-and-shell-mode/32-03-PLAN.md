# Plan 32-03: Shell Mode Status + Bash Invocation Tracking

**Wave:** 2 (depends on 32-01 for topWidget pattern)
**Requirements:** SHELL-01, SHELL-02

## Objective

Add shell mode status bar indicator and bash invocation tracking to match AMP's shell mode architecture. The shell mode status appears on the InputArea top-left border overlay. Bash invocations are tracked in AppState with show/hide timers.

## Tasks

### Task 1: Add bash invocation state to AppState

In `packages/flitter-cli/src/state/app-state.ts`, add fields:
```typescript
/** Active bash invocations â€” matches AMP bashInvocations array. */
bashInvocations: BashInvocation[] = [];
/** Pending bash invocations by ID â€” matches AMP pendingBashInvocations. */
pendingBashInvocations: Map<string, BashInvocation> = new Map();
/** Timestamps when each invocation was first shown. */
bashInvocationShownAt: Map<string, number> = new Map();
/** Removal timer handles by invocation ID. */
bashInvocationRemoveTimers: Map<string, ReturnType<typeof setTimeout>> = new Map();
/** Current shell mode status for border display: 'shell' | 'hidden' | null. */
currentShellModeStatus: 'shell' | 'hidden' | null = null;
```

Add BashInvocation interface to `state/types.ts`:
```typescript
export interface BashInvocation {
  id: string;
  command: string;
  startedAt: number;
  status: 'running' | 'completed' | 'failed';
}
```

Add methods:
- `addBashInvocation(invocation)` â€” adds to array, sets shownAt, notifies
- `removeBashInvocation(id)` â€” removes from array, clears timer, notifies
- `setShellModeStatus(status)` â€” sets currentShellModeStatus, notifies

### Task 2: Wire shell mode status to InputArea border

In `packages/flitter-cli/src/widgets/app-shell.ts`:
- Read `appState.currentShellModeStatus` and compute `isRunningShell`
- Pass `isRunningShell: appState.bashInvocations.length > 0 || appState.pendingBashInvocations.size > 0` to InputArea

In `packages/flitter-cli/src/widgets/border-builders.ts` (`buildBottomLeftOverlay`):
- Already handles `isRunningShell` â€” verify it shows "Running shell command..." status

For top-left shell mode indicator, update `buildTopLeftOverlay` to accept optional `shellModeStatus` and render "shell mode" or "shell mode (incognito)" text using theme `app.shellMode`/`app.shellModeHidden` colors when active.

### Task 3: Create BashInvocationsWidget

Create `packages/flitter-cli/src/widgets/bash-invocations.ts`:

A simple StatelessWidget that displays running bash commands. AMP reference: BJR widget.
- Accepts `invocations: BashInvocation[]`
- Renders a Column of Text rows, each showing the command string
- Used in the chat area or InputArea depending on AMP placement

Note: The actual display integration depends on the streaming/processing flow which is wired in the session layer. This task creates the widget and state infrastructure; full integration with the bash tool execution pipeline is deferred to when bash_tool support is added.

### Task 4: Wire shell mode detection from InputArea to AppState

In `packages/flitter-cli/src/widgets/input-area.ts`:
- When shell mode changes (detected in `_handleTextChange`), fire a callback `onShellModeChange?: (mode: ShellMode) => void`
- Add this prop to InputAreaProps

In `packages/flitter-cli/src/widgets/app-shell.ts`:
- Pass `onShellModeChange: (mode) => { this.widget.appState.setShellModeStatus(mode === 'background' ? 'hidden' : mode ? 'shell' : null); }` to InputArea
