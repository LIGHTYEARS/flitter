# Plan 33-01: Redesign HITL Dialog Layout

**Phase**: 33 — HITL Confirmation Overhaul
**Wave**: 1 (no dependencies)
**Requirements**: HITL-01, HITL-02, HITL-04

## Goal

Replace the current `PermissionDialog` StatelessWidget (which uses generic `DialogOverlay` → `Dialog` → `SelectionList`) with a new StatefulWidget that renders AMP's exact HITL confirmation layout:
- Full-width bordered box matching InputArea width (HITL-02)
- "Allow [tool_name]?" title with bold styling (HITL-04)
- Command content preview above option buttons (HITL-01)
- Radio-style option list with ▸● / ○ indicators
- Footer hint: "↑↓ navigate • Enter select • Esc cancel"

## Detailed Steps

### Step 1: Extend `PermissionRequest` types

**File**: `packages/flitter-cli/src/state/permission-types.ts`

Add structured content preview to `PermissionRequest`:

```typescript
/** Structured content preview displayed above options in the HITL dialog. */
export interface PermissionContentPreview {
  /** Title line (e.g., "Run this command?", "Allow editing file:"). */
  readonly header: string;
  /** Command string for bash tool calls (e.g., "sleep 60"). */
  readonly command?: string;
  /** Working directory for bash tool calls. */
  readonly cwd?: string;
  /** File path for edit/create tool calls. */
  readonly filePath?: string;
  /** Reason/rule text (e.g., "Matches built-in permissions rule 25: ask Bash"). */
  readonly reason?: string;
  /** Hint text (e.g., "Press ? for full command"). */
  readonly hint?: string;
  /** JSON preview for generic tool calls. */
  readonly json?: string;
}
```

Add `contentPreview` to `PermissionRequest`:
```typescript
export interface PermissionRequest {
  readonly requestId: string;
  readonly toolCall: PermissionToolCall;
  readonly options: ReadonlyArray<PermissionOption>;
  /** Structured content preview for AMP-style HITL dialog. */
  readonly contentPreview?: PermissionContentPreview;
}
```

### Step 2: Rewrite `PermissionDialog` as StatefulWidget

**File**: `packages/flitter-cli/src/widgets/permission-dialog.ts`

Replace entirely. New structure:

```
PermissionDialog (StatefulWidget)
  PermissionDialogState
    _selectedIndex: number
    build():
      Stack (fit: expand)
        Positioned(all:0) → Container(mask)
        Column(mainAxisAlignment: end)
          Container(
            decoration: BoxDecoration(border: rounded, warning color)
            child: Column
              — Title text: "Allow [toolName]?" bold, or content.header
              — Content preview (command/filePath/json)
              — Reason text (dim cyan)
              — Radio options with ▸● / ○ indicators
              — Footer hint
          )
    handleKeyEvent():
      ArrowUp/k → move up
      ArrowDown/j → move down
      Enter → confirm
      Escape → cancel
```

Key layout details matching AMP golden:
- Border uses `base.warning` color, `rounded` style
- Full terminal width (no maxWidth constraint)
- Positioned at bottom of screen (Column mainAxisAlignment: 'end')
- Content: header in bold, `$ command` in green/cyan, reason in dim cyan
- Options: `▸●` for selected (bold blue), `○` for unselected

### Step 3: Update `AppState.requestPermission()` to build content preview

**File**: `packages/flitter-cli/src/state/app-state.ts`

Update the builder in `requestPermission()` to construct `PermissionContentPreview` from the `toolCall` data before passing to the dialog.

Format content based on `toolCall.kind`:
- `bash` → `{header: "Run this command?", command: rawInput.command}`
- `edit_file` → `{header: "Allow editing file:", filePath: rawInput.path}`
- `create_file` → `{header: "Allow creating file:", filePath: rawInput.path}`
- default → `{header: "Allow [toolName]?", json: JSON.stringify(rawInput)}`
