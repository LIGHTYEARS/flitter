# Plan 33-02: Inverted-Color Options and Feedback Input Mode

**Phase**: 33 — HITL Confirmation Overhaul
**Wave**: 2 (depends on 33-01)
**Requirements**: HITL-03, HITL-05

## Goal

Add the remaining AMP HITL features on top of the layout from 33-01:
- Inverted-color block style for options with `[y]`/`[n]`/`[a]` shortcut labels and `[Alt+N]` hints (HITL-03)
- "Provide feedback" / "Deny with feedback" mode for inline text input within the dialog (HITL-05)

## Detailed Steps

### Step 1: Add `reject_with_feedback` option kind

**File**: `packages/flitter-cli/src/state/permission-types.ts`

Add to `PermissionOptionKind`:
```typescript
export type PermissionOptionKind =
  | 'allow_once'
  | 'allow_always'
  | 'reject_once'
  | 'reject_always'
  | 'reject_with_feedback';
```

Add feedback response type:
```typescript
/** Result of a permission dialog interaction. */
export type PermissionResult =
  | { type: 'simple'; value: string }
  | { type: 'deny-with-feedback'; feedback: string }
  | null;
```

### Step 2: Implement inverted-color option rendering with Alt+N labels

**File**: `packages/flitter-cli/src/widgets/permission-dialog.ts`

Update `PermissionDialogState.build()`:

Each option row renders as:
```
▸● Approve [Alt+1]           (selected: bold, primary color, ▸● prefix)
 ○ Allow All for This Session [Alt+2]  (unselected: ○ prefix, dim shortcut)
 ○ Allow All for Every Session [Alt+3]
 ○ Deny with feedback [Alt+4]
```

AMP rendering pattern per option:
- Selected: `▸●` prefix, label in bold + primary color, `[Alt+N]` in dim cyan
- Unselected: ` ○` prefix, label in default color, `[Alt+N]` in dim cyan

Add `handleKeyEvent()` handling for Alt+1..Alt+9:
- Alt+1 selects and confirms option[0]
- Alt+2 selects and confirms option[1]
- etc.

### Step 3: Implement feedback input mode

**File**: `packages/flitter-cli/src/widgets/permission-dialog.ts`

Add state fields:
```typescript
private _feedbackInputActive = false;
private _feedbackText = '';
```

When user selects "Deny with feedback" (option with kind `reject_with_feedback`):
- Set `_feedbackInputActive = true`
- Render feedback UI instead of options:

```
✗ Denied — tell Amp what to do instead
› [text input with cursor]
Enter send  •  Esc cancel
```

Layout (matching AMP `buildFeedbackInput()`):
- Title: "✗ Denied" (bold red) + " — " (dim) + "tell Amp what to do instead" (foreground)
- Input row: "› " (bold primary) + TextInput (with placeholder)
- Footer: "Enter" (primary) + " send" (dim) + "  •  " (dim) + "Esc" (primary) + " cancel" (dim)
- Bordered with primary color, rounded

Key handling in feedback mode:
- Enter → submitFeedback (dispatch `{type: "deny-with-feedback", feedback}`)
- Escape → exit feedback mode, return to options
- All other keys → forward to text input

### Step 4: Update `AppState` response handling

**File**: `packages/flitter-cli/src/state/app-state.ts`

Update `requestPermission()` and `resolvePermission()` to handle the new compound `PermissionResult` type.

The `onSelect` callback from the dialog must distinguish between:
- Simple selection: `{type: 'simple', value: optionId}`
- Feedback: `{type: 'deny-with-feedback', feedback: text}`

## Verification

1. `bunx tsc --noEmit -p packages/flitter-cli/tsconfig.json` passes
2. Options show `[Alt+1]`..`[Alt+N]` shortcut labels in dim cyan
3. Alt+1..Alt+N directly selects and confirms the corresponding option
4. Selected option shows `▸●` prefix with bold blue text
5. Unselected options show ` ○` prefix with default text
6. Selecting "Deny with feedback" switches to feedback input UI
7. Feedback UI shows "✗ Denied" header, text input with placeholder, Enter/Esc footer
8. Enter in feedback mode submits feedback text
9. Escape in feedback mode returns to option selection
10. Empty feedback submission falls back to simple "no" response
