# Plan 34-01 — ActivityGroup Widget with Tree-Lines and Summary (Wave 1)

**Requirements:** ACTV-01, ACTV-02, ACTV-03
**Wave:** 1 (no dependencies)

## Objective

Build the ActivityGroup StatefulWidget (G1R equivalent) that renders a collapsible
group of subagent actions with tree-line visual hierarchy, BrailleSpinner animation
for active state, and summary aggregation for collapsed state.

## Files

### New
- `packages/flitter-cli/src/widgets/activity-group.ts`

### Modified
- `packages/flitter-cli/src/utils/icon-registry.ts` — add tree-line character icons

## Implementation Steps

### Step 1: Add tree-line icons to icon-registry.ts

Add these entries to the ICONS object:
```
'tree.branch': '├── '   // non-last child connector
'tree.leaf':   '╰── '   // last child connector
'tree.vertical': '│   '  // continuation line
```

Note: these are multi-character strings (4 chars), not single glyphs. They are
used as string prefixes, not measured as icon width.

### Step 2: Create ActivityGroup StatefulWidget

Port z1R/G1R from AMP source with these exact behaviors:

**Props (ActivityGroupProps):**
- `actions: ActivityAction[]` — child actions in this group
- `summary: { success: number; error: number }` — aggregation counts
- `hasInProgress: boolean` — whether any action is currently running

**ActivityAction type:**
- `kind: 'tool_call' | 'text'` — discriminator
- `toolCall?: ToolCallItem` — for tool_call actions
- `text?: string` — for text message actions
- `isLast: boolean` — computed flag for tree-line rendering

**State (ActivityGroupState):**
- `expanded: boolean` (default false)
- `_spinner: BrailleSpinner` — 200ms animation interval
- `_animationTimer: interval | null`
- `visibleActionCount: number` — staggered reveal
- `_pendingAppendTimer: timeout | null`

**Build:**
1. Header row:
   - Active: `"{spinner.toBraille()} " ` in theme.app.toolRunning color
   - Inactive: `"✓ "` in theme.app.toolSuccess color
   - Summary text: `"Subagent"` bold + description dim + chevron ▼/▶
   - Collapsed: additionally shows `"N ✓ | M ✗"` count
   
2. Body (when expanded):
   - For each visible action (up to visibleActionCount):
     - Prefix: `├── ` (dim) for non-last, `╰── ` (dim) for last
     - Content: ToolHeader row for tool_call, plain Text for text
   - 4-space indent for tree structure

**Lifecycle:**
- initState: set visibleActionCount = actions.length, start animation if active
- didUpdateWidget: handle action list changes, animation state transitions
- dispose: stop all timers

### Step 3: Wire ActivityGroup into ChatView

Modify `buildAssistantTurnWidget()` in chat-view.ts:
- For root tool calls that are Task/sa__/tb__ types, group their children
  into an ActivityGroup widget instead of flat rendering
- Pass child tool calls as ActivityAction items with kind='tool_call'
- Compute summary from child tool statuses

## Verification

- TypeScript compiles without errors (`bun run typecheck`)
- ActivityGroup renders spinner when hasInProgress=true
- Collapsed state shows checkmark/cross summary counts
- Expanded state shows tree-line hierarchy with ├── and ╰── characters
- Staggered reveal animation works (90ms per action)
