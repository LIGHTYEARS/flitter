# Plan 35-01: Image Paste Handler, popImage, Kitty Protocol, Image Preview

**Wave**: 1 (no dependencies)
**Requirements**: IMG-01, IMG-02, IMG-03, IMG-04

## Objective

Implement the complete image attachment pipeline matching AMP's GhR class:
- Ctrl+V paste handler with clipboard image reading and isUploadingImageAttachments spinner
- Backspace popImage() to remove last attached image
- Kitty graphics protocol detection and image rendering
- Full-screen image preview overlay with save dialog

## Files to Create

### `packages/flitter-cli/src/utils/clipboard-image.ts`
- `readImageFromClipboard(): Promise<ImageAttachment | null>` — reads image from system clipboard
- Uses platform detection: `pngpaste -` on macOS, `xclip -selection clipboard -t image/png -o` on Linux
- Returns `{ data: Buffer, mimeType: string, width: number, height: number }` or null
- AMP pattern: async with try/catch, returns null on failure

### `packages/flitter-cli/src/utils/kitty-graphics.ts`
- `supportsKittyGraphics(): boolean` — detect Kitty protocol support via TERM_PROGRAM / terminal query
- `transmitImage(imageId: number, data: Buffer, opts: { width?: number, height?: number }): string` — encode image as Kitty protocol escape sequence
- `clearImage(imageId: number): string` — clear a transmitted image
- AMP pattern: `kittyGraphics` module with 8 references, imageId counter, base64 chunked transmission

### `packages/flitter-cli/src/widgets/image-preview-overlay.ts`
- `ImagePreviewOverlay` StatefulWidget matching AMP's ImagePreview (57 refs)
- Props: `imageData: Buffer`, `mimeType: string`, `onDismiss: () => void`, `onSave?: (path: string) => void`
- Layout: FocusScope (absorbs keys) > Center > bordered container with Kitty image render
- Escape dismisses, 's' opens save dialog
- If Kitty not supported, show placeholder text with dimensions
