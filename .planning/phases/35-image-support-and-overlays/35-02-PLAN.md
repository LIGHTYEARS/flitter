# Plan 35-02: Toast Notification System + Generic Confirmation Overlay

**Wave**: 1 (no dependencies)
**Requirements**: OVLY-01, OVLY-02

## Objective

Implement the toast notification system and generic confirmation overlay matching AMP's patterns:
- ToastController with show(), auto-dismiss timer, type-based styling
- Generic confirmation overlay (yes/no) for exit, clear input, cancel processing

## Files to Create

### `packages/flitter-cli/src/widgets/toast-overlay.ts`
- `ToastController` class matching AMP's `x1R`:
  - `show(message: string, type?: 'success' | 'error' | 'info', durationMs?: number): void`
  - Default duration: 3000ms for success, 5000ms for error, 3000ms for info
  - Auto-dismiss via setTimeout
  - Listener pattern (addListener/removeListener) matching OverlayManager
  - `current: { message: string, type: ToastType } | null` getter
  - `dismiss(): void` — clears current toast
- `ToastOverlay` StatelessWidget:
  - Layout: anchored bottom-center, non-modal (priority 10)
  - Styling: success=green, error=red, info=cyan text on dim background
  - Single-line text with type-colored icon prefix (checkmark/x/info)
  - No FocusScope (non-modal, doesn't absorb keys)

### `packages/flitter-cli/src/widgets/confirmation-overlay.ts`
- `ConfirmationOverlay` StatefulWidget matching AMP's `TTT/aTT` pattern:
  - Props: `title: string`, `message?: string`, `onConfirm: () => void`, `onCancel: () => void`, `onDismiss: () => void`
  - Layout: FocusScope (modal) > Center > bordered container
  - Title in bold, message in dim
  - Option buttons: `[y] Yes  [n] No` with inverted-color block style
  - Key handling: 'y' → onConfirm, 'n' → onCancel, Escape → onDismiss
- AMP state fields: `isConfirmingExit`, `isConfirmingClearInput`, `isConfirmingCancelProcessing`
- AMP timeouts: `exitConfirmTimeout`, `clearInputConfirmTimeout`, `cancelProcessingConfirmTimeout`
