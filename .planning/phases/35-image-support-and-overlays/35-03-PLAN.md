# Plan 35-03: Context Detail, Context Analyze, File Changes Overlays

**Wave**: 2 (depends on 35-02 for overlay infrastructure patterns)
**Requirements**: OVLY-03, OVLY-04, OVLY-05

## Objective

Implement three information overlays matching AMP's GhR state pattern:
- Context detail: token breakdown shown when clicking context percentage
- Context analyze: modal with dependency analysis data
- File changes: overlay listing all files modified in current session

## Files to Create

### `packages/flitter-cli/src/widgets/context-detail-overlay.ts`
- `ContextDetailOverlay` StatelessWidget matching AMP's `isShowingContextDetailOverlay`:
  - Props: `usage: UsageInfo | null`, `contextWindowPercent: number`, `onDismiss: () => void`
  - Layout: FocusScope (modal) > Center > bordered container
  - Content rows: "Input tokens: N", "Output tokens: N", "Cache read: N", "Cache write: N"
  - Progress bar showing context window fill level
  - Cost breakdown if cost data available
  - Escape dismisses

### `packages/flitter-cli/src/widgets/context-analyze-overlay.ts`
- `ContextAnalyzeOverlay` StatelessWidget matching AMP's `isShowingContextAnalyzeModal`:
  - Props: `items: ConversationItem[]`, `usage: UsageInfo | null`, `onDismiss: () => void`
  - Layout: FocusScope (modal) > Center > bordered container with scrollable content
  - Sections: "Messages" count, "Tool calls" count, "System prompt" size estimate
  - Per-message token estimate breakdown (simple heuristic: chars / 4)
  - AMP's `contextAnalyzeDeps` pattern: list dependencies (files mentioned, tools used)

### `packages/flitter-cli/src/widgets/file-changes-overlay.ts`
- `FileChangesOverlay` StatelessWidget matching AMP's `isShowingFileChangesOverlay`:
  - Props: `files: FileChangeEntry[]`, `onDismiss: () => void`
  - Layout: FocusScope (modal) > Center > bordered container
  - Each file: status icon (+ green for created, ~ yellow for modified, - red for deleted) + relative path
  - Empty state: "No files changed in this session"
  - Escape dismisses
