<objective>
消除 factory.ts 中冗余的适配器套适配器层（adapter-of-adapter）。

当前 factory.ts 重新实现了 @mariozechner/pi-ai 已原生提供的功能：provider 映射（PROVIDER_MAP/REVERSE_PROVIDER_MAP）、model 解析（resolveModel + DEFAULT_MODELS）、显示名称（PROVIDER_NAMES）。这些都是 pi-ai 的职责，应当删除。

保留 pi-ai 无法完成的逻辑：
1. OAuth token-store — chatgpt-codex/copilot/antigravity 从磁盘加载令牌
2. baseUrl 覆盖 — 用于 OpenAI 兼容代理（Volcano Engine Ark 等）
3. antigravity User-Agent 注入 — pi-ai 不注入每个 provider 的自定义 header
4. DISPLAY_NAMES — 少量需要特殊大小写的名称（xAI、OpenAI 等）

Purpose: 减少约 120 行冗余映射代码，消除 flitter-cli 与 pi-ai 之间的双重 ID 命名系统，让 pi-ai 成为 provider/model 元数据的唯一来源。
Output: 简化后的 factory.ts（~120 行 → ~80 行），更新后的 provider.ts、config.ts、auth.test.ts。
</objective>

**factory.ts 重写：**

删除以下常量和函数（约 120 行冗余代码）：
- `PROVIDER_MAP` (L21-37) — 不再需要 flitter-cli ID → pi-ai ID 映射
- `REVERSE_PROVIDER_MAP` (L44-68) — 不再需要反向映射
- `PROVIDER_NAMES` (L73-89) — 大部分用 capitalize 或 KnownProvider 直接派生
- `DEFAULT_MODELS` (L96-112) — 改用 `getModels(provider)[0].id`
- `resolveModel()` (L122-153) — 前缀匹配不必要，直接用 getModel()

保留并重构：

1. **DISPLAY_NAMES**（新增，约 8 行）— 仅保留需要特殊大小写的名称：
```typescript
const DISPLAY_NAMES: Record<string, string> = {
  'xai': 'xAI',
  'openai': 'OpenAI',
  'openai-codex': 'ChatGPT / Codex',
  'github-copilot': 'GitHub Copilot',
  'google-antigravity': 'Antigravity (Gemini)',
  'google': 'Google Gemini',
  'google-vertex': 'Google Vertex',
  'kimi-coding': 'Moonshot (Kimi)',
};
```
其余 provider 使用 capitalize 函数派生（首字母大写）。

2. **getDisplayName(providerId: string): string** — 先查 DISPLAY_NAMES，否则 capitalize。

3. **getDefaultModel(provider: string): string** — 调用 `getModels(provider as KnownProvider)`，返回 `[0].id`。如果 getModels 抛出异常则 fallback 到 `'claude-sonnet-4-20250514'`。此函数替代 DEFAULT_MODELS 常量。导出此函数供 config.ts 使用。

4. **createProvider(config: ProviderConfig): Provider** — 简化流程：
   - 删除 `PROVIDER_MAP[config.id]` 间接层 → 直接用 `config.id` 作为 pi-ai provider key
   - model 解析：`config.model ?? getDefaultModel(config.id)`
   - API key 解析：保留 OAuth switch/case 不变（chatgpt-codex/copilot/antigravity），default 分支调用 `getEnvApiKey(config.id as KnownProvider)`
   - 模型获取：`let model = getModel(config.id as KnownProvider, modelId as any)`，如果抛出异常则 `try { model = getModels(config.id as KnownProvider)[0]; }` 作为 fallback
   - baseUrl 覆盖：保持不变
   - display name：调用 `getDisplayName(config.id)`
   - antigravity User-Agent：保持不变（检查 `config.id === 'google-antigravity'`，注意 ID 已改为 pi-ai 命名）
   - 返回 `new PiAiProvider(model, config.id as ProviderId, displayName, apiKey, headers)`

5. **autoDetectProvider(): ProviderConfig | null** — 简化：
   - 直接 `for (const p of getProviders())` + `getEnvApiKey(p)` → `return { id: p as ProviderId, apiKey: key }`
   - 删除 `REVERSE_PROVIDER_MAP[p]` 间接层 — 直接使用 pi-ai provider 名称
   - OAuth fallback 保持不变，但 ID 更新：`hasValidToken('chatgpt-codex')` → `return { id: 'openai-codex' }`，`hasValidToken('copilot')` → `return { id: 'github-copilot' }`，`hasValidToken('antigravity')` → `return { id: 'google-antigravity' }`

重要：OAuth token-store 的 key 名称（'chatgpt-codex', 'copilot', 'antigravity'）是磁盘上的文件名，不能改。但 createProvider 的 OAuth switch/case 需要匹配新的 pi-ai ProviderId，即 `case 'openai-codex':` / `case 'github-copilot':` / `case 'google-antigravity':`。loadToken() 调用仍然传入旧的 token-store key。
