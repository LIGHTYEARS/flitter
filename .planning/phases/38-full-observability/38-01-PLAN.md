# Plan 38-01: Tracer infrastructure

<objective>
Build the core tracing module (TraceStore) with span lifecycle operations aligned to AMP's traceStore pattern, and extend logger.ts with a generic `writeEntry()` function and `kind` field so spans and structured errors can be emitted alongside regular logs in the same NDJSON file.
</objective>

<must_haves>
- TraceStore with startSpan / addSpanEvent / setSpanAttributes / endSpan operations
- Span data model matching 38-CONTEXT.md schema (traceId, spanId, parentSpanId, name, startTime, endTime, duration, attributes, events, status)
- `writeEntry()` in logger.ts that bypasses level checking and writes raw NDJSON entries to the existing logStream
- All existing log entries gain `kind: "log"` field; new span entries use `kind: "span"`; structured errors use `kind: "error"`
- TraceStore operations are safe to call even when no log file is open (no-op / stderr fallback)
</must_haves>

## Task 1: Add `writeEntry()` function and `kind` field to logger.ts

<read_first>
- packages/flitter-cli/src/utils/logger.ts
</read_first>

<action>
1. In `logger.ts`, add a new exported function `writeEntry` after the `closeLogFile()` function (around line 90):

```typescript
/**
 * Write a raw NDJSON entry to the log file, bypassing level checks.
 * Used by the tracer for span records and structured error records.
 * Falls back to stderr JSON output if no log file is open.
 */
export function writeEntry(entry: Record<string, unknown>): void {
  const line = JSON.stringify(entry) + '\n';
  if (logStream) {
    logStream.write(line);
  } else {
    process.stderr.write(line);
  }
}
```

2. Modify the `buildJsonEntry()` function (line 138) to add `kind: "log"` to all regular log entries. Change:
```typescript
const entry: Record<string, unknown> = {
  ts: new Date().toISOString(),

};
```
to:
```typescript
const entry: Record<string, unknown> = {
  kind: 'log',
  ts: new Date().toISOString(),
  level: level.toUpperCase(),

```

3. Modify `writeErrorLog()`: when writing to logStream, overwrite `kind` to `"error"` for error/fatal levels. After `const entry = buildJsonEntry(level, message, err, data);` (line 207), add:
```typescript

```
This makes error-level entries discoverable via `jq 'select(.kind == "error")'`.

Create a new file `packages/flitter-cli/src/utils/tracer.ts` with the following contents:

1. **SpanEvent interface:**
```typescript
/** A timestamped event recorded within a span. */
export interface SpanEvent {
  name: string;
  time: string;        // ISO 8601
  attributes?: Record<string, unknown>;
}
```

2. **Span interface:**
```typescript
/** A completed or in-flight trace span, emitted as a single NDJSON entry on close. */
export interface Span {
  kind: 'span';
  traceId: string;
  spanId: string;
  parentSpanId: string | null;
  name: string;
  startTime: string;    // ISO 8601
  endTime?: string;     // ISO 8601, set on close
  duration?: number;    // ms, computed on close
  attributes: Record<string, unknown>;
  events: SpanEvent[];
  status: 'ok' | 'error';
}
```

3. **SpanHandle type:**
```typescript
/** Opaque handle returned by startSpan, used to reference active spans. */
export interface SpanHandle {
  readonly spanId: string;
  readonly traceId: string;
}
```

4. **TraceStore class** with the following members and methods:

- Private `_spans: Map<string, Span>` — active (unclosed) spans keyed by spanId
- Private `_generateId(): string` — returns `crypto.randomUUID()` (Bun built-in, zero deps)

- `startSpan(name: string, parentSpanId: string | null, attributes?: Record<string, unknown>, traceId?: string): SpanHandle`
  - Generate spanId via `_generateId()`
  - If traceId not provided and parentSpanId is not null: look up parent span and inherit traceId. If parent not found, generate new traceId.
  - If traceId not provided and parentSpanId is null: generate new traceId via `_generateId()`
  - Create Span object: `{ kind: 'span', traceId, spanId, parentSpanId, name, startTime: new Date().toISOString(), attributes: attributes ?? {}, events: [], status: 'ok' }`
  - Store in `_spans`
  - Return `{ spanId, traceId }`

- `addSpanEvent(spanId: string, name: string, attributes?: Record<string, unknown>): void`
  - Look up span in `_spans`; if not found, return (no-op)
  - Push `{ name, time: new Date().toISOString(), attributes }` into span.events

- `setSpanAttributes(spanId: string, attributes: Record<string, unknown>): void`
  - Look up span in `_spans`; if not found, return (no-op)
  - `Object.assign(span.attributes, attributes)`

- `endSpan(spanId: string, status?: 'ok' | 'error', finalAttributes?: Record<string, unknown>): void`
  - Look up span in `_spans`; if not found, return (no-op)
  - If `finalAttributes` provided, `Object.assign(span.attributes, finalAttributes)`
  - Set `span.endTime = new Date().toISOString()`
  - Set `span.duration = new Date(span.endTime).getTime() - new Date(span.startTime).getTime()`
  - If status provided, set `span.status = status`
  - Call `writeEntry(span as unknown as Record<string, unknown>)` from logger.ts
  - Remove from `_spans`

5. **createTracer factory function** (aligned with AMP's createTracer/DLR pattern):
```typescript
/**
 * Create a child-span factory bound to a parent span.
 * Aligned with AMP's createTracer(parentSpan) → DLR(traceStore, parentSpan) pattern.
 */
export function createTracer(store: TraceStore, parentHandle: SpanHandle): {
  startActiveSpan: <T>(name: string, attributes: Record<string, unknown>, fn: (handle: SpanHandle) => Promise<T>) => Promise<T>;
} {
  return {
    async startActiveSpan<T>(name: string, attributes: Record<string, unknown>, fn: (handle: SpanHandle) => Promise<T>): Promise<T> {
      const child = store.startSpan(name, parentHandle.spanId, attributes, parentHandle.traceId);
      try {
        return await fn(child);
      } catch (err) {
        store.endSpan(child.spanId, 'error');
        throw err;
      } finally {
        // endSpan is idempotent (no-op if already ended above)
        store.endSpan(child.spanId, undefined);
      }
    },
  };
}
```

Wait — `endSpan` should be truly idempotent: if span is not in `_spans`, it's a no-op. The error path calls `endSpan('error')` which removes from map, then finally calls `endSpan(undefined)` which finds nothing — no-op. Correct.

But the finally should not overwrite the error status. Fix: change finally to only call endSpan if the span is still active:
```typescript
finally {
  // Only end if not already ended by the catch block
  if (store.isActive(child.spanId)) {
    store.endSpan(child.spanId, 'ok');
  }
}
```

Add an `isActive(spanId: string): boolean` method to TraceStore:
```typescript
/** Check if a span is still active (not yet ended). */
isActive(spanId: string): boolean {
  return this._spans.has(spanId);
}
```

6. **Module-level singleton** for convenience:
```typescript
/** Module-level TraceStore singleton for the CLI process. */
export const traceStore = new TraceStore();
```

7. Import `writeEntry` from logger.ts at the top of tracer.ts:
```typescript

```
