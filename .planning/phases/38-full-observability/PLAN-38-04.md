<action>
Create `packages/flitter-cli/src/__tests__/tracer.test.ts` with the following test cases using `bun:test` (`describe`, `test`, `expect`):

```typescript
import { describe, test, expect, beforeEach, afterEach, mock } from 'bun:test';
```

**Test suite: TraceStore**

1. **"startSpan returns a unique spanId"** — call `startSpan('agent')` twice, assert the returned IDs are different strings.

2. **"startSpan with parent creates child span"** — call `startSpan('agent')` → get parentId, then `startSpan('inference', parentId)` → get childId. Assert both are non-empty strings and parentId !== childId.

3. **"endSpan writes a span record to logger"** — mock `writeEntry` from `../utils/logger`. Call `startSpan('agent')`, then `endSpan(spanId, 'ok')`. Assert `writeEntry` was called with an object containing:
   - `kind: 'span'`
   - `name: 'agent'`
   - `spanId: <the returned id>`
   - `status: 'ok'`
   - `startTime` is an ISO 8601 string
   - `endTime` is an ISO 8601 string
   - `duration` is a non-negative number

4. **"endSpan with attributes merges into span record"** — start span, end with `endSpan(id, 'ok', { model: 'gpt-4' })`. Assert writeEntry was called with `attributes.model === 'gpt-4'`.

5. **"addSpanEvent records events on the span"** — start span, add event `addSpanEvent(id, 'first-token', { ttftMs: 150 })`, end span. Assert writeEntry output contains `events` array with one entry having `name: 'first-token'` and `attributes.ttftMs: 150`.

6. **"setSpanAttributes merges attributes"** — start span, set attributes `{ a: 1 }`, then set `{ b: 2 }`, end span. Assert both `a: 1` and `b: 2` are in the final span record's attributes.

7. **"endSpan on already-ended span is a no-op"** — start span, end span twice. Assert writeEntry called exactly once.

8. **"startSpan returns SpanHandle with spanId and traceId"** — call `startSpan('agent')`, assert result has both `spanId` (non-empty string) and `traceId` (non-empty string).

9. **"nested spans share the same traceId"** — start parent span, start child span with `parentHandle.spanId`. Assert child's `traceId === parent's traceId` (both from SpanHandle).

Mock strategy: mock `writeEntry` from `'../utils/logger'` using `mock.module()` or by importing and overriding. The tracer module should be testable with just the writeEntry mock.
</action>

2. **"pipeline sink emits frame-overrun warning for slow frames"** — Get the sink function installed by `initPipelineBridge()`. 4. **"pipeline sink ignores non-FRAME tags"** — Call sink with `('BUILD', 'some build message')`. Assert `log.warn` was NOT called.

5. **"teardownPipelineBridge resets the sink"** — mock `resetPipelineLogSink` from flitter-core. Call `teardownPipelineBridge()`. Assert `resetPipelineLogSink` was called.

Mock strategy: mock both `../utils/logger` (for `log.warn`) and `flitter-core` (for `setPipelineLogSink`/`resetPipelineLogSink`).
